# Pre-Commit Checklist

Чек-лист проверки перед коммитом изменений.

---

## Чек-лист (7 пунктов)

- [ ] **1.1** Изменения протестированы локально
- [ ] **1.2** Код проходит линтинг
- [ ] **1.3** Форматирование применено (prettier, black, etc.)
- [ ] **1.4** Сообщение коммита следует Conventional Commits
- [ ] **1.5** Изменения атомарные (один коммит = одна задача)
- [ ] **1.6** Нет незакоммиченных отладочных изменений
- [ ] **1.7** Pre-commit хуки прошли успешно

---

## Conventional Commits Формат

```
<type>(<scope>): <description>

[optional body]

[optional footer(s)]
```

**Типы:**
- `feat:` Новая функция
- `fix:` Исправление ошибки
- `docs:` Изменения документации
- `style:` Форматирование, отступы
- `refactor:` Рефакторинг кода
- `test:` Добавление тестов
- `chore:` Обновление зависимостей, инструменты

---

## Использование

```bash
# Автоматическая проверка
.qwen/checklists/validate-checklist.sh --name "Pre-Commit"

# Pre-commit review скрипт
.qwen/scripts/git/pre-commit-review.sh "feat: add new feature"
```
