# Release Checklist

Чек-лист для выпуска релиза.

---

## Чек-лист (12 пунктов)

- [ ] **1.1** Все тесты проходят успешно
- [ ] **1.2** Все PR закрыты или перенесены
- [ ] **1.3** CHANGELOG обновлен
- [ ] **1.4** Версия обновлена (SemVer)
- [ ] **1.5** Тег создан (vX.Y.Z)
- [ ] **1.6** Релиз задокументирован
- [ ] **1.7** Зависимости обновлены
- [ ] **1.8** Безопасность проверена
- [ ] **1.9** Документация актуализирована
- [ ] **1.10** Артефакты собраны
- [ ] **1.11** Уведомления отправлены
- [ ] **1.12** Post-release проверка пройдена

---

## Процесс релиза

### 1. Подготовка
```bash
# Проверка тестов
npm test

# Проверка качества
.qwen/scripts/run_quality_gate.sh
```

### 2. Обновление версии
```bash
# Обновление .version
echo "1.0.0" > .version

# Обновление package.json
npm version 1.0.0 --no-git-tag-version
```

### 3. CHANGELOG
```markdown
## [1.0.0] - 2026-03-21

### Added
- Новая функция 1
- Новая функция 2

### Changed
- Изменение 1

### Fixed
- Исправление 1
```

### 4. Создание тега
```bash
git add .
git commit -m "chore: release v1.0.0"
git tag -a v1.0.0 -m "Release v1.0.0"
git push origin v1.0.0
```

---

## Использование

```bash
# Автоматическая проверка
.qwen/checklists/validate-checklist.sh --name "Release"

# Auto-tag скрипт
.qwen/scripts/git/auto-tag-release.sh "v1.0.0" "Release v1.0.0"
```
