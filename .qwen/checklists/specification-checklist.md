# Specification Checklist

Чек-лист проверки спецификаций.

---

## Чек-лист (9 пунктов)

- [ ] **1.1** spec.md существует для задачи
- [ ] **1.2** Требования четко сформулированы
- [ ] **1.3** Требования тестируемы
- [ ] **1.4** Критерии приемки определены
- [ ] **1.5** Границы задачи определены
- [ ] **1.6** Зависимости документированы
- [ ] **1.7** Риски идентифицированы
- [ ] **1.8** Сроки реалистичны
- [ ] **1.9** Стейкхолдеры согласовали

---

## Шаблон спецификации

```markdown
# Specification: {Название}

## Цель
{Описание цели}

## Требования
1. {Требование 1}
2. {Требование 2}

## Критерии приемки
- [ ] Критерий 1
- [ ] Критерий 2

## Границы
**Включено:**
- {что включено}

**Не включено:**
- {что не включено}

## Зависимости
- {зависимость 1}

## Риски
- {риск 1}

## Сроки
- Начало: {дата}
- Конец: {дата}
```
