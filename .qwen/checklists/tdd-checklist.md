# TDD Checklist

Чек-лист для Test-Driven Development.

---

## Чек-лист (6 пунктов)

- [ ] **1.1** Тест написан до реализации (Red)
- [ ] **1.2** Тест падает как ожидается (Red)
- [ ] **1.3** Минимальная реализация для прохождения теста (Green)
- [ ] **1.4** Тест проходит успешно (Green)
- [ ] **1.5** Рефакторинг кода (Refactor)
- [ ] **1.6** Все тесты проходят после рефакторинга

---

## TDD Цикл

```
┌─────────────┐
│    Red      │ ← Написать падающий тест
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   Green     │ ← Написать минимальную реализацию
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  Refactor   │ ← Улучшить код, сохранить тесты
└──────┬──────┘
       │
       ▼
   (повтор)
```

---

## Правила TDD

1. **Не писать тест без реализации**
2. **Не писать больше теста чем нужно**
3. **Не писать больше кода чем нужно для теста**
