## Ввод пользователя

```text
$ARGUMENTS
```

Вы **ДОЛЖНЫ** учитывать ввод пользователя перед продолжением (если не пустой).

## Описание команды

Команда `/health-bugs` запускает полный рабочий процесс поиска и анализа ошибок в коде проекта согласно паттерну **Command → Agent → Skill**:

```
Command (health-bugs.md)
  ↓ Agent tool
Agent (bug-hunter с预загруженным skill: bug-hunter)
  ↓ Skill tool
Skill (bug-hunter - специализированный анализ кода)
```

## Параметры

- `$ARGUMENTS` - необязательные параметры для настройки анализа:
  - Конкретные файлы или директории для проверки
  - Типы ошибок для поиска (logic, security, performance)
  - Уровень детализации отчета (brief, detailed)

## Процесс выполнения

1. **Command уровень:** Получает пользовательский ввод и маршрутизирует к агенту
2. **Agent уровень:** 
   - Собирает контекст проекта (читает файлы кода)
   - Определяет область анализа
   - Вызывает навык bug-hunter
3. **Skill уровень:**
   - Выполняет специализированный анализ кода
   - Классифицирует найденные проблемы
   - Генерирует структурированный отчет

## Возвращаемое значение

Команда возвращает:
- **Отчет о найденных ошибках** с описанием каждой проблемы
- **Классификацию по критичности** (Critical, High, Medium, Low)
- **Рекомендации по устранению** для каждой проблемы
- **Статистику анализа** (количество файлов, строк, проблем)

## Примеры использования

### Поиск всех типов ошибок
```bash
/health-bugs
```

### Анализ конкретной директории
```bash
/health-bugs src/api/
```

### Поиск только уязвимостей безопасности
```bash
/health-bugs --type=security src/auth/
```
