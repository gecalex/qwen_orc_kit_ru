## Ввод пользователя

```text
$ARGUMENTS
```

Вы **ДОЛЖНЫ** учитывать ввод пользователя перед продолжением (если не пустой).

## Описание команды

Команда `/health-deps` запускает полный рабочий процесс аудита зависимостей проекта. Она использует агента `dependency-analyzer` для анализа файлов зависимостей и выявления потенциальных проблем.

## Параметры

- `$ARGUMENTS` - необязательные параметры для настройки анализа (например, конкретные файлы зависимостей для проверки)

## Процесс выполнения

1. Собирает файлы зависимостей проекта (package.json, requirements.txt, Cargo.toml и т.д.)
2. Запускает агента `dependency-analyzer` с полным контекстом и критериями валидации
3. Агент анализирует зависимости на наличие проблем
4. Генерирует отчет о состоянии зависимостей
5. Предлагает рекомендации по улучшению зависимости

## Возвращаемое значение

Команда возвращает:
- Отчет о зависимостях с описанием найденных проблем
- Рекомендации по обновлению зависимостей
- Приоритезированный список проблем для исправления
- Статистику по типам и количеству проблем с зависимостями
