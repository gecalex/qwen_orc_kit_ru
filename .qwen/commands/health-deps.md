## Ввод пользователя

```text
$ARGUMENTS
```

Вы **ДОЛЖНЫ** учитывать ввод пользователя перед продолжением (если не пустой).

## Описание команды

Команда `/health-deps` запускает полный рабочий процесс аудита зависимостей проекта согласно паттерну **Command → Agent → Skill**:

```
Command (health-deps.md)
  ↓ Agent tool
Agent (dependency-analyzer с预загруженным skill: dependency-auditor)
  ↓ Skill tool
Skill (dependency-auditor - специализированный аудит зависимостей)
```

## Параметры

- `$ARGUMENTS` - необязательные параметры для настройки анализа:
  - Конкретные файлы зависимостей для проверки
  - Типы проблем для поиска (outdated, vulnerable, unused, conflicts)
  - Уровень детализации отчета (brief, detailed)

## Процесс выполнения

1. **Command уровень:** Получает пользовательский ввод и маршрутизирует к агенту
2. **Agent уровень:** 
   - Собирает файлы зависимостей проекта
   - Определяет менеджер пакетов и экосистему
   - Вызывает навык dependency-auditor
3. **Skill уровень:**
   - Выполняет специализированный аудит зависимостей
   - Классифицирует найденные проблемы по приоритету
   - Генерирует структурированный отчет с рекомендациями

## Возвращаемое значение

Команда возвращает:
- **Отчет о зависимостях** с описанием найденных проблем
- **Классификацию по приоритету** (Critical, High, Medium, Low)
- **Рекомендации по обновлению** для каждой зависимости
- **Статистику анализа** (количество зависимостей, проблем, уязвимостей)

## Примеры использования

### Полный аудит зависимостей
```bash
/health-deps
```

### Анализ конкретного файла зависимостей
```bash
/health-deps package.json
```

### Поиск только уязвимых зависимостей
```bash
/health-deps --type=vulnerable
```

### Проверка на устаревшие зависимости
```bash
/health-deps --type=outdated
```
