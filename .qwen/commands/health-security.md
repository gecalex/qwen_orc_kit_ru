## Ввод пользователя

```text
$ARGUMENTS
```

Вы **ДОЛЖНЫ** учитывать ввод пользователя перед продолжением (если не пустой).

## Описание команды

Команда `/health-security` запускает полный рабочий процесс обнаружения и анализа уязвимостей безопасности в проекте согласно паттерну **Command → Agent → Skill**:

```
Command (health-security.md)
  ↓ Agent tool
Agent (security-analyzer с预загруженным skill: security-analyzer)
  ↓ Skill tool
Skill (security-analyzer - специализированный анализ безопасности)
```

## Параметры

- `$ARGUMENTS` - необязательные параметры для настройки анализа:
  - Конкретные файлы или директории для проверки
  - Типы уязвимостей для поиска (sql-injection, xss, csrf, credentials)
  - Уровень детализации отчета (brief, detailed)

## Процесс выполнения

1. **Command уровень:** Получает пользовательский ввод и маршрутизирует к агенту
2. **Agent уровень:** 
   - Собирает контекст проекта (читает файлы кода, конфигурации)
   - Определяет область анализа безопасности
   - Вызывает навык security-analyzer
3. **Skill уровень:**
   - Выполняет специализированный анализ на уязвимости
   - Классифицирует найденные проблемы по уровню риска
   - Генерирует структурированный отчет о безопасности

## Возвращаемое значение

Команда возвращает:
- **Отчет о безопасности** с описанием найденных уязвимостей
- **Классификацию по уровню риска** (Critical, High, Medium, Low, Info)
- **Рекомендации по устранению** для каждой уязвимости
- **Статистику анализа** (количество файлов, проверенных паттернов, уязвимостей)

## Примеры использования

### Полный анализ безопасности
```bash
/health-security
```

### Анализ конкретной директории
```bash
/health-security src/api/
```

### Поиск только SQL-инъекций
```bash
/health-security --type=sql-injection src/database/
```

### Проверка на утечки учетных данных
```bash
/health-security --type=credentials .
```
