# SpecKit Command: Analyze

**Назначение:** Анализ требований пользователя

**Версия:** 1.0.0

---

## Описание

Команда `speckit.analyze` выполняет первичный анализ требований от пользователя, собирает информацию, выявляет противоречия и генерирует отчет о требованиях.

## Использование

```bash
/qwen speckit.analyze
```

## Функционал

### 1. Сбор требований
- Запрос начальных требований у пользователя
- Документирование функциональных требований
- Документирование нефункциональных требований
- Идентификация стейкхолдеров

### 2. Анализ полноты
- Проверка наличия всех обязательных элементов
- Выявление недостающей информации
- Оценка детализации требований
- Проверка измеримости критериев успеха

### 3. Выявление противоречий
- Поиск конфликтующих требований
- Выявление технических противоречий
- Обнаружение зависимостей между требованиями
- Идентификация рисков

### 4. Генерация отчета
- Создание `specs/{ID}/requirements.md`
- Структурирование требований по категориям
- Приоритизация требований (MoSCoW)
- Рекомендации по уточнению

## Выходные артефакты

- `specs/{ID}/requirements.md` - Документ требований
- `specs/{ID}/analysis-report.md` - Отчет об анализе
- `specs/{ID}/questions.md` - Список вопросов для уточнения

## Интеграция

**Следующая команда:** `speckit.specify`

**Зависимости:** Нет

## Пример выполнения

```markdown
# Requirements Analysis Report

**Статус:** ✅ Завершено
**Дата:** 2026-03-21
**Агент:** spec_analyzer

## Собранные требования
- Функциональные: 15 требований
- Нефункциональные: 8 требований
- Ограничения: 5 ограничений

## Анализ полноты
- Полнота: 75%
- Недостающие элементы: 3 пункта

## Выявленные противоречия
- Противоречие 1: Описание и контекст
- Противоречие 2: Описание и контекст

## Рекомендации
1. Уточнить требование X
2. Добавить детали к требованию Y
3. Разрешить противоречие Z
```
