# SpecKit Command: Checklist

**Назначение:** Создание чек-листа приемки

**Версия:** 1.0.0

---

## Описание

Команда `speckit.checklist` генерирует детальный чек-лист приемки на основе спецификации, включая критерии приемки и acceptance tests.

## Использование

```bash
/qwen speckit.checklist
```

## Функционал

### 1. Генерация checklist.md
- Использование шаблона чек-листа
- Извлечение критериев из spec.md
- Создание проверяемых пунктов
- Организация по категориям

### 2. Критерии приемки
- Функциональные критерии
- Нефункциональные критерии
- Критерии качества кода
- Критерии документации
- Критерии производительности

### 3. Acceptance tests
- Создание сценариев тестирования
- Определение ожидаемых результатов
- Установка критериев прохождения
- Документирование шагов воспроизведения

### 4. Валидация чек-листа
- Проверка полноты покрытия
- Верификация измеримости критериев
- Подтверждение выполнимости тестов
- Утверждение стейкхолдерами

## Выходные артефакты

- `specs/{ID}/checklist.md` - Основной чек-лист
- `specs/{ID}/acceptance-tests.md` - Acceptance тесты
- `specs/{ID}/acceptance-criteria.md` - Критерии приемки
- `specs/{ID}/validation-report.md` - Отчет валидации

## Интеграция

**Предыдущая команда:** `speckit.implement`

**Следующая команда:** `speckit.tasks`

**Зависимости:** `specs/{ID}/spec.md`, `specs/{ID}/implementation-summary.md`

## Пример структуры checklist.md

```markdown
# Acceptance Checklist: {Project Name}

## Функциональные критерии
- [ ] FR-001: Пользователь может аутентифицироваться
  - Тест: Ввод корректных учетных данных
  - Ожидаемый результат: Успешный вход, редирект на dashboard
- [ ] FR-002: Пользователь может создать новый ресурс
  - Тест: Заполнение формы создания
  - Ожидаемый результат: Ресурс создан, отображается в списке

## Нефункциональные критерии
- [ ] NFR-001: Время отклика API < 200ms
  - Тест: Load test с 100 запросами
  - Ожидаемый результат: 95% запросов < 200ms
- [ ] NFR-002: Покрытие тестами > 80%
  - Тест: Запуск coverage отчета
  - Ожидаемый результат: Coverage ≥ 80%

## Качество кода
- [ ] QC-001: Нет критических предупреждений линтера
- [ ] QC-002: Все тесты проходят
- [ ] QC-003: Документация актуальна

## Итоговый статус
- Пройдено: 0/15
- Не пройдено: 0/15
- Не тестировалось: 15/15
```
