# SpecKit Command: Clarify

**Назначение:** Уточнение спецификации

**Версия:** 1.0.0

---

## Описание

Команда `speckit.clarify` анализирует спецификацию на предмет пробелов, неоднозначностей и недостающей информации, генерирует вопросы для пользователя и обновляет спецификацию.

## Использование

```bash
/qwen speckit.clarify
```

## Функционал

### 1. Анализ пробелов в spec.md
- Проверка полноты каждого раздела
- Выявление неоднозначных формулировок
- Обнаружение недостающих деталей
- Проверка согласованности требований

### 2. Вопросы пользователю
- Генерация списка уточняющих вопросов
- Приоритизация вопросов по важности
- Категоризация по разделам спецификации
- Форматирование для удобного ответа

### 3. Обновление спецификации
- Интеграция ответов пользователя
- Переписывание неоднозначных разделов
- Добавление недостающих деталей
- Версионирование изменений

### 4. Валидация уточнений
- Проверка разрешения неоднозначностей
- Подтверждение полноты ответов
- Обновление связанных разделов
- Фиксация окончательных решений

## Выходные артефакты

- `.qwen/specify/.qwen/specify/specs/{ID}/spec.md` - Обновленная спецификация
- `.qwen/specify/.qwen/specify/specs/{ID}/clarification-questions.md` - Список вопросов
- `.qwen/specify/.qwen/specify/specs/{ID}/clarification-answers.md` - Ответы пользователя
- `.qwen/specify/.qwen/specify/specs/{ID}/spec-changelog.md` - История изменений

## Интеграция

**Предыдущая команда:** `speckit.specify`

**Следующая команда:** `speckit.plan`

**Зависимости:** `.qwen/specify/.qwen/specify/specs/{ID}/spec.md`

## Пример вопросов для уточнения

```markdown
# Clarification Questions

## Раздел 2: Функциональные требования
1. **FR-003:** Какой метод аутентификации предпочтителен? (JWT/OAuth2/Session)
2. **FR-007:** Требуется ли поддержка мобильных устройств?

## Раздел 3: Нефункциональные требования
3. **NFR-002:** Какой целевой показатель времени отклика? (<100ms, <500ms, <1s)
4. **NFR-005:** Какие требования к доступности? (99%, 99.9%, 99.99%)

## Раздел 6: Критерии успеха
5. **SC-001:** Как измерять "улучшение производительности"? (метрики, базовые значения)

## Приоритет
- 🔴 Высокий: Вопросы 1, 3, 5
- 🟡 Средний: Вопросы 2, 4
- 🟢 Низкий: Остальные
```
