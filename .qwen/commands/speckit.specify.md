# SpecKit Command: Specify

**Назначение:** Создание спецификации проекта

**Версия:** 1.0.0

---

## Описание

Команда `speckit.specify` генерирует полную спецификацию проекта на основе требований, создавая документ `spec.md` со всеми обязательными разделами.

## Использование

```bash
/qwen speckit.specify
```

## Функционал

### 1. Генерация spec.md по шаблону
- Использование шаблона `.qwen/specify/templates/spec-template.md`
- Адаптация под конкретный проект
- Заполнение всех разделов

### 2. 9 обязательных разделов
1. **Обзор проекта** - Цель, контекст, стейкхолдеры
2. **Функциональные требования** - Детальное описание функций
3. **Нефункциональные требования** - Производительность, безопасность, масштабируемость
4. **Ограничения** - Технические, временные, ресурсные ограничения
5. **Предположения** - Допущения и зависимости
6. **Критерии успеха** - Измеримые метрики успеха
7. **Архитектурные решения** - Высокоуровневая архитектура
8. **Интерфейсы** - API, UI, интеграции
9. **Риски** - Идентифицированные риски и митигация

### 3. Критерии успеха
- Определение измеримых метрик
- Установление пороговых значений
- Определение методов валидации
- Создание acceptance criteria

### 4. Ограничения и предположения
- Документирование всех ограничений
- Фиксация предположений
- Оценка влияния на проект
- План управления рисками

## Выходные артефакты

- `.qwen/specify/.qwen/specify/specs/{ID}/spec.md` - Основная спецификация
- `.qwen/specify/.qwen/specify/specs/{ID}/spec-summary.md` - Краткое резюме
- `.qwen/specify/.qwen/specify/specs/{ID}/glossary.md` - Глоссарий терминов

## Интеграция

**Предыдущая команда:** `speckit.analyze`

**Следующая команда:** `speckit.clarify`

**Зависимости:** `.qwen/specify/.qwen/specify/specs/{ID}/requirements.md`
