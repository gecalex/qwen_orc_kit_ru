# SpecKit Command: Tasks

**Назначение:** Генерация задач и матрицы traceability

**Версия:** 1.0.0

---

## Описание

Команда `speckit.tasks` создает документ tasks.md с полной матрицей traceability, приоритизацией задач и назначением агентов.

## Использование

```bash
/qwen speckit.tasks
```

## Функционал

### 1. Создание tasks.md
- Использование шаблона `.qwen/specify/templates/task-template.md`
- Генерация задач из плана и спецификации
- Структурирование по категориям
- Добавление метаданных

### 2. Матрица traceability
- Связывание задач с требованиями
- Отслеживание покрытия требований
- Выявление непокрытых требований
- Поддержание актуальности связей

### 3. Приоритизация
- Оценка важности каждой задачи
- Оценка срочности выполнения
- Расчет приоритета (важность × срочность)
- Сортировка по приоритету

### 4. Назначение агентов
- Определение требуемых навыков
- Выбор подходящего агента
- Назначение ответственных
- Балансировка нагрузки

## Выходные артефакты

- `.qwen/specify/.qwen/specify/specs/{ID}/tasks.md` - Основной документ задач
- `.qwen/specify/.qwen/specify/specs/{ID}/traceability-matrix.md` - Матрица traceability
- `.qwen/specify/.qwen/specify/specs/{ID}/agent-assignments.md` - Назначения агентов
- `.qwen/specify/.qwen/specify/specs/{ID}/priority-board.md` - Доска приоритетов

## Интеграция

**Предыдущая команда:** `speckit.checklist`

**Следующая команда:** `speckit.constitution`

**Зависимости:** `.qwen/specify/.qwen/specify/specs/{ID}/spec.md`, `.qwen/specify/.qwen/specify/specs/{ID}/plan.md`, `.qwen/specify/.qwen/specify/specs/{ID}/checklist.md`

## Пример структуры tasks.md

```markdown
# Tasks: {Project Name}

## Матрица Traceability

| Задача | Требования | Статус | Приоритет | Агент |
|--------|------------|--------|-----------|-------|
| T-001 | FR-001, NFR-001 | Pending | P0 | work_dev_specialist |
| T-002 | FR-002, FR-003 | Pending | P0 | work_dev_specialist |
| T-003 | NFR-002, QC-001 | Pending | P1 | work_test_specialist |
| T-004 | DOC-001 | Pending | P2 | work_doc_writer |

## Задачи P0 (Критические)
### T-001: Реализация аутентификации
- **Описание:** JWT аутентификация пользователей
- **Требования:** FR-001, NFR-001
- **Оценка:** 4 часа
- **Агент:** work_dev_specialist
- **Зависимости:** Нет

### T-002: CRUD операции
- **Описание:** Создание, чтение, обновление, удаление ресурсов
- **Требования:** FR-002, FR-003
- **Оценка:** 6 часов
- **Агент:** work_dev_specialist
- **Зависимости:** T-001

## Задачи P1 (Важные)
### T-003: Тестирование
- **Описание:** Unit и integration тесты
- **Требования:** NFR-002, QC-001
- **Оценка:** 4 часа
- **Агент:** work_test_specialist
- **Зависимости:** T-001, T-002

## Задачи P2 (Желательные)
### T-004: Документация
- **Описание:** API документация
- **Требования:** DOC-001
- **Оценка:** 2 часа
- **Агент:** work_doc_writer
- **Зависимости:** T-002
```
