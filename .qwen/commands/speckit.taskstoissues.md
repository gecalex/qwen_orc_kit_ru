## Ввод пользователя

```text
$ARGUMENTS
```

Вы **ДОЛЖНЫ** учитывать ввод пользователя перед продолжением (если не пусто).

## Процесс выполнения

1. Запустить `.qwen/specify/scripts/bash/check-prerequisites.sh --json --require-tasks --include-tasks` из корня репозитория и проанализировать список FEATURE_DIR и AVAILABLE_DOCS. Все пути должны быть абсолютными. Для одинарных кавычек в аргументах, таких как "I'm Groot", используйте экранирующий синтаксис: например 'I'\''m Groot' (или двойные кавычки, если возможно: "I'm Groot").
1. Из выполненного скрипта извлечь путь к **tasks**.
1. Получить удаленный репозиторий Git, выполнив:

```bash
git config --get remote.origin.url
```

> [!ВНИМАНИЕ]
> ПРОДОЛЖАЙТЕ К СЛЕДУЮЩИМ ШАГАМ ТОЛЬКО ЕСЛИ УДАЛЕННЫЙ РЕПОЗИТОРИЙ ЯВЛЯЕТСЯ URL-АДРЕСОМ GITHUB

1. Для каждой задачи в списке используйте сервер GitHub MCP для создания новой проблемы в репозитории, которая соответствует удаленному Git-репозиторию.

> [!ВНИМАНИЕ]
> НИКОГДА НЕ СОЗДАВАЙТЕ ПРОБЛЕМЫ В РЕПОЗИТОРИЯХ, КОТОРЫЕ НЕ СООТВЕТСТВУЮТ УДАЛЕННОМУ URL-АДРЕСУ
