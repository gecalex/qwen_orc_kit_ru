# template-switcher

**Назначение:** Переключение между режимами шаблона (быстрый старт ↔ полный)

---

## Использование

```bash
# Переключиться на быстрый старт
qwen template-switcher quickstart

# Переключиться на полный шаблон
qwen template-switcher full

# Проверить статус
qwen template-switcher status

# Справка
qwen template-switcher help
```

---

## Алиасы

| Команда | Описание |
|---------|----------|
| `template-switcher quickstart` | Переключиться на быстрый старт |
| `template-switcher qs` | Короткая версия |
| `template-switcher q` | Самая короткая версия |
| `template-switcher full` | Переключиться на полный шаблон |
| `template-switcher f` | Короткая версия |
| `template-switcher status` | Показать текущий режим |
| `template-switcher s` | Короткая версия |
| `template-switcher help` | Показать справку |

## Что делает скрипт

### Быстрый старт (quickstart):

**Оставляет:**
- ✅ 7 агентов (базовые оркестраторы + bug-fixer/hunter)
- ✅ 10 навыков (валидация, безопасность, анализ)
- ✅ 5 команд (Quality Gates, security, orchestration)
- ✅ 3 скрипта (анализ состояния, security, bugs)

**Удаляет:**
- ❌ Все специализированные агенты (backend, frontend, planning, research)
- ❌ Большинство навыков (21 из 31)
- ❌ Большинство команд (17 из 22)
- ❌ Большинство скриптов (~17 из ~20)

### Полный шаблон (full):

**Восстанавливает:**
- ✅ Все 25 агентов
- ✅ Все 31 навыков
- ✅ Все 22 команды
- ✅ Все ~20 скриптов
- ✅ Всю документацию (40 файлов)

## Резервное копирование

Перед переключением в режим быстрого старта скрипт **автоматически создает резервную копию**:

```
.qwen/.backup-20260318-143022/
├── agents/
├── skills/
└── commands/
```

**Для ручного возврата** используйте резервную копию:
```bash
cp -r .qwen/.backup-*/agents/* .qwen/agents/
cp -r .qwen/.backup-*/skills/* .qwen/skills/
cp -r .qwen/.backup-*/commands/* .qwen/commands/
```

---

## Кэш шаблонов

При первом переключении в полный режим скрипт создает кэш:

```
.qwen/template-cache/
├── agents/      # Полный набор агентов
├── skills/      # Полный набор навыков
├── commands/    # Полный набор команд
├── scripts/     # Полный набор скриптов
└── docs/        # Полная документация
```

**При последующих переключениях** используется кэш.

## Примеры использования

### Пример 1: Быстрый старт нового проекта

```bash
# 1. Создаем новый проект
mkdir my-new-project
cd my-new-project

# 2. Копируем шаблон
cp -r /path/to/qwen-orchestrator-kit/* .

# 3. Переключаемся на быстрый старт
qwen template-switcher quickstart

# Готово! Проект настроен для быстрой разработки
```

### Пример 2: Переключение в полный режим

```bash
# Проект вырос, нужны все возможности
qwen template-switcher full

# Все компоненты восстановлены!
```

## Автоматическое определение режима

Скрипт автоматически определяет текущий режим по количеству агентов:

- **≤ 10 агентов** → Быстрый старт
- **> 10 агентов** → Полный шаблон

---

## Предупреждения

⚠️ **Перед переключением:**
1. Убедитесь, что все изменения закоммичены
2. Скрипт создает резервную копию, но лучше иметь git commit
3. Переключение может занять 5-10 секунд

⚠️ **После переключения:**
1. Проверьте, что все необходимые компоненты на месте
2. Запустите `qwen template-switcher status` для проверки
3. При проблемах восстановите из резервной копии

## Устранение проблем

### Проблема: Скрипт не находит кэш

**Решение:**
```bash
# Создайте кэш вручную
mkdir -p .qwen/template-cache
cp -r .qwen/agents/* .qwen/template-cache/agents/
cp -r .qwen/skills/* .qwen/template-cache/skills/
cp -r .qwen/commands/* .qwen/template-cache/commands/
```

### Проблема: Ошибка при переключении

**Решение:**
1. Проверьте права на файлы
2. Убедитесь, что ни один файл не открыт в редакторе
3. Восстановите из резервной копии

### Проблема: Пропали нужные агенты

**Решение:**
```bash
# Переключитесь на полный режим
qwen template-switcher full

# Или восстановите из резервной копии
cp -r .qwen/.backup-*/agents/* .qwen/agents/
```
