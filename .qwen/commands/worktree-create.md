## Ввод пользователя

```text
$ARGUMENTS
```

Вы **ДОЛЖНЫ** учитывать ввод пользователя перед продолжением (если не пустой).
Ожидается аргумент в формате "feature-name" или "feature/description".

## Описание команды

Команда `/worktree-create` создает новый worktree для изолированной разработки фичи. Это позволяет разрабатывать несколько фич параллельно без загрязнения основной ветки.

## Параметры

- `$ARGUMENTS` - название фичи для создания worktree (например, "feature/new-auth-system")

## Процесс выполнения

1. Проверяет, является ли текущий каталог Git-репозиторием
2. Создает новую ветку для фичи на основе текущей ветки
3. Создает новый worktree в подкаталоге `.worktrees/`
4. Переключается в новый worktree
5. Выводит информацию о созданном worktree

## Возвращаемое значение

Команда возвращает:
- Информацию о созданном worktree
- Путь к новому worktree
- Инструкции по работе с новым worktree
