# Индекс агентов Qwen Orchestrator Kit

## Обзор

Этот файл содержит индекс всех агентов в системе Qwen Orchestrator Kit, сгруппированных по функциональности и доменам. Он предназначен для улучшения навигации и поиска подходящих агентов для выполнения задач.

## Структура индекса

Агенты в этом индексе сгруппированы по следующим категориям:
- **Оркестраторы (Orchestrators)** - координируют выполнение задач между другими агентами
- **Воркеры (Workers)** - выполняют конкретные задачи
- **Специализированные агенты** - выполняют специфические функции

Каждый агент в индексе имеет:
- Имя файла
- Тип (оркестратор/воркер)
- Домен (dev, frontend, backend, testing, research, security, planning, health, doc)
- Краткое описание функций
- Основные инструменты, к которым имеет доступ

## Поиск агентов

Для поиска агента по функциональности используйте следующие критерии:

1. **Определите тип задачи:**
   - Анализ → `code_analyzer`, `security_analyzer`, `dependency_analyzer`
   - Генерация → `component_generator`, `test_generator`, `meta_agent`
   - Валидация → `api_validator`, `quality_checker`
   - Координация → `*_coordinator`, `*_orchestrator`
   - Исправление → `bug_fixer`, `bug_hunter`

2. **Определите домен задачи:**
   - `dev` - общая разработка
   - `frontend` - UI/UX
   - `backend` - API, серверная часть
   - `testing` - тестирование
   - `security` - безопасность
   - `planning` - планирование
   - `health` - здоровье проекта
   - `research` - исследования
   - `doc` - документация

3. **Выберите тип агента:**
   - Оркестратор - для координации сложных задач
   - Воркер - для выполнения конкретных задач
   - Специализированный - для уникальных функций
