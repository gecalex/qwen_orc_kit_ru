# Соглашение об именовании агентов

## Обзор

Этот документ описывает систему именования агентов в проекте Qwen Code CLI. Система именования позволяет легко определять тип и область применения агента.

## Структура именования

Агенты именуются по следующему шаблону:

```
{префикс}_{домен}_{имя_агента}.md
```

### Префиксы

- `orc_` - для оркестраторов (orchestrators)
- `work_` - для воркеров (workers)

### Домены

Используйте следующие домены:

- `dev` - разработка
- `frontend` - фронтенд
- `backend` - бэкенд
- `doc` - документация
- `testing` - тестирование
- `research` - исследования
- `health` - проверки состояния
- `security` - безопасность

### Имена агентов

Имена агентов должны быть понятными и описательными, отражать основную функцию агента.

## Примеры

### Оркестраторы

- `orc_dev_task_coordinator.md` - оркестратор для координации задач в домене разработки
- `orc_frontend_ui_coordinator.md` - оркестратор для координации UI задач в фронтенд домене
- `orc_backend_api_coordinator.md` - оркестратор для координации API задач в бэкенд домене
- `orc_testing_quality_assurer.md` - оркестратор для обеспечения качества в тестировании
- `orc_security_security_orchestrator.md` - оркестратор для задач безопасности

### Воркеры

- `work_dev_code_analyzer.md` - воркер для анализа кода в домене разработки
- `work_frontend_component_generator.md` - воркер для генерации компонентов в фронтенд домене
- `work_backend_api_validator.md` - воркер для валидации API в бэкенд домене
- `work_testing_test_generator.md` - воркер для генерации тестов в тестировании
- `work_doc_tech_translator_ru.md` - воркер для перевода технической документации

## Рекомендации

1. Используйте понятные и описательные имена
2. Следуйте шаблону `{префикс}_{домен}_{имя_агента}`
3. Используйте kebab-case для имен агентов
4. Убедитесь, что домен точно отражает область применения агента
5. Избегайте слишком длинных имен (рекомендуется не более 3-4 слов)
