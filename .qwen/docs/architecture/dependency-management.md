# Руководство по управлению зависимостями

## Общие принципы управления зависимостями

### 1. Актуальность
- Регулярно обновляйте зависимости до актуальных версий
- Используйте автоматизированные инструменты для отслеживания устаревших зависимостей
- Проверяйте совместимость при обновлении зависимостей

### 2. Безопасность
- Проверяйте зависимости на наличие известных уязвимостей
- Используйте инструменты для аудита безопасности зависимостей
- Обновляйте зависимости при обнаружении уязвимостей

### 3. Минимизация
- Используйте только необходимые зависимости
- Удаляйте неиспользуемые зависимости
- Предпочитайте легковесные альтернативы при выборе зависимостей

## Типы проблем с зависимостями

### 1. Устаревшие зависимости
- **Описание**: Зависимости, которые не обновлялись длительное время
- **Риски**: Отсутствие исправлений ошибок и уязвимостей, несовместимость с новыми версиями
- **Обнаружение**: Сравнение версий с актуальными в репозиториях

### 2. Уязвимые зависимости
- **Описание**: Зависимости с известными уязвимостями безопасности
- **Риски**: Возможность эксплуатации уязвимостей в приложении
- **Обнаружение**: Проверка через базы данных уязвимостей

### 3. Неиспользуемые зависимости
- **Описание**: Зависимости, которые объявлены, но не используются в коде
- **Риски**: Увеличение размера проекта, потенциальные уязвимости
- **Обнаружение**: Анализ кода на использование зависимостей

### 4. Конфликты зависимостей
- **Описание**: Проблемы совместимости между разными версиями зависимостей
- **Риски**: Ошибки выполнения, непредсказуемое поведение
- **Обнаружение**: Анализ дерева зависимостей

## Инструменты управления зависимостями

### 1. Агент анализа зависимостей
- Используйте команду `/health-deps` для запуска комплексной проверки зависимостей
- Агент анализирует зависимости на наличие устаревших, уязвимых и неиспользуемых компонентов
- Генерирует отчет с рекомендациями по улучшению зависимостей

### 2. Скрипты проверки
- Используйте скрипт `scripts/orchestration-tools/update-dependencies.sh` для аудита зависимостей
- Скрипт автоматически проверяет зависимости на устаревание и уязвимости
- Создает детализированные отчеты с рекомендациями

### 3. Навыки аудита
- Используйте навык `dependency-auditor` для проверки конкретных файлов зависимостей
- Навык может быть интегрирован в другие агенты для проверки зависимостей

## Практики управления зависимостями

### 1. Регулярные проверки
- Проводите регулярные проверки зависимостей на устаревание и уязвимости
- Используйте автоматизированные инструменты для мониторинга зависимостей
- Обновляйте зависимости по мере выхода новых версий с исправлениями

### 2. Lock-файлы
- Используйте lock-файлы для фиксации точных версий зависимостей
- Обновляйте lock-файлы при каждом изменении зависимостей
- Включайте lock-файлы в систему контроля версий

### 3. Автоматизация
- Настройте автоматические проверки зависимостей в CI/CD
- Используйте инструменты для автоматического обновления зависимостей
- Настройте уведомления о новых версиях и уязвимостях зависимостей

## Рекомендации по выбору зависимостей

### 1. Популярность и поддержка
- Предпочитайте зависимости с активной поддержкой
- Проверяйте частоту обновлений и количество звезд/установок
- Оценивайте сообщество вокруг зависимости

### 2. Лицензирование
- Проверяйте лицензии зависимостей на совместимость с проектом
- Учитывайте ограничения открытых лицензий
- Документируйте лицензии используемых зависимостей

### 3. Размер и сложность
- Оценивайте размер зависимости и ее влияние на проект
- Предпочитайте модульные зависимости вместо монолитных библиотек
- Учитывайте сложность интеграции и сопровождения
