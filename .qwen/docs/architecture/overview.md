# Архитектура Qwen Code CLI

## Обзор

Архитектура Qwen Code CLI представляет собой систему оркестрации агентов, предназначенную для автоматизации разработки программного обеспечения. Система состоит из различных типов агентов, навыков, скриптов и вспомогательных компонентов, которые работают вместе для выполнения задач разработки.

## Компоненты системы

### Агенты

Агенты - это основные исполнительные единицы в системе. Они бывают двух типов:

1. **Оркестраторы (orc_*)** - координируют выполнение задач между другими агентами
2. **Воркеры (work_*)** - выполняют конкретные задачи

Агенты именуются по схеме: `{типы}_{домен}_{имя}`, где:
- `orc_` - для оркестраторов
- `work_` - для воркеров
- `{домен}` - область применения (dev, frontend, backend, testing, research и т.д.)

### Навыки (Skills)

Навыки - это переиспользуемые функции, которые могут быть вызваны агентами для выполнения часто используемых операций. Они находятся в директории `.qwen/skills/`.

### Скрипты

Система включает несколько типов скриптов:

1. **Скрипты оркестрации** (`scripts/orchestration-tools/`) - управляют выполнением задач
2. **Скрипты контрольных точек качества** (`scripts/quality-gates/`) - выполняют проверки качества
3. **Скрипты создания агентов** (`scripts/agent-creation/`) - автоматизируют создание новых агентов

## Фаза планирования (Phase 0)

Система включает автоматизированную фазу планирования, которая:
- Анализирует задачи
- Определяет необходимые типы агентов
- Назначает исполнителей
- Создает план выполнения

## Стандартизация отчетов

Все агенты генерируют отчеты в унифицированном формате, который включает:
- Заголовок с метаданными
- Исполнительное резюме
- Выполненная работа
- Внесенные изменения
- Результаты валидации
- Метрики выполнения
- Обнаруженные ошибки
- Следующие шаги
- Артефакты выполнения

## Контрольные точки качества

Система включает контрольные точки качества, которые могут быть:
- Блокирующими (останавливают выполнение при ошибках)
- Неблокирующими (предупреждают о проблемах)

## Интеграция с MCP

Система поддерживает интеграцию с MCP (Model Context Protocol) серверами для расширения возможностей агентов.

## Оркестрация

Оркестраторы поддерживают:
- Итеративные процессы
- Возвращение управления
- Сложные рабочие процессы

## Структура проекта

```
.qwen/
├── agents/              # Агенты
│   ├── orc_{domain}_*.md    # Оркестраторы
│   └── work_{domain}_*.md   # Воркеры
├── skills/              # Навыки
├── templates/           # Шаблоны
└── configs/             # Конфигурации

scripts/
├── orchestration-tools/ # Скрипты оркестрации
├── quality-gates/       # Скрипты контрольных точек качества
└── agent-creation/      # Скрипты создания агентов

state/                   # Временные файлы состояния
└── ...

docs/
└── architecture/        # Документация по архитектуре
    ├── agent-orchestration.md
    ├── quality-gates.md
    └── mcp-integration.md
```

## Рекомендации по разработке

1. Следуйте системе именования агентов
2. Используйте существующие навыки при возможности
3. Генерируйте отчеты в стандартизированном формате
4. Интегрируйте контрольные точки качества в рабочие процессы
5. Используйте фазу планирования для анализа задач
