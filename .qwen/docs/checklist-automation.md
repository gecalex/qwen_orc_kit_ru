# Checklist Automation

Система автоматизации чек-листов для Qwen Orchestrator Kit.

## Обзор

Checklist Automation предоставляет автоматическую валидацию по чек-листам, интерактивное прохождение проверок и сохранение прогресса для обеспечения качества процессов разработки.

## Быстрый старт

### Автоматическая валидация

```bash
# Проверка всех чек-листов
.qwen/checklists/validate-checklist.sh --all

# Проверка конкретного чек-листа
.qwen/checklists/validate-checklist.sh --name "Pre-Flight"

# С авто-исправлением
.qwen/checklists/validate-checklist.sh --name "Pre-Flight" --auto-fix
```

### Интерактивный запуск

```bash
# Запуск чек-листа
.qwen/checklists/checklist-runner.sh "Pre-Flight"

# Список доступных чек-листов
.qwen/checklists/checklist-runner.sh --list

# Возобновить сеанс
.qwen/checklists/checklist-runner.sh --resume

# Статус сохраненных прогрессов
.qwen/checklists/checklist-runner.sh --status
```

## Компоненты

### 1. Validate Checklist

**Назначение:** Автоматическая валидация по чек-листам

**Функционал:**
- Чтение чек-листа из Markdown файла
- Проверка каждого пункта (авто/ручная)
- Вычисление процента выполнения
- Генерация рекомендаций
- Авто-исправление проблем (где возможно)

**Использование:**
```bash
# Все чек-листы
.qwen/checklists/validate-checklist.sh --all

# Конкретный чек-лист
.qwen/checklists/validate-checklist.sh --name "Pre-Flight"

# С авто-исправлением
.qwen/checklists/validate-checklist.sh --name "Initialization" --auto-fix

# Тихий режим
.qwen/checklists/validate-checklist.sh --all -q

# Вывод в другую директорию
.qwen/checklists/validate-checklist.sh --all -o /tmp/reports
```

**Выходные данные:**
```
Pre-Flight Checklist: 8/10 (80%)

✅ Git репозиторий инициализирован
✅ Ветка develop существует
❌ .gitignore существует - НЕ ВЫПОЛНЕНО
❌ constitution.md существует - НЕ ВЫПОЛНЕНО
⚠️ Изменения протестированы - ТРЕБУЕТ ПРОВЕРКИ

Recommendations:
1. .gitignore существует - доступно авто-исправление
2. constitution.md существует - требует ручного исправления
3. Изменения протестированы - проверьте вручную
```

**Типы проверок:**

| Тип | Описание | Примеры |
|-----|----------|---------|
| Auto | Автоматическая проверка | Существование файлов, директорий |
| Manual | Ручная проверка | Качество кода, тесты |
| Auto-fix | Авто-исправление | Создание файлов |

### 2. Checklist Runner

**Назначение:** Интерактивный запуск чек-листов

**Функционал:**
- Пошаговое прохождение чек-листа
- Отметка выполненных пунктов
- Сохранение прогресса
- Генерация отчета о завершении

**Использование:**
```bash
# Запуск чек-листа
.qwen/checklists/checklist-runner.sh "Pre-Flight"

# Список чек-листов
.qwen/checklists/checklist-runner.sh --list

# Возобновление
.qwen/checklists/checklist-runner.sh --resume

# Статус
.qwen/checklists/checklist-runner.sh --status
```

**Команды в интерактивном режиме:**

| Команда | Описание |
|---------|----------|
| `y`, `yes`, `✅` | Выполнено |
| `n`, `no`, `❌` | Не выполнено |
| `s`, `skip` | Пропустить |
| `r`, `retry` | Повторить текущий |
| `q`, `quit` | Выйти с сохранением |
| `h`, `help` | Помощь |

**Прогресс бар:**

## Чек-листы детально

### Pre-Flight Checklist (10 пунктов)

Проверка готовности проекта к работе.

**Пункты:**
1. Git репозиторий инициализирован
2. Ветка develop существует
3. .gitignore существует
4. README.md существует
5. .version существует
6. package.json существует
7. .qwen/ директория существует
8. .qwen/agents/ директория существует
9. .qwen/mcp.json существует
10. constitution.md существует

### Pre-Commit Checklist (7 пунктов)

Проверка перед коммитом изменений.

**Пункты:**
1. Изменения протестированы
2. Код проходит линтинг
3. Форматирование применено
4. Сообщение коммита следует Conventional Commits
5. Изменения атомарные
6. Нет отладочных изменений
7. Pre-commit хуки прошли

### Pre-Merge Checklist (8 пунктов)

Проверка перед слиянием ветки.

**Пункты:**
1. Все тесты проходят
2. Code review выполнен
3. CI/CD прошел
4. Ветка обновлена
5. Конфликты разрешены
6. CHANGELOG обновлен
7. Документация обновлена
8. Метрики не деградировали

### TDD Checklist (6 пунктов)

Test-Driven Development цикл.

**Пункты:**
1. Тест написан до реализации (Red)
2. Тест падает как ожидается (Red)
3. Минимальная реализация (Green)
4. Тест проходит (Green)
5. Рефакторинг кода (Refactor)
6. Все тесты проходят

### Agent Assignment Checklist (6 пунктов)

Назначение агентов на задачи.

**Пункты:**
1. Агент имеет специализацию
2. Агент доступен
3. Агент имеет навыки
4. Приоритет соответствует
5. nextAgent определен
6. Fallback агент определен

### Specification Checklist (9 пунктов)

Проверка спецификаций.

**Пункты:**
1. spec.md существует
2. Требования четко сформулированы
3. Требования тестируемы
4. Критерии приемки определены
5. Границы задачи определены
6. Зависимости документированы
7. Риски идентифицированы
8. Сроки реалистичны
9. Стейкхолдеры согласовали

### Phase 0 Checklist (7 пунктов)

Планирование задачи.

**Пункты:**
1. Задача проанализирована
2. Требования извлечены
3. Подзадачи идентифицированы
4. Исполнители назначены
5. План выполнения создан
6. Приоритеты установлены
7. План валидирован

### Initialization Checklist (9 пунктов)

Инициализация проекта.

**Пункты:**

### Release Checklist (12 пунктов)

Выпуск релиза.

**Пункты:**
1. Все тесты проходят

3. CHANGELOG обновлен
4. Версия обновлена
5. Тег создан

**Пункты:**

3. Тесты покрывают критический функционал

## Интеграция

### В оркестраторы

Добавьте проверку чек-листов в оркестраторы:

```markdown
### Post-Task Checklist Validation
- Проверка чек-листа после задачи
- Валидация результатов
- Рекомендации по улучшению
```

**Пример:**
```bash
# После завершения задачи
.qwen/checklists/validate-checklist.sh --name "Phase 0"
```

### В Feedback System

```bash
# Интеграция в generate-all.sh
.qwen/feedback/generate-all.sh

# Пропустить проверку чек-листов
.qwen/feedback/generate-all.sh --skip-checklists
```

### CI/CD Pipeline

```yaml
# GitHub Actions
name: Checklist Validation

on:
  pull_request:
    branches: [main, develop]

jobs:
  checklists:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Pre-Merge Checklist
        run: |
          .qwen/checklists/validate-checklist.sh --name "Pre-Merge"
      
      - name: Health Check
        run: |
          .qwen/checklists/validate-checklist.sh --name "Health Check"
```

## Оценка

### Процент выполнения

| Процент | Статус | Описание |
|---------|--------|----------|
| 90-100% | ✅ Passed | Отлично |
| 70-89% | ⚠️ Warning | Хорошо |
| 50-69% | ⚠️ Warning | Требует улучшения |
| <50% | ❌ Failed | Критические проблемы |

### Типы статусов пунктов

| Статус | Описание |
|--------|----------|
| ✅ Passed | Автоматически подтверждено |
| ❌ Failed | Не выполнено |
| ⚠️ Manual | Требует ручной проверки |
| ⏳ Pending | Ожидает проверки |
| ⏭️ Skipped | Пропущено |

## Troubleshooting

### Проблемы и решения

**Проблема:** Чек-лист не найден
```bash
# Показать список доступных
.qwen/checklists/checklist-runner.sh --list
```

**Проблема:** jq не найден
```bash
apt-get install jq  # или brew install jq
```

**Проблема:** Прерван сеанс
```bash
# Возобновить
.qwen/checklists/checklist-runner.sh --resume
```

**Проблема:** Ложные срабатывания
- Некоторые пункты требуют ручной проверки
- Используйте `--auto-fix` для автоматических исправлений
