# Справочник по инструментам Qwen Code CLI

## Краткие пояснения инструментов

- **Edit** - инструмент для замены текста в файле
- **ExitPlanMode** - инструмент для выхода из режима планирования и перехода к реализации
- **Glob** - инструмент для поиска файлов по glob-шаблону
- **Grep** - инструмент для поиска шаблона регулярного выражения в содержимом файлов
- **ListFiles** - инструмент для перечисления файлов и подкаталогов в указанном пути
- **ReadFile** - инструмент для чтения содержимого файла
- **ReadManyFiles** - инструмент для чтения содержимого нескольких файлов
- **SaveMemory** - инструмент для сохранения информации в долговременную память
- **Shell** - инструмент для выполнения команд оболочки
- **Skill** - инструмент для вызова специализированных навыков
- **Task** - инструмент для делегирования задач специализированным субагентам
- **TodoWrite** - инструмент для создания и управления списками задач
- **WebFetch** - инструмент для получения содержимого с указанного URL
- **WebSearch** - инструмент для выполнения веб-поиска
- **WriteFile** - инструмент для записи содержимого в файл

## Подробное описание инструментов

### 1. Edit (edit)

Инструмент для замены текста в файле. По умолчанию требует, чтобы old_string был уникально сопоставлен с одним местоположением; для изменения всех вхождений установите replace_all в true.

**Параметры:**
- `file_path` (строка, обязательный): Абсолютный путь к файлу для изменения
- `old_string` (строка, обязательный): Точный исходный текст для замены
- `new_string` (строка, обязательный): Новый текст для замены old_string
- `replace_all` (булево, опциональный): Заменить ли все вхождения old_string

**Поведение:**
- Если old_string пуст и файл не существует, создает новый файл
- Находит уникальное совпадение (если replace_all = false) или все совпадения (если replace_all = true)
- Включает многоступенчатый механизм коррекции для повышения надежности редактирования

**Подтверждение:** Да. Перед записью в файл показывается сравнение различий (diff) и запрашивается подтверждение пользователя.

### 2. ExitPlanMode (exit_plan_mode)

Инструмент используется, когда вы находитесь в режиме планирования и завершили представление плана реализации. Он запрашивает у пользователя одобрение или отклонение плана и переходит из режима планирования в режим реализации.

**Параметры:**
- `plan` (строка, обязательный): план реализации, который вы хотите представить пользователю для одобрения. Должен быть кратким, форматированным в markdown, планом, описывающим шаги реализации.

**Поведение:**
- Представляет план пользователю для одобрения
- Запрашивает подтверждение для продолжения реализации
- Переходит из режима планирования в режим реализации на основе ответа пользователя

**Подтверждение:** Да. Запрашивает у пользователя одобрение плана.

### 3. Glob (glob)

Поиск файлов, соответствующих определенному glob-шаблону (например: src/**/*.ts, *.md), с возвратом абсолютных путей, отсортированных по времени модификации (новые первыми).

**Параметры:**
- `pattern` (строка, обязательный): Glob-шаблон для сопоставления (например, "*.py", "src/**/*.js")
- `path` (строка, опциональный): Каталог для поиска. Если не указан, используется текущий рабочий каталог

**Поведение:**
- Ищет файлы, соответствующие glob-шаблону в указанном каталоге
- Возвращает список абсолютных путей, отсортированных по последнему изменению
- По умолчанию следует правилам .gitignore и .qwenignore
- Результат ограничен максимум 100 файлами

**Подтверждение:** Нет

### 4. Grep (grep_search)

Поиск шаблона регулярного выражения в содержимом файлов в указанном каталоге. Может фильтровать файлы по glob-шаблону. Возвращает строки с совпадениями, а также пути к файлам и номера строк.

**Параметры:**
- `pattern` (строка, обязательный): Шаблон регулярного выражения для поиска в содержимом файлов
- `path` (строка, опциональный): Файл или каталог для поиска. По умолчанию текущий рабочий каталог
- `glob` (строка, опциональный): Glob-шаблон для фильтрации файлов
- `limit` (число, опциональный): Ограничение вывода первыми N совпадающими строками

**Поведение:**
- Использует ripgrep для быстрого поиска при возможности
- Возвращает строки с совпадениями с указанием пути к файлу и номера строки
- По умолчанию регистронезависимый поиск
- Следует шаблонам .gitignore и .qwenignore

**Подтверждение:** Нет

### 5. ListFiles (list_directory)

Перечисляет непосредственно содержащиеся файлы и подкаталоги в указанном пути к каталогу. При желании можно игнорировать элементы, соответствующие предоставленному glob-шаблону.

**Параметры:**
- `path` (строка, обязательный): Абсолютный путь к каталогу, содержимое которого нужно перечислить
- `ignore` (массив строк, опциональный): Список glob-шаблонов для исключения из списка
- `respect_git_ignore` (булево, опциональный): Следовать ли шаблонам .gitignore при перечислении файлов

**Поведение:**
- Возвращает список имен файлов и каталогов
- Указывает, является ли каждый элемент каталогом
- Сортирует элементы: каталоги первыми, затем по алфавиту

**Подтверждение:** Нет

### 6. ReadFile (read_file)

Инструмент используется для чтения и возврата содержимого указанного файла. Он может обрабатывать текстовые файлы, изображения (PNG, JPG, GIF, WEBP, SVG, BMP) и PDF-файлы. Для текстовых файлов можно указать диапазон строк для чтения.

**Параметры:**
- `path` (строка, обязательный): Абсолютный путь к файлу, который нужно прочитать
- `offset` (число, опционально): Для текстовых файлов, номер строки, с которой начинать чтение (нумерация с 0). Требует одновременного указания limit
- `limit` (число, опционально): Для текстовых файлов, максимальное количество строк для чтения. Если не указано, читается либо значение по умолчанию (например, 2000 строк), либо весь файл целиком, если это возможно

**Поведение:**
- Для текстовых файлов: Возвращает содержимое файла. При использовании offset и limit возвращаются только строки в указанном диапазоне
- Для изображений и PDF-файлов: Возвращает содержимое файла в виде структуры данных в формате base64, подходящей для использования моделью
- Для других бинарных файлов: Пытается распознать и пропустить их, возвращая сообщение о том, что это общий бинарный файл

**Подтверждение:** Нет

### 7. ReadManyFiles (read_many_files)

Инструмент для чтения содержимого из нескольких файлов или каталогов, часто используется командой @. Позволяет читать и объединять содержимое нескольких файлов в одном вызове.

**Параметры:**
- `paths` (массив строк, обязательный): Массив glob-шаблонов или путей относительно целевого каталога
- `useDefaultExcludes` (булево, опциональный): Применять ли список шаблонов по умолчанию (например, node_modules, .git, бинарные файлы)
- `exclude` (массив строк, опциональный): Дополнительные glob-шаблоны для исключения
- `include` (массив строк, опциональный): Дополнительные glob-шаблоны для включения (переопределяет exclude)
- `recursive` (булево, опциональный): Рекурсивный ли поиск (по умолчанию true)
- `file_filtering_options` (объект, опциональный): Опции фильтрации файлов (respect_git_ignore, respect_qwen_ignore)

**Поведение:**
- Читает содержимое файлов, соответствующих шаблонам
- Объединяет содержимое в одну строку с разделителями
- Для текстовых файлов: вставляет содержимое с префиксами
- Для изображений/PDF: включает в base64 формате
- Вставляет маркеры начала и конца контента

**Подтверждение:** Нет

### 8. SaveMemory (save_memory)

Инструмент позволяет сохранять и вспоминать информацию между сессиями Qwen Code. Он добавляет указанные факты в файл контекста пользователя (~/.qwen/QWEN.md по умолчанию), которые затем загружаются в последующих сессиях.

**Параметры:**
- `fact` (строка, обязательный): конкретный факт или информация для запоминания
- `scope` (строка, опциональный): область сохранения ("global" для пользовательского уровня ~/.qwen/QWEN.md, "project" для проектного уровня)

**Поведение:**
- Сохраняет указанный факт в файл памяти
- Файл памяти представляет собой обычный текстовый файл Markdown
- Информация доступна в будущих сессиях

**Подтверждение:** Нет

### 9. Shell (run_shell_command)

Инструмент позволяет взаимодействовать с底层 системой, запускать скрипты или выполнять операции командной строки.

**Параметры:**
- `command` (строка, обязательный): точная команда оболочки для выполнения
- `description` (строка, опциональный): краткое описание назначения команды, которое будет отображаться пользователю
- `directory` (строка, опциональный): каталог для выполнения команды (относительно корня проекта)
- `is_background` (логический тип, обязательный): запускать ли команду в фоновом режиме

**Поведение:**
- На Windows команда выполняется через cmd.exe /c, на других платформах - через bash -c
- Возвращает подробную информацию о выполнении, включая команду, каталог, стандартные потоки вывода и ошибок, код выхода и список PID фоновых процессов
- Для интерактивных команд требуется установка параметра tools.shell.enableInteractiveShell в true

**Подтверждение:** Да. Запрашивает подтверждение перед выполнением команды.

### 10. Skill (skill)

Инструмент для вызова специализированных навыков (skills), которые представляют собой модульные возможности, расширяющие функциональность Qwen Code.

**Параметры:**
- `skill` (строка, обязательный): имя навыка для вызова

**Поведение:**
- Навыки вызываются автоматически моделью на основе запроса пользователя и описания навыка
- Каждый навык состоит из директории с файлом SKILL.md, содержащим YAML-метаданные и инструкции
- Навыки могут содержать инструкции, примеры, вспомогательные скрипты и шаблоны

**Подтверждение:** Зависит от конкретного навыка

### 11. Task (task)

Инструмент используется для запуска специализированных субагентов, которые могут автономно выполнять сложные, многоступенчатые задачи.

**Параметры:**
- `description` (строка, обязательный): Краткое (3-5 слов) описание задачи для отслеживания пользователем
- `prompt` (строка, обязательный): Подробный запрос задачи для субагента
- `subagent_type` (строка, обязательный): Тип специализированного агента, который будет использоваться для выполнения задачи

**Поведение:**
- Делегирует работу специализированным агентам, у которых есть доступ к собственному набору инструментов
- Обеспечивает параллельное выполнение задач и специализированную экспертизу
- Предоставляет живые обновления статуса выполнения субагента

**Подтверждение:** Зависит от конкретного субагента

### 12. TodoWrite (todo_write)

Инструмент для создания и управления структурированными списками задач во время сеансов программирования.

**Параметры:**
- `todos` (массив объектов, обязательный): Обновленный список задач

Каждая задача в массиве имеет следующие свойства:
- `id` (строка, обязательный): Уникальный идентификатор задачи
- `content` (строка, обязательный): Содержание задачи
- `status` (строка, обязательный): Статус задачи ("pending", "in_progress", "completed")

**Поведение:**
- Создает и управляет структурированными списками задач
- Позволяет отслеживать прогресс выполнения задач
- Обновляет статус задач в реальном времени

**Подтверждение:** Нет

### 13. WebFetch (web_fetch)

Инструмент для получения контента с указанного URL и его последующей обработки с помощью модели ИИ.

**Параметры:**
- `url` (строка, обязательный): URL-адрес, с которого нужно получить контент
- `prompt` (строка, обязательный): Запрос, описывающий, какую информацию нужно извлечь из содержимого страницы

**Поведение:**
- Преобразует HTML в текст
- Обрабатывает содержимое с помощью ИИ
- Обновляет HTTP до HTTPS (для безопасности)
- Преобразует GitHub blob-ссылки в raw-формат для лучшего доступа к содержимому

**Подтверждение:** Да. Запрашивает подтверждение перед загрузкой данных.

### 14. WebSearch (web_search)

Инструмент для выполнения веб-поиска и получения информации из интернета.

**Параметры:**
- `query` (строка, обязательный): поисковый запрос
- `provider` (строка, опционально): конкретный провайдер ("dashscope", "tavily", "google")

**Поведение:**
- Поддерживает несколько поисковых провайдеров (DashScope, Tavily, Google)
- Возвращает краткий ответ с указанием источников
- Использует настроенные API-ключи для доступа к провайдерам

**Подтверждение:** Нет

### 15. WriteFile (write_file)

Инструмент для записи содержимого в файл. Если файл существует, он будет перезаписан. Если файла нет, он будет создан вместе с необходимыми родительскими каталогами.

**Параметры:**
- `file_path` (строка, обязательный): Абсолютный путь к файлу для записи
- `content` (строка, обязательный): Содержимое для записи в файл

**Поведение:**
- Записывает предоставленное содержимое в указанный файл
- Создает родительские каталоги при необходимости

**Подтверждение:** Да. Перед записью показывается разница изменений и запрашивается подтверждение пользователя.
