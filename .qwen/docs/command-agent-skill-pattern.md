# Command → Agent → Skill Pattern

## Архитектура паттерна

Паттерн **Command → Agent → Skill** представляет собой трехуровневую архитектуру для организации рабочих процессов в Qwen Code. Этот паттерн обеспечивает четкое разделение ответственности, модульность и переиспользование компонентов.

```
┌─────────────────────────────────────────────────────────────────┐
│                    Command (Entry Point)                        │
│  • Точка входа для пользователя                                 │
│  • Определение рабочего процесса                                │
│  • Маршрутизация к агенту                                       │
│  • YAML Frontmatter конфигурация                                │
└─────────────────────────────────────────────────────────────────┘
                              ↓
                    (Agent Tool Call)
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                    Agent (Data Fetching)                        │
│  • Получение данных с预загруженным навыком                      │
│  • Контекстуальный анализ                                       │
│  • Вызов навыка для обработки                                   │
│  • Markdown инструкция с tools секцией                          │
└─────────────────────────────────────────────────────────────────┘
                              ↓
                     (Skill Tool Call)
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                    Skill (Output Creation)                      │
│  • Независимое создание вывода                                  │
│  • Специализированная логика                                    │
│  • SKILL.md с метаданными                                       │
│  • Возврат структурированного результата                        │
└─────────────────────────────────────────────────────────────────┘
```

## Сепарация ответственности

### Command (Команда)

**Назначение:** Точка входа для пользовательского взаимодействия

**Ответственность:**
- Определение имени команды и триггера (например, `/health-deps`)
- Описание рабочего процесса в YAML frontmatter
- Маршрутизация к соответствующему агенту через `handoffs`
- Обработка пользовательского ввода (`$ARGUMENTS`)
- Документирование процесса выполнения

**Расположение:** `.qwen/commands/*.md`

**Структура файла:**
```markdown
---
description: Краткое описание команды
handoffs:
  - label: Описание задачи для агента
    agent: имя-агента
    prompt: Детальный промпт для агента
    send: true
---

## Ввод пользователя

```text
$ARGUMENTS
```

## Описание команды
...
```

### Agent (Агент)

**Назначение:** Получение данных и контекстуальный анализ

**Ответственность:**
- Предзагрузка соответствующего навыка
- Сбор контекста проекта (файлы, директории, зависимости)
- Анализ данных в предметной области
- Вызов навыка для специализированной обработки
- Формирование промежуточных результатов

**Расположение:** `.qwen/agents/*.md`

**Структура файла:**
```markdown
---
name: имя-агента
description: Описание назначения агента
tools:
 - read_file
 - read_many_files
 - skill: имя-навыка
---

# Агент: Название

## Назначение
...

## Процесс работы
...
```

### Skill (Навык)

**Назначение:** Независимое создание вывода со специализированной логикой

**Ответственность:**
- Инкапсуляция специализированной логики
- Обработка данных и генерация вывода
- Возврат структурированного результата
- Переиспользование между различными агентами

**Расположение:** `.qwen/skills/имя-навыка/SKILL.md`

**Структура файла:**
```markdown
---
name: имя-навыка
description: Описание навыка
---

# Название навыка

## Когда использовать
...

## Инструкции
...

## Формат ввода
...

## Формат вывода
...
```

## Примеры реализации

### Пример 1: Аудит зависимостей

**Command:** `.qwen/commands/health-deps.md`
```markdown
---
description: Запускает аудит зависимостей проекта
handoffs:
  - label: Выполнить анализ зависимостей
    agent: dependency-analyzer
    prompt: Проанализировать зависимости проекта на уязвимости и устаревшие версии
    send: true
---
```

**Agent:** `.qwen/agents/work_dev_dependency_analyzer.md`
```markdown
---
name: dependency-analyzer
description: Анализ зависимостей проекта
tools:
 - read_file
 - read_many_files
 - skill: dependency-auditor
---
```

**Skill:** `.qwen/skills/dependency-auditor/SKILL.md`
```markdown
---
name: dependency-auditor
description: Аудит зависимостей для проверки проекта на наличие проблем
---
```

## Best Practices

### 1. Принцип единственной ответственности

- **Command:** Только маршрутизация и описание рабочего процесса
- **Agent:** Только сбор контекста и анализ
- **Skill:** Только специализированная обработка и вывод

### 2. Именование

- **Commands:** Глагол-существительное в kebab-case (`health-deps`, `run-quality-gate`)
- **Agents:** Префикс по типу (`work_`, `orc_`) + описание
- **Skills:** Существительное в kebab-case (`bug-hunter`, `code-cleaner`)

### 3. Документирование

- Каждая команда должна иметь полное описание процесса
- Каждый агент должен документировать свои инструменты
- Каждый навык должен указывать форматы ввода/вывода

### 4. Переиспользование навыков

- Навыки должны быть максимально универсальными
- Избегайте дублирования логики в навыках
- Один навык может использоваться несколькими агентами

### 5. Обработка ошибок

- Команды должны документировать возможные ошибки
- Агенты должны обрабатывать ошибки сбора контекста
- Навыки должны возвращать структурированные сообщения об ошибках

## Рефакторинг существующих команд

### Шаг 1: Анализ текущей команды

1. Определите текущую логику команды
2. Выделите специализированную логику в навык
3. Определите контекст, необходимый агенту

### Шаг 2: Создание навыка

```bash
mkdir -p .qwen/skills/имя-навыка
```

Создайте `.qwen/skills/имя-навыка/SKILL.md`:
```markdown
---
name: имя-навыка
description: Описание навыка
---

# Название

## Когда использовать
...
```

### Шаг 3: Обновление агента

Добавьте skill в tools секцию агента:
```markdown
tools:
 - read_file
 - skill: имя-навыка
```

### Шаг 4: Обновление команды

Обновите handoffs для указания правильного агента:
```markdown
handoffs:
  - label: Описание задачи
    agent: имя-агента
    prompt: Детальный промпт
    send: true
```

### Шаг 5: Тестирование

1. Протестируйте команду с различными вводами
2. Убедитесь, что навык вызывается корректно
3. Проверьте формат вывода

## Миграция с монолитных команд

### До (монолитная команда):
```markdown
---
description: Команда делает всё сама
---

## Процесс
1. Читает файлы
2. Анализирует код
3. Генерирует отчет
4. Исправляет проблемы
```

### После (Command → Agent → Skill):
```markdown
---
description: Команда маршрутизирует к агенту
handoffs:
  - label: Анализ
    agent: analyzer
    prompt: Проанализировать код
    send: true
---
```

Агент использует навык для специализированной обработки.

## Преимущества паттерна

1. **Модульность:** Каждый компонент независим и переиспользуем
2. **Тестируемость:** Легко тестировать каждый уровень отдельно
3. **Масштабируемость:** Новые команды используют существующие навыки
4. **Поддерживаемость:** Изменения локализованы в одном компоненте
5. **Документированность:** Четкая структура упрощает понимание

## Антипаттерны

### ❌ Неправильно: Агент выполняет логику навыка
```markdown
# Агент делает всё сам
- Читает файлы
- Анализирует код
- Генерирует отчет
- Исправляет проблемы
```

### ✅ Правильно: Агент использует навык
```markdown
# Агент координирует
tools:
 - read_file
 - skill: code-analyzer
```

### ❌ Неправильно: Команда содержит бизнес-логику
```markdown
## Процесс
1. Для каждого файла .js:
   - Прочитать содержимое
   - Найти console.log
   - Записать в отчет
```

### ✅ Правильно: Команда маршрутизирует
```markdown
handoffs:
  - label: Найти проблемы кода
    agent: code-analyzer
    prompt: Анализировать код на проблемы
    send: true
```

## Заключение

Паттерн Command → Agent → Skill обеспечивает чистую архитектуру для рабочих процессов Qwen Code. Следование этому паттерну упрощает разработку, тестирование и поддержку команд, агентов и навыков.
