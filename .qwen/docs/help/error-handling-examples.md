**Версия:** 1.0.0 (v0.5.0)  
**Дата:** 20 марта 2026  
**Назначение:** Руководство по обработке типичных ошибок

---

## 1. Git не инициализирован

**Ошибка:**
```
❌ Git репозиторий не инициализирован
```

**Причина:**
- Разработка началась без инициализации Git

**Решение:**
```bash
# Инициализировать Git
git init

# Создать ветку develop
git branch develop

# Создать .gitignore
cp .qwen/templates/gitignore .gitignore

# Или автоматически
.qwen/scripts/orchestration-tools/initialize-project.sh
```

**Профилактика:**
- Запускать `initialize-project.sh` перед началом разработки
- Pre-Flight проверки обнаруживают проблему

## 2. Quality Gate не прошел

**Ошибка:**
```
❌ Gate 1: Pre-Execution Gate не пройден
❌ 3 проверки не пройдены
```

**Причина:**
- Отсутствуют необходимые файлы
- Спецификация не заполнена
- Фаза 0 не завершена

**Решение:**
```bash
# Запустить проверку с деталями
.qwen/scripts/quality-gates/check-planning.sh .qwen/specify/specs/{ID}

# Исправить ошибки (см. вывод скрипта)
# ...

# Повторить проверку
.qwen/scripts/quality-gates/check-planning.sh .qwen/specify/specs/{ID}
```

**Профилактика:**
- Следовать чеклистам
- Запускать Pre-Flight проверки

## 3. Агент отсутствует

**Ошибка:**
```
❌ Агент work_rust_developer не найден
❌ Невозможно выполнить задачу без специализированного агента
```

**Причина:**
- Агент не создан вовремя
- План Фазы 0 не определил требуемого агента

**Решение:**
```bash
# Создать агента через work_dev_meta_agent
task '{
  "subagent_type": "work_dev_meta_agent",
  "description": "Создать агента work_rust_developer",
  "prompt": "Создать специализированного агента для Rust разработки"
}'

# Или вручную
cp .qwen/templates/worker-template.md .qwen/agents/work_rust_developer.md
# Отредактировать под требования
```

**Профилактика:**
- Запускать Фазу 0 перед разработкой
- Проверять наличие агентов в phase0-agents.json

## 4. Коммит в main напрямую

**Ошибка:**
```
❌ Нарушение Git workflow
❌ Коммит в ветку main запрещен
```

**Причина:**
- Разработка велась напрямую в main
- Ветка develop не создана

**Решение:**
```bash
# Создать ветку develop
git branch develop

# Переключиться на develop
git checkout develop

# Если коммит уже сделан в main
git reset --hard HEAD~1  # Отменить коммит
git checkout develop
git cherry-pick <commit-hash>  # Перенести коммит
```

**Профилактика:**
- Pre-Flight проверки обнаруживают проблему
- Следовать Git workflow

## 5. Spec не соответствует требованиям

**Ошибка:**
```
❌ Gate 5: Pre-Implementation Gate не пройден
❌ 2 раздела spec.md отсутствуют
```

**Причина:**
- Спецификация не заполнена полностью
- Требования не тестируемые
- Критерии успеха не измеримы

**Решение:**
```bash
# Запустить проверку с деталями
.qwen/scripts/quality-gates/check-specifications.sh .qwen/specify/specs/{ID}

# Исправить spec.md
# Добавить отсутствующие разделы:
# - Краткое описание
# - Контекст
# - Акторы
# - Требования
# - Сценарии использования
# - Условия успеха
# - Ограничения
# - Предположения
# - Риски

# Повторить проверку
.qwen/scripts/quality-gates/check-specifications.sh .qwen/specify/specs/{ID}
```

**Профилактика:**
- Использовать speckit.specify для создания
- Использовать speckit.clarify для уточнения
- Следовать шаблону спецификации

## 6. Тесты не проходят

**Ошибка:**
```
❌ cargo test failed
❌ 3 tests failed
```

**Причина:**
- Код не соответствует требованиям
- Тесты устарели
- Зависимости не установлены

**Решение:**
```bash
# Запустить тесты с деталями
cargo test --workspace -- --nocapture

# Исправить код
# ...

# Или обновить тесты (если устарели)
# ...

# Повторить тесты
cargo test --workspace
```

**Профилактика:**
- TDD: тесты ДО кода
- Запускать тесты после каждого изменения

## 7. Pre-commit хук не проходит

**Ошибка:**
```
❌ Pre-commit hook failed
❌ cargo clippy: 5 warnings
```

**Причина:**
- Код не отформатирован
- Линтеры не пройдены
- Тесты не проходят

**Решение:**
```bash
# Отформатировать код
cargo fmt --all

# Исправить предупреждения
cargo clippy --workspace -- -D warnings

# Запустить тесты
cargo test --workspace

# Повторить коммит
git commit
```

**Профилактика:**
- Запускать линтеры до коммита
- Использовать IDE с автоформатированием

## 9. Агент создан постфактум

**Ошибка:**
```
⚠️ Агент work_rust_developer создан ПОСЛЕ начала разработки
⚠️ Код написан без специализированного агента
```

**Причина:**
- Фаза 0 не определила требуемого агента

```bash

# Если агент отсутствует:
# 1. Обновить Фазу 0
.qwen/specify/scripts/phase0-analyzer.sh .qwen/specify/specs/{ID}

# 2. Создать агента
task '{
  "subagent_type": "work_dev_meta_agent",
  "description": "Создать недостающего агента",
  "prompt": "..."
}'

# 3. Перегенерировать задачи
speckit.tasks
```

**Профилактика:**
- Запускать Фазу 0 перед разработкой
- Проверять phase0-agents.json

## 10. State/ файлы в main

**Ошибка:**
```
❌ state/ обнаружена в main
❌ Файлы разработки не должны быть в main
```

**Причина:**
- .gitignore не настроен
- Файлы разработки закоммичены

**Решение:**
```bash
# Добавить state/ в .gitignore
echo "state/" >> .gitignore

# Удалить из Git (но оставить на диске)
git rm -r --cached state/

# Закоммитить изменения
git add .gitignore
git commit -m "chore: добавить state/ в .gitignore"
```

**Профилактика:**
- Использовать initialize-project.sh
- Pre-Flight проверки обнаруживают проблему

## 11. Нет журнала вызовов агентов

**Ошибка:**
```
⚠️ Журнал вызовов агентов отсутствует
⚠️ Невозможно аудировать выполнение задач
```

**Причина:**
- log-agent-call.sh не используется
- Агенты вызываются напрямую

**Решение:**
```bash
# Создать директорию логов
mkdir -p .qwen/logs

# Логировать вызовы агентов
.qwen/scripts/agent-tools/log-agent-call.sh orc_dev_task_coordinator P1-T01 started

# Проверить журнал
cat .qwen/logs/agent-calls.log
```

**Профилактика:**
- Интегрировать log-agent-call.sh в оркестраторы
- Требовать логирование в документации

## 12. Сборка не проходит

**Ошибка:**
```
❌ cargo build failed
❌ Compilation error
```

**Причина:**
- Синтаксические ошибки
- Зависимости не установлены
- Конфликты версий

**Решение:**
```bash
# Установить зависимости
cargo fetch

# Проверить синтаксис
cargo check --workspace

# Исправить ошибки компиляции
# ...

# Повторить сборку
cargo build --workspace
```

**Профилактика:**
- Запускать cargo check перед коммитом
- Использовать IDE с проверкой типов
