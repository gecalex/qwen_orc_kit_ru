# Руководство по разработке агентов

## Типы агентов

В системе Qwen Orchestrator Kit существуют два основных типа агентов:

### Оркестраторы
- Имя начинается с префикса `orc_`
- Координируют выполнение сложных многопроходных рабочих процессов
- Управляют другими агентами (воркерами)
- Используют шаблон возврата управления
- Следуют стандартизированному формату отчетности

### Воркеры
- Имя начинается с префикса `work_`
- Выполняют конкретные задачи из файлов плана
- Следуют шаблону возврата управления
- Используют стандартизированный формат отчетности

## Создание оркестратора

### Структура оркестратора

1. **YAML заголовок** - с обязательными полями
2. **Назначение** - описание роли и ответственности
3. **Использование сервера MCP** - рекомендации по использованию внешних сервисов
4. **Инструкции** - пошаговое описание логики работы
5. **Рекомендации MCP в файлах плана** - пример структуры плана
6. **Шаблон возврата управления** - описание процесса завершения работы
7. **Стандартизированная отчетность** - формат отчетов
8. **Интеграция навыков** - список используемых навыков

### Пример структуры

```markdown
# Оркестратор {название}

## Назначение
[Описание роли и ответственности]

## Использование сервера MCP
[Рекомендации по использованию внешних сервисов]

## Инструкции
[Пошаговое описание логики работы]

## Рекомендации MCP в файлах плана
[Пример структуры плана]

## Шаблон возврата управления
[Описание процесса завершения работы]

## Стандартизированная отчетность
[Формат отчетов]

## Интеграция навыков
[Список используемых навыков]
```

## Создание воркера

### Структура воркера

1. **YAML заголовок** - с обязательными полями
2. **Назначение** - описание роли и ответственности
3. **Использование сервера MCP** - рекомендации по использованию внешних сервисов
4. **Инструкции** - пошаговое описание логики работы (обычно 5 фаз)
5. **Формат файла плана** - пример структуры плана
6. **Шаблон возврата управления** - описание процесса завершения работы
7. **Стандартизированная отчетность** - формат отчетов
8. **Интеграция навыков** - список используемых навыков

## Лучшие практики

### Именование
- Используйте kebab-case для имен агентов
- Включайте домен в имя (например, `orc_dev_task_coordinator`)
- Убедитесь, что имя отражает функциональность агента

### Описание
- Пишите ясные и информативные описания
- Указывайте основное назначение агента
- Для оркестраторов укажите шаблон возврата управления и стандартизированную отчетность

### Инструменты
- Включайте только необходимые инструменты
- Следуйте порядку перечисления инструментов
- Не используйте квадратные скобки для перечисления

### MCP интеграция
- Указывайте, когда использовать MCP серверы
- Приводите примеры конкретных инструментов
- Объясняйте причину использования

## Проверка качества

Перед использованием агента убедитесь, что:
- Все обязательные поля присутствуют
- Структура соответствует шаблону
- Описания ясны и понятны
- Примеры актуальны и работоспособны
- Агент проходит все автоматические проверки

## Отладка

Для отладки агентов:
- Используйте логирование для отслеживания выполнения
- Проверяйте файлы плана на корректность
- Тестируйте каждый этап отдельно
- Используйте навыки для анализа ошибок
