# Руководство по использованию мета-агента

## Обзор

Мета-агент (`work_dev_meta_agent`) - это специализированный агент для автоматического создания новых агентов (оркестраторов и воркеров) на основе заданных параметров. Он генерирует агентов с необходимыми компонентами и документацией в соответствии с установленными шаблонами и стандартами.

## Назначение

Мета-агент предназначен для:
- Упрощения процесса создания новых агентов
- Обеспечения согласованности структуры и содержания агентов
- Автоматизации рутинных задач при разработке новых компонентов системы
- Соблюдения стандартов именования и структуры агентов

## Использование

### Через команду task

Для создания нового агента используйте команду `task` с указанием типа мета-агента:

```bash
task '{
  "subagent_type": "work_dev_meta_agent",
  "description": "Создание нового агента",
  "prompt": "Создай новый воркер агент для обработки задач в домене dev с именем work_dev_example_worker и описанием 'Агент для обработки примерных задач'"
}'
```

### Параметры создания агента

При создании агента необходимо указать следующие параметры:

- **Тип агента**: `orc` (оркестратор) или `work` (воркер)
- **Домен**: `dev`, `frontend`, `backend`, `testing`, `research`, `security`
- **Имя**: в формате kebab-case (например: `example-worker`, `my-task-handler`)
- **Описание**: краткое описание функций агента

### Примеры

#### Создание воркера для домена dev

```bash
task '{
  "subagent_type": "work_dev_meta_agent",
  "description": "Создание воркера для обработки задач в домене dev",
  "prompt": "Создай новый воркер агент для обработки задач в домене dev с именем work_dev_code_analyzer и описанием 'Агент для анализа кода и выявления потенциальных проблем'"
}'
```

#### Создание оркестратора для домена testing

```bash
task '{
  "subagent_type": "work_dev_meta_agent",
  "description": "Создание оркестратора для тестирования",
  "prompt": "Создай новый оркестратор агент для координации задач тестирования в домене testing с именем orc_testing_quality_assurer и описанием 'Оркестратор для координации задач обеспечения качества'"
}'
```

## Структура создаваемого агента

Мета-агент создает агентов со следующей структурой:

1. **YAML заголовок** с полями:
   - `name`: полное имя агента в формате `{типы}_{домен}_{имя}`
   - `description`: описание функций агента
   - `color`: цвет в соответствии с доменом
   - `tools`: стандартный набор инструментов

2. **Раздел "Назначение"**: описание функций агента

3. **Раздел "Использование сервера MCP"**: рекомендации по использованию MCP-серверов

4. **Раздел "Инструкции"**: пошаговое описание логики работы агента

5. **Раздел "Формат файла плана"**: пример структуры плана для агента

6. **Раздел "Шаблон возврата управления"**: описание процесса завершения работы

7. **Раздел "Стандартизированная отчетность"**: формат отчетов агента

8. **Раздел "Интеграция навыков"**: список используемых навыков

## Цветовая схема

Агенты используют следующую цветовую схему в зависимости от домена:

- `dev`: blue
- `frontend`: purple
- `backend`: orange
- `testing`: green
- `research`: cyan
- `security`: red

## Стандартный набор инструментов

Все создаваемые агенты включают следующий стандартный набор инструментов:

```yaml
tools:
 - read_file
 - write_file
 - edit
 - run_shell_command
 - grep_search
 - glob
 - web_search
 - web_fetch
 - read_many_files
 - todo_write
 - skill
 - task
```

## Проверки качества

После создания агента автоматически запускаются проверки качества:

- Проверка корректности YAML заголовка
- Проверка структуры агента
- Проверка соответствия стандартам именования

## Обновление индекса

Если существует файл `.qwen/docs/agents-index.md`, мета-агент автоматически добавляет созданный агент в индекс.

## Ограничения и рекомендации

1. Имя агента должно быть уникальным в системе
2. Имя должно следовать формату kebab-case
3. Домен должен быть одним из разрешенных
4. Описание должно быть информативным и отражать функции агента
5. После создания агента рекомендуется дополнить его специфичной логикой в разделе "Инструкции"

## Устранение неполадок

### Агент с таким именем уже существует

Если при создании агента появляется сообщение о том, что агент с таким именем уже существует, вы можете:
- Выбрать другое имя для агента
- Подтвердить перезапись существующего агента (будьте осторожны, это приведет к потере изменений)

### Ошибка валидации параметров

Убедитесь, что:
- Тип агента указан как `orc` или `work`
- Домен указан из разрешенного списка
- Имя соответствует формату kebab-case
- Описание содержит достаточное количество информации

## Следующие шаги

После создания агента рекомендуется:

1. Проверить сгенерированный файл на корректность
2. Дополнить раздел "Инструкции" специфичной логикой агента
3. Добавить необходимые проверки и валидации
4. Протестировать агента на простых задачах
5. Обновить документацию, если это необходимо
