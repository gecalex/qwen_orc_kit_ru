# Процессы обеспечения качества

## Цель обеспечения качества

Процессы обеспечения качества в Qwen Orchestrator Kit направлены на:
- Поддержание соответствия кода и документации стандартам
- Обеспечение стабильности и надежности агентов
- Автоматизацию проверок для снижения ручного труда
- Раннее выявление и устранение проблем

## Контрольные точки качества

### Pre-Execution Gate
- Проверка корректности постановки задачи
- Проверка наличия всех необходимых ресурсов
- Проверка совместимости с текущей архитектурой

### Post-Execution Gate
- Верификация результата под-агента
- Проверка корректности выполнения задачи
- Проверка соответствия ожидаемым результатам

### Pre-Commit Gate
- Проверки линтером (линтинг)
- Запуск тестов
- Проверка типизации
- Проверка соответствия YAML стандартам

### Pre-Merge Gate
- Интеграционные проверки
- Проверка покрытия тестами
- Проверка безопасности
- Проверка соответствия спецификациям

### Pre-Implementation Gate
- Проверка качества спецификаций
- Проверка однозначности требований
- Проверка тестируемости требований

## Автоматические проверки

### Проверка YAML заголовков
Используйте навык `yaml-header-validator` для проверки:
- Наличия всех обязательных полей
- Корректности формата перечисления инструментов
- Соответствия именования шаблону
- Правильности использования поля `task`

### Проверка структуры агента
Используйте навык `agent-structure-checker` для проверки:
- Наличия всех обязательных разделов
- Правильности порядка разделов
- Соответствия содержания шаблону

### Проверка стиля и формата
Используйте навык `format-validator` для проверки:
- Стиля написания документации
- Формата markdown
- Согласованности именования

## Ручные проверки

Некоторые аспекты требуют ручной проверки:
- Логическая согласованность агента
- Качество описаний
- Полнота покрытия функционала
- Качество примеров использования

## Инструменты проверки

### Навыки для проверок
- `run-quality-gate` - выполнение комплексных проверок
- `validate-plan-file` - проверка файлов плана
- `validate-report-file` - проверка отчетов
- `parse-error-logs` - анализ логов ошибок
- `format-markdown-table` - форматирование таблиц

### Скрипты проверки
- `scripts/quality-gates/check-security.sh` - проверка безопасности
- `scripts/quality-gates/check-coverage.sh` - проверка покрытия тестами
- `scripts/quality-gates/check-bundle-size.sh` - проверка размера бандла
- `scripts/quality-gates/check-typing.sh` - проверка типизации
- `scripts/quality-gates/check-linting.sh` - проверка линтером

## Процесс проверки

### При создании нового агента
1. Создание на основе шаблона
2. Проверка YAML заголовка
3. Проверка структуры агента
4. Проверка соответствия функционала описанию
5. Тестирование функционала
6. Формирование отчета о проверке

### При обновлении существующего агента
1. Анализ изменений
2. Проверка обратной совместимости
3. Тестирование обновленной функциональности
4. Проверка влияния на другие компоненты
5. Обновление документации при необходимости

## Ответственность за качество

- **Разработчик агента**: отвечает за первоначальное качество
- **Рецензент кода**: проверяет соответствие стандартам
- **Система CI/CD**: автоматически выполняет проверки
- **Агент-специалист**: обеспечивает актуальность стандартов

## Метрики качества

Отслеживайте следующие метрики:
- Количество ошибок в агентах
- Время на исправление ошибок
- Покрытие тестами
- Соответствие стандартам
- Частота рефакторинга

## Непрерывное улучшение

Процессы обеспечения качества должны постоянно улучшаться:
- Анализ частых ошибок и добавление соответствующих проверок
- Автоматизация рутинных проверок
- Обновление стандартов на основе обратной связи
- Обучение разработчиков лучшим практикам
