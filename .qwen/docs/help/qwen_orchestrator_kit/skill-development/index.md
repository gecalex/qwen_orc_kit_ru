# Руководство по разработке навыков

## Что такое навык

Навык (skill) в системе Qwen Orchestrator Kit - это переиспользуемая функция, которую агенты могут вызывать для выполнения часто используемых операций. Навыки обеспечивают согласованность и упрощают разработку агентов.

## Структура навыка

Навыки находятся в директории `.qwen/skills/{название-навыка}/SKILL.md` и имеют следующую структуру:

### YAML заголовок
```yaml
---
name: имя-навыка
description: Что делает. Используйте когда [конкретный сценарий].
---
```

### Разделы навыка

1. **Когда использовать** - сценарии применения
2. **Инструкции** - пошаговое описание логики
3. **Формат ввода** - структура входных данных
4. **Формат вывода** - структура выходных данных
5. **Примеры** - практические примеры использования

## Создание навыка

### Шаги для создания навыка

1. Создайте директорию в `.qwen/skills/{название-навыка}/`
2. Внутри создайте файл `SKILL.md` со следующей структурой:
   - YAML заголовок с именем и описанием
   - Раздел "Когда использовать" - описывает, когда использовать навык
   - Раздел "Инструкции" - инструкции по использованию
   - Форматы ввода и вывода
   - Примеры использования

### Пример структуры

```markdown
---
name: имя-навыка
description: Что делает. Используйте когда [конкретный сценарий].
---

# Имя навыка

## Когда использовать
- Сценарий 1
- Сценарий 2

## Инструкции
1. Шаг 1
2. Шаг 2

## Формат ввода
{Ожидаемая структура ввода}

## Формат вывода
{Ожидаемая структура вывода}

## Примеры
{Примеры использования}
```

## Типы навыков

### Навыки обработки данных
- `parse-error-logs` - парсинг логов ошибок
- `parse-git-status` - парсинг статуса git
- `format-markdown-table` - форматирование таблиц markdown

### Навыки валидации
- `run-quality-gate` - выполнение контрольных точек качества
- `validate-plan-file` - валидация файлов плана
- `validate-report-file` - валидация файлов отчетов

### Навыки форматирования
- `generate-report-header` - генерация заголовков отчетов
- `format-commit-message` - форматирование сообщений коммитов
- `render-template` - рендеринг шаблонов

### Навыки анализа
- `calculate-priority-score` - расчет приоритетных баллов
- `parse-package-json` - извлечение метаданных из package.json
- `extract-version` - извлечение версий из строк

## Лучшие практики

### Для разработки:
- Навыки должны быть компактными (<100 строк логики)
- Каждый навык должен решать одну задачу
- Навыки должны быть хорошо документированы
- Используйте ясные имена, отражающие функциональность
- Обеспечьте понятные примеры использования

### Для интеграции:
- Агенты могут вызывать навыки с помощью инструмента `Skill`
- Используйте навыки для повторно используемых операций
- Интегрируйте навыки в рабочие процессы агентов
- Обеспечьте обработку ошибок при вызове навыков

## Проверка качества

Перед использованием навыка убедитесь, что:
- Имя отражает функциональность
- Описание ясно и информативно
- Примеры актуальны и работоспособны
- Навык решает конкретную задачу
- Форматы ввода/вывода четко определены

## Использование навыков

Для вызова навыка из агента используйте инструмент `Skill`:

```json
{
  "skill": "parse-error-logs",
  "parameters": {
    "logType": "typescript",
    "logContent": "...",
    "maxErrors": 5
  }
}
```

## Автоматизация

Навыки могут быть интегрированы в автоматические процессы:
- Контрольные точки качества
- Проверки формата
- Анализ ошибок
- Генерация отчетов
