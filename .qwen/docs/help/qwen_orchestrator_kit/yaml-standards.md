# Стандарты для YAML заголовков

## Общие требования

Каждый агент и навык в системе Qwen Orchestrator Kit должен иметь корректный YAML заголовок, соответствующий следующим стандартам:

### Обязательные поля

1. **name** - уникальное имя компонента
2. **description** - краткое описание функций
3. **tools** - список инструментов, которые может использовать компонент
4. **color** - цветовая метка для визуализации

### Правила для поля `name`

- Используйте kebab-case (например, `my-awesome-agent`)
- Для оркестраторов используйте префикс `orc_`
- Для воркеров используйте префикс `work_`
- Включите домен в имя (например, `orc_dev_task_coordinator`)

### Правила для поля `description`

- Краткое, но информативное описание функций компонента
- Используйте активный залог
- Укажите основное назначение компонента
- Для оркестраторов добавьте информацию о шаблоне возврата управления и стандартизированной отчетности

### Правила для поля `tools`

Инструменты должны быть перечислены в следующем порядке:

```yaml
tools:
 - read_file
 - write_file
 - edit
 - run_shell_command
 - grep_search
 - glob
 - web_search
 - web_fetch
 - read_many_files
 - todo_write
 - skill
 - task
```

Примечания:
- Поле `task` должно присутствовать только у оркестраторов
- Все инструменты перечисляются по одному в строке с дефисом
- Не используйте квадратные скобки для перечисления инструментов

### Правила для поля `color`

- Используйте цвета в нижнем регистре
- Цвет должен соответствовать домену агента:
  - `blue` - для общих утилит
  - `green` - для тестирования
  - `red` - для безопасности
  - `orange` - для бэкенда
  - `purple` - для фронтенда
  - `cyan` - для планирования

## Пример корректного заголовка

### Для оркестратора:

```yaml
---
name: orc_dev_task_coordinator
description: Используйте для координации задач разработки в области разработки. Реализует итерационную логику и шаблон возврата управления. Следует стандартизированному формату отчетности.
tools:
 - read_file
 - write_file
 - edit
 - run_shell_command
 - grep_search
 - glob
 - web_search
 - web_fetch
 - read_many_files
 - todo_write
 - skill
 - task
color: blue
---
```

### Для воркера:

```yaml
---
name: work_dev_code_analyzer
description: Активно используйте для выполнения задач анализа кода из файлов плана в области разработки. Следует стандартизированному формату отчетности и реализует шаблон возврата управления.
tools:
 - read_file
 - write_file
 - edit
 - run_shell_command
 - grep_search
 - glob
 - web_search
 - web_fetch
 - read_many_files
 - todo_write
 - skill
color: green
---
```

## Проверка соответствия

Для автоматической проверки соответствия используйте навык `yaml-header-validator`, который проверяет:
- Наличие всех обязательных полей
- Корректность формата перечисления инструментов
- Соответствие именования шаблону
- Правильность использования поля `task`

## Частые ошибки

1. Использование квадратных скобок для перечисления инструментов
2. Пропуск обязательных полей
3. Неправильное именование (использование camelCase или PascalCase)
4. Отсутствие поля `task` у оркестраторов
5. Наличие поля `task` у воркеров
6. Неправильный регистр в названиях инструментов
