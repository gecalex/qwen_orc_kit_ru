# План поэтапной реализации недостающих компонентов

## Этап 1: Подготовительные работы (неделя 1)

1.1. Создание базовой структуры для новой системы агентов
- Введение системы именования: `orc_{domain}_имя_агента.md` для оркестраторов и `work_{domain}_имя_агента.md` для воркеров
- Создание примеров агентов с новой структурой именования

1.2. Анализ существующих агентов и их классификация
- Классификация текущих агентов по новой системе
- Определение доменов для каждого агента
- Создание миграционного плана для существующих агентов

## Этап 2: Реализация фазы планирования (неделя 2-3)

2.1. Создание фазы 0 планирования
- Реализация алгоритма анализа задач (P001)
- Создание механизма определения требуемых типов агентов
- Реализация логики назначения исполнителей

2.2. Интеграция фазы планирования в существующий процесс Speckit
- Модификация команд `/speckit.plan` и `/speckit.tasks` для учета фазы планирования
- Создание шаблона для задач планирования

## Этап 3: Реализация механизма создания агентов (неделя 4)

3.1. Создание meta-agent-v3 для Qwen Code CLI
- Адаптация meta-agent-v3 под плоскую структуру агентов
- Реализация создания агентов с правильной структурой и именованием
- Обеспечение параллельного создания нескольких агентов

3.2. Интеграция механизма создания агентов с фазой планирования
- Автоматическое создание отсутствующих агентов во время фазы планирования
- Обновление логики анализа задач для учета возможности создания агентов

## Этап 4: Стандартизация отчетов (неделя 5)

4.1. Создание стандартизированного формата отчетов
- Разработка шаблона отчета для агентов
- Создание навыка `generate-report-header` для Qwen Code CLI

4.2. Интеграция стандартизированных отчетов в существующие агенты
- Модификация существующих агентов для использования нового формата отчетов
- Обновление оркестраторов для обработки стандартизированных отчетов

## Этап 5: Расширение системы контрольных точек качества (неделя 6)

5.1. Реализация расширенной системы контрольных точек
- Создание навыка `run-quality-gate` для Qwen Code CLI
- Реализация блокирующих и неблокирующих проверок
- Интеграция контрольных точек в оркестрационные процессы

5.2. Создание пользовательских скриптов для проверок
- Реализация скриптов для проверки покрытия, безопасности, размера бандла и т.д.

## Этап 6: Расширение набора навыков (неделя 7)

6.1. Создание новых навыков для часто используемых операций
- Навыки для парсинга логов, валидации планов, форматирования данных
- Навыки для работы с git, package.json, ошибками и т.д.

6.2. Интеграция навыков в агентов
- Обновление агентов для использования новых навыков
- Создание документации по использованию навыков

## Этап 7: Интеграция с MCP (неделя 8)

7.1. Расширение интеграции с MCP серверами
- Реализация рекомендаций по использованию MCP в зависимости от типа задачи
- Интеграция MCP в плановые файлы

7.2. Обновление агентов для учета MCP
- Модификация агентов для использования рекомендованных MCP серверов
- Создание логики выбора подходящих MCP серверов

## Этап 8: Расширение возможностей оркестрации (неделя 9)

8.1. Реализация сложных сценариев оркестрации
- Внедрение паттерна "возвращения управления"
- Создание оркестраторов с итеративной логикой

8.2. Интеграция новых возможностей оркестрации в существующие процессы
- Обновление существующих оркестраторов
- Создание новых сложных оркестраторов для специфических задач

## Этап 9: Тестирование и валидация (неделя 10)

9.1. Комплексное тестирование новых компонентов
- Тестирование фазы планирования
- Тестирование механизма создания агентов
- Тестирование стандартизированных отчетов
- Тестирование контрольных точек качества

9.2. Валидация интеграции всех компонентов
- Проверка совместной работы всех новых компонентов
- Тестирование полного цикла от планирования до реализации

## Этап 10: Документирование и обучение (неделя 11)

10.1. Создание документации для новых компонентов
- Документация по фазе планирования
- Руководство по созданию агентов
- Документация по новым навыкам и контрольным точкам

10.2. Обновление существующей документации
- Обновление QWEN.md с учетом новых возможностей
- Обновление руководства по использованию
- Создание примеров использования новых компонентов

## Этап 11: Завершение и оптимизация (неделя 12)

11.1. Финальная проверка и оптимизация
- Проверка производительности новых компонентов
- Оптимизация процессов
- Устранение выявленных проблем

11.2. Подготовка к релизу
- Создание релизных заметок
- Подготовка миграционных руководств
- Финальное тестирование

Все этапы реализуются с учетом специфики Qwen Code CLI, включая плоскую структуру агентов и адаптацию существующих компонентов под новую архитектуру.
