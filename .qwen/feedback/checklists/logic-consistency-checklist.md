# Logic Consistency Checklist

Чек-лист для проверки логической целостности проекта.

---

## 1. Документация

- [ ] **1.1** Отсутствуют противоречия между документами
- [ ] **1.2** Версии в документации консистентны
- [ ] **1.3** Терминология используется единообразно
- [ ] **1.4** Определения терминов не противоречат друг другу
- [ ] **1.5** Инструкции не конфликтуют между собой

## 2. Пути и ссылки

- [ ] **2.1** Все относительные пути в документации существуют
- [ ] **2.2** Ссылки на файлы не битые
- [ ] **2.3** Имена файлов консистентны (kebab-case/snake_case)
- [ ] **2.4** Структура директорий соответствует документации
- [ ] **2.5** Cross-references между документами работают

## 3. Код

- [ ] **3.1** Отсутствует дублирование логики
- [ ] **3.2** Утилитные функции вынесены в общие модули
- [ ] **3.3** Конфигурация не дублируется
- [ ] **3.4** Константы определены централизованно
- [ ] **3.5** Шаблоны кода используются консистентно

## 4. Мертвый код

- [ ] **4.1** Неиспользуемые функции удалены
- [ ] **4.2** Неиспользуемые переменные удалены
- [ ] **4.3** Закомментированный код удален или документирован
- [ ] **4.4** Deprecated код помечен и планируется к удалению
- [ ] **4.5** Unused imports удалены

## 5. Агентская система

- [ ] **5.1** Ссылки на агентов существуют
- [ ] **5.2** nextAgent указывает на существующего агента
- [ ] **5.3** Отсутствуют orphan агенты (без вызовов)
- [ ] **5.4** Цепочки агентов логически завершены
- [ ] **5.5** Возврат управления реализован корректно

## 6. Конфигурация

- [ ] **6.1** Конфигурационные файлы не противоречат друг другу
- [ ] **6.2** Значения по умолчанию консистентны
- [ ] **6.3** Переменные окружения документированы
- [ ] **6.4** Секреты не закоммичены в репозиторий
- [ ] **6.5** Конфигурация валидируется

## 7. Зависимости

- [ ] **7.1** Версии зависимостей зафиксированы
- [ ] **7.2** Отсутствуют конфликтующие зависимости
- [ ] **7.3** Peer dependencies удовлетворены
- [ ] **7.4** Dev dependencies отделены от production
- [ ] **7.5** Зависимости регулярно обновляются

## 8. Тесты

- [ ] **8.1** Тесты не противоречат спецификациям
- [ ] **8.2** Тесты не дублируют друг друга
- [ ] **8.3** Mock данные консистентны
- [ ] **8.4** Test fixtures актуальны
- [ ] **8.5** Coverage отчеты точны

## 9. CI/CD

- [ ] **9.1** Pipeline скрипты не дублируются
- [ ] **9.2** Стадии pipeline логически упорядочены
- [ ] **9.3** Артефакты собираются консистентно
- [ ] **9.4** Environment переменные CI согласованы
- [ ] **9.5** Rollback процедуры определены

## 10. Архитектура

- [ ] **10.1** Архитектурные решения документированы (ADR)
- [ ] **10.2** ADR не противоречат друг другу
- [ ] **10.3** Слои архитектуры разделены четко
- [ ] **10.4** Границы модулей определены
- [ ] **10.5** Нарушения архитектуры отслеживаются

---

## Типы противоречий

### Критические
- 🚨 Конфликтующие требования в spec
- 🚨 Битые ссылки на критическую документацию
- 🚨 Циклические зависимости между модулями
- 🚨 Конфликт версий зависимостей

### Серьезные
- ⚠️ Дублирование бизнес-логики
- ⚠️ Несогласованность терминологии
- ⚠️ Мертвый код в production путях
- ⚠️ Несоответствие реализации spec

### Минорные
- 📝 Опечатки в документации
- 📝 Неактуальные примеры
- 📝 Лишние импорты
- 📝 Неиспользуемые переменные

---

## Метрики целостности

| Метрика | Формула | Цель |
|---------|---------|------|
| Documentation Consistency | (Непротиворечивые документы / Всего документов) × 100 | 100% |
| Dead Code Ratio | (Мертвые строки / Всего строк) × 100 | <5% |
| Duplicate Code Ratio | (Дублированные строки / Всего строк) × 100 | <10% |
| Broken Links | Количество битых ссылок | 0 |
| Orphan Agents | Агенты без вызовов | 0 |

---

## Использование

Проверяйте этот чек-лист:
- Перед каждым релизом
- После крупных рефакторингов
- При добавлении новой функциональности
- Регулярно (рекомендуется каждые 2 недели)

**Автоматическая проверка:** Запустите `logic-consistency-analyzer.sh` для автоматической проверки.

**Оценка:**
- ✅ 90-100%: Отлично
- ⚠️ 70-89%: Требуется улучшение
- ❌ <70%: Критические проблемы
