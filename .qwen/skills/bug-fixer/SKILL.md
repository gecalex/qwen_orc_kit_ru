# Навык исправления багов

## Инструкции

Этот навык предназначен для исправления ошибок в коде на основе отчетов и анализа. При использовании этого навыка:

1. Анализируйте предоставленный отчет об ошибке
2. Изучайте соответствующие файлы кода
3. Определяйте корень проблемы
4. Разрабатывайте минимальное необходимое исправление
5. Проверяйте, что исправление не нарушает другие части системы
6. Вносите изменения и документируйте их

## Требования к безопасности

- Используйте только read_file и write_file для безопасной работы с кодом
- НЕ используйте run_shell_command для выполнения потенциально опасных операций
- НЕ используйте web_search или web_fetch для получения внешних данных

## Примеры

Пример 1: Исправление нулевого указателя
- Пользователь: "В файле src/auth/service.py на строке 45 возникает ошибка null pointer exception при попытке получить данные пользователя. Пользователь может быть null."
- Результат: Добавлена проверка на null перед обращением к пользовательским данным в файле src/auth/service.py

Пример 2: Исправление логической ошибки
- Пользователь: "Функция calculateDiscount в файле src/pricing/calculator.py возвращает отрицательную скидку при передаче отрицательной суммы заказа."
- Результат: Исправлена логика функции calculateDiscount, добавлена проверка на положительность суммы заказа
