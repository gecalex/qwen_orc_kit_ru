# Навык поиска багов

## Инструкции

Этот навык предназначен для поиска ошибок, уязвимостей и проблем в коде. При использовании этого навыка:

1. Ищите логические ошибки в коде
2. Выявляйте потенциальные уязвимости безопасности
3. Обнаруживайте проблемы с производительностью
4. Идентифицируйте мертвый или неиспользуемый код
5. Проверяйте соответствие кода лучшим практикам

## Требования к безопасности

- Используйте только read_file и read_many_files для безопасного анализа кода
- web_search только для получения информации из доверенных источников
- НЕ используйте write_file, run_shell_command или task для изменения системы

## Примеры

Пример 1: Поиск уязвимостей безопасности
- Пользователь: "Проанализируй файлы в директории src/api/ на предмет уязвимостей безопасности, таких как SQL-инъекции, XSS, CSRF."
- Результат: Отчет с перечнем найденных потенциальных уязвимостей в файлах src/api/, их уровень критичности и рекомендации по устранению

Пример 2: Поиск логических ошибок
- Пользователь: "Найди потенциальные логические ошибки в модуле аутентификации в файлах src/auth/"
- Результат: Отчет с описанием потенциальных логических ошибок в аутентификации, включая возможные обходы проверок, неправильную обработку ошибок и т.д.
