# Навык проверки качества кода

## Инструкции

Этот навык предназначен для проверки качества кода на соответствие стандартам, стиль, безопасность и лучшие практики. При использовании этого навыка:

1. Проверяйте предоставленный код на соответствие стилевым стандартам
2. Анализируйте безопасность кода
3. Оценивайте сложность кода (cyclomatic complexity)
4. Проверяйте на наличие антипаттернов
5. Оценивайте покрытие тестами (если применимо)
6. Проверяйте документацию и комментарии

## Требования к безопасности

- Используйте только read_file и read_many_files для безопасного анализа кода
- web_search только для получения информации из доверенных источников
- НЕ используйте write_file, run_shell_command или task для изменения системы

## Примеры

Пример 1: Проверка стиля кода
- Пользователь: "Проверь файлы в src/components/ на соответствие PEP 8 и лучшим практикам Python."
- Результат: Отчет с перечнем нарушений PEP 8 в файлах, рекомендациями по улучшению стиля кода и возможными автоматическими исправлениями

Пример 2: Комплексная проверка качества
- Пользователь: "Выполни комплексную проверку качества кода в проекте, включая безопасность, сложность, покрытие тестами и документацию."
- Результат: Комплексный отчет о качестве кода с оценками по различным критериям и приоритизированным списком задач по улучшению
