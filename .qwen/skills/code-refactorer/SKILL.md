# Навык рефакторинга кода

## Инструкции

Этот навык предназначен для рефакторинга кода с целью улучшения читаемости, производительности и сопровождаемости. При использовании этого навыка:

1. Анализируйте предоставленный код на наличие проблем с архитектурой, дублированием, сложностью
2. Применяйте паттерны проектирования для улучшения структуры кода
3. Упрощайте сложные выражения и функции
4. Улучшайте именование переменных, функций и классов
5. Разбивайте большие функции и классы на более мелкие, с одной ответственностью
6. Удаляйте дублирующийся код и выносите общие элементы в переиспользуемые компоненты

## Требования к безопасности

- Используйте только read_file и write_file для безопасной работы с кодом
- НЕ изменяйте логику работы программы при рефакторинге
- НЕ используйте run_shell_command для выполнения потенциально опасных операций
- Проверяйте, что рефакторинг не нарушает существующую функциональность

## Примеры

Пример 1: Упрощение сложной функции
- Пользователь: "Рефактори эту функцию, чтобы она была более читаемой и соответствовала принципам чистого кода."
- Результат: Улучшенная версия функции с лучшим именованием, разбитая на более мелкие функции

Пример 2: Улучшение структуры класса
- Пользователь: "Рефактори этот класс, чтобы он соответствовал принципам SOLID."
- Результат: Переработанная структура класса с разделением ответственностей
