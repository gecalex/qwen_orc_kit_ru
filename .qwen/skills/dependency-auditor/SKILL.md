# Навык аудита зависимостей

## Инструкции

Этот навык предназначен для проверки проекта на наличие проблем с зависимостями. При использовании этого навыка:

1. Анализируйте файлы зависимостей проекта (package.json, requirements.txt, Cargo.toml и т.д.)
2. Проверяйте зависимости на уязвимости, используя внешние источники информации
3. Определяйте неиспользуемые зависимости через анализ кода
4. Сравнивайте версии зависимостей с актуальными
5. Выявляйте конфликты зависимостей
6. Документируйте все найденные проблемы с рекомендациями по их устранению

## Типы проблем для проверки

1. Устаревшие зависимости
2. Зависимости с известными уязвимостями
3. Неиспользуемые зависимости
4. Отсутствующие зависимости
5. Конфликты версий зависимостей
6. Лицензионные конфликты

## Примеры

Пример 1: Обнаружение устаревшей зависимости
- Вход: Файл package.json с lodash@4.17.11
- Результат: Обнаружена устаревшая зависимость - lodash@4.17.11, актуальная версия - lodash@4.17.21

Пример 2: Обнаружение уязвимой зависимости
- Вход: Файл package.json с express@4.17.1
- Результат: Обнаружена уязвимая зависимость - express@4.17.1, уязвимость CVE-2020-28168
