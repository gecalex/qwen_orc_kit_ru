# Навык генерации документации

## Инструкции

Этот навык предназначен для генерации технической документации для кода, API и архитектурных решений. При использовании этого навыка:

1. Анализируйте предоставленный код или архитектурные компоненты
2. Определяйте основные функции, интерфейсы и зависимости
3. Создавайте четкую и понятную документацию в формате Markdown
4. Включайте примеры использования, где это уместно
5. Обновляйте существующую документацию при внесении изменений

## Требования к безопасности

- Используйте только read_file и read_many_files для безопасного анализа кода
- НЕ используйте write_file, run_shell_command или task для изменения системы без явного указания
- Сохраняйте конфиденциальность информации при создании документации

## Примеры

Пример 1: Генерация документации для API
- Пользователь: "Создай документацию для этого API endpoint в формате Markdown."
- Результат: Файл документации с описанием endpoint, параметров, возвращаемых значений и примеров использования

Пример 2: Документирование функции
- Пользователь: "Напиши документацию для этой функции в соответствии с Google Python Style Guide."
- Результат: Документация с описанием параметров, возвращаемого значения и примера использования
