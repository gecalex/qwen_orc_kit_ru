# Навык генерации заголовков отчетов

## Когда использовать
- При создании любого отчета агентом
- При формировании заголовков для отчетов об ошибках
- При создании отчетов по аудиту безопасности
- При генерации отчетов о зависимостях
- При создании любых выводов, требующих последовательного форматирования

## Инструкции

### Фаза 1: Сбор метаданных
1.1. Определить статус выполнения (завершен/не завершен/в процессе)
1.2. Зафиксировать продолжительность выполнения
1.3. Записать имя агента
1.4. Указать текущую фазу

### Фаза 2: Форматирование заголовка
2.1. Создать заголовок отчета
2.2. Добавить метаданные в формате YAML или Markdown
2.3. Включить исполнительное резюме

### Фаза 3: Генерация разделов
3.1. Добавить раздел "Выполненная работа"
3.2. Добавить раздел "Внесенные изменения"
3.3. Добавить раздел "Результаты валидации"
3.4. Добавить раздел "Метрики"
3.5. Добавить раздел "Обнаруженные ошибки" (если есть)
3.6. Добавить раздел "Следующие шаги"
3.7. Добавить раздел "Артефакты"

## Формат ввода
```json
{
  "agentName": "string",
  "status": "completed|in_progress|failed",
  "duration": "string",
  "phase": "number|string",
  "tasksCompleted": ["string"],
  "filesModified": ["string"],
  "validationResults": {},
  "metrics": {},
  "errors": ["string"],
  "artifacts": ["string"]
}
```

## Формат вывода
Markdown отчет со стандартизированным заголовком и разделами

## Примеры

### Пример 1: Отчет об анализе кода
**Вход:**
```json
{
  "agentName": "work_dev_code_analyzer",
  "status": "completed",
  "duration": "2 мин 34 сек",
  "phase": 2,
  "tasksCompleted": ["Анализ структуры", "Проверка стандартов"],
  "filesModified": [],
  "validationResults": {"passed": true},
  "metrics": {"filesAnalyzed": 42, "errorsFound": 0},
  "errors": [],
  "artifacts": ["reports/code-analysis.md"]
}
```

**Выход:**
```markdown
# Отчет: Анализ кода завершен

**Статус:** завершен  
**Продолжительность:** 2 мин 34 сек  
**Агент:** work_dev_code_analyzer  
**Фаза:** 2

## Исполнительное резюме
Проведен полный анализ кодовой базы...

## Выполненная работа
- [x] Анализ структуры проекта
- [x] Проверка соответствия стандартам

## Внесенные изменения
Нет

## Результаты валидации
Все проверки пройдены успешно.

## Метрики
- Файлов проанализировано: 42
- Ошибок найдено: 0

## Обнаруженные ошибки
Нет

## Следующие шаги
- Приступить к реализации

## Артефакты
- `reports/code-analysis.md`
```

### Пример 2: Отчет об исправлении ошибок
**Вход:**
```json
{
  "agentName": "work_health_bug_fixer",
  "status": "completed",
  "duration": "15 мин 42 сек",
  "phase": 3,
  "tasksCompleted": ["Исправление ошибок линтинга", "Удаление мертвого кода"],
  "filesModified": ["src/main.js", "src/utils.js"],
  "validationResults": {"passed": true, "lintPassed": true},
  "metrics": {"bugsFixed": 5, "linesChanged": 47},
  "errors": [],
  "artifacts": ["reports/bug-fix-report.md"]
}
```

## Интеграция с агентами

Навык должен вызываться в конце выполнения задачи агента:

```markdown
## Завершение задачи
- Собрать метаданные выполнения
- Вызвать generate-report-header Skill
- Сформировать итоговый отчет
- Сохранить артефакты
```
