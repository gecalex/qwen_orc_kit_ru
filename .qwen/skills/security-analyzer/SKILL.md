# Навык анализа безопасности

## Инструкции

Этот навык предназначен для анализа безопасности кода и выявления потенциальных уязвимостей. При использовании этого навыка:

1. Проанализируйте предоставленный код на наличие известных паттернов уязвимостей
2. Проверьте конфигурационные файлы и настройки безопасности
3. Оцените обработку пользовательского ввода и внешних данных
4. Классифицируйте уязвимости по уровню критичности
5. Предложите конкретные методы устранения уязвимостей

## Требования к безопасности

- Используйте только read_file и read_many_files для безопасного анализа кода
- web_search только для получения информации из доверенных источников
- НЕ используйте write_file, run_shell_command или task для изменения системы

## Примеры

Пример 1: Поиск SQL-инъекций
- Пользователь: "Проанализируй файлы в директории src/database/ на предмет возможных SQL-инъекций."
- Результат: Отчет с перечнем потенциальных SQL-инъекций в файлах src/database/, уровень критичности и рекомендации по использованию параметризованных запросов

Пример 2: Комплексный аудит безопасности
- Пользователь: "Выполни комплексный аудит безопасности веб-приложения, включая проверку обработки пользовательского ввода, аутентификации и авторизации."
- Результат: Комплексный отчет безопасности с уязвимостями, отсортированными по критичности, и подробными рекомендациями по устранению
