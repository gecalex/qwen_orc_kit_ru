# Навык сканирования безопасности

## Инструкции

Этот навык предназначен для проверки кода на наличие потенциальных уязвимостей безопасности. При использовании этого навыка:

1. Анализируйте предоставленные файлы кода на наличие известных паттернов уязвимостей
2. Ищите жестко закодированные учетные данные
3. Проверяйте на наличие потенциальных SQL-инъекций
4. Проверяйте на наличие потенциальных XSS-уязвимостей
5. Ищите потенциальные утечки чувствительной информации
6. Документируйте все найденные уязвимости с рекомендациями по их устранению

## Типы уязвимостей для проверки

1. Жестко закодированные учетные данные (пароли, API-ключи и т.д.)
2. Потенциальные SQL-инъекции
3. Потенциальные XSS-уязвимости
4. Небезопасная десериализация
5. Утечки чувствительной информации через логирование
6. Неправильная проверка входных данных
7. Уязвимости в логике приложения

## Примеры

Пример 1: Обнаружение жестко закодированных паролей
- Вход: Файл с содержимым "password = 'hardcoded_password'"
- Результат: Обнаружена потенциальная уязвимость - жестко закодированный пароль в строке X

Пример 2: Обнаружение потенциальной SQL-инъекции
- Вход: Файл с содержимым "query = f'SELECT * FROM users WHERE id = {user_input}'"
- Результат: Обнаружена потенциальная уязвимость - SQL-инъекция в строке X
