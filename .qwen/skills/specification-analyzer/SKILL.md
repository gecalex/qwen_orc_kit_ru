# Навык анализа спецификаций

## Инструкции

Этот навык предназначен для анализа спецификаций на полноту, ясность и соответствие требованиям. При использовании этого навыка:

1. Проверяйте, содержит ли спецификация все необходимые разделы:
   - Краткое описание
   - Контекст
   - Акторы
   - Функциональные и нефункциональные требования
   - Сценарии использования
   - Условия успеха
   - Ограничения
   - Предположения
   - Возможные риски

2. Оценивайте ясность и однозначность формулировок
3. Проверяйте, можно ли требования протестировать
4. Определяйте, есть ли противоречия между разделами
5. Оценивайте полноту покрытия функциональности
6. Проверяйте, соответствуют ли условия успеха функциональным требованиям

## Требования к безопасности

- Используйте только read_file для безопасного анализа спецификаций
- НЕ используйте write_file, run_shell_command или task для изменения системы без явного указания
- Сохраняйте конфиденциальность информации при анализе спецификаций

## Примеры

Пример 1: Проверка полноты спецификации
- Пользователь: "Проанализируй эту спецификацию на полноту и ясность."
- Результат: Отчет с перечнем недостающих разделов, неоднозначных формулировок и рекомендациями по улучшению

Пример 2: Проверка тестопригодности требований
- Пользователь: "Проверь, можно ли протестировать функциональные требования в этой спецификации."
- Результат: Анализ требований на тестопригодность с указанием проблемных пунктов и предложениями по улучшению
