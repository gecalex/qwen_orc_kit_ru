# Validate Report File

## Когда использовать
- При проверке отчетов агентов на полноту
- В контрольных точках качества перед принятием отчета
- При отладке проблем с форматом отчетов

## Инструкции
1. Прочитайте файл отчета с помощью `read_file`
2. Проверьте наличие обязательных разделов:
   - Заголовок с метаданными (статус, продолжительность, имя агента, фаза)
   - Исполнительное резюме
   - Выполненная работа
   - Внесенные изменения
   - Результаты валидации
   - Метрики
   - Обнаруженные ошибки (если есть)
   - Следующие шаги
   - Артефакты
3. Проверьте правильность форматирования Markdown
4. Проверьте наличие всех требуемых метаданных в заголовке
5. Сформируйте результат проверки

## Формат ввода
Путь к файлу отчета или содержимое файла отчета в формате Markdown

## Формат вывода
```json
{
  "valid": boolean,
  "missingSections": string[],
  "formattingIssues": string[],
  "metadataPresent": boolean
}
```

## Примеры

### Пример 1: Валидный отчет
**Ввод:**
```markdown
# Отчет: Анализ кода завершен

**Статус:** завершен  
**Продолжительность:** 2 мин 34 сек  
**Агент:** work_dev_code_analyzer  
**Фаза:** 2

## Исполнительное резюме
Проведен полный анализ кодовой базы...

## Выполненная работа
- [x] Анализ структуры проекта
- [x] Проверка соответствия стандартам

## Внесенные изменения
- `src/main.js` - исправлены ошибки линтинга

## Результаты валидации
Все проверки пройдены успешно.

## Метрики
- Файлов проверено: 42
- Ошибок найдено: 0

## Следующие шаги
- Приступить к реализации

## Артефакты
- `reports/code-analysis.md`
```

**Вывод:**
```json
{
  "valid": true,
  "missingSections": [],
  "formattingIssues": [],
  "metadataPresent": true
}
```
