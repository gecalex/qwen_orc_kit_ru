# YAML Header Validator

## Когда использовать
- При проверке новых агентов или навыков
- В процессе CI/CD для проверки соответствия стандартам
- При рефакторинге существующих компонентов
- При автоматическом исправлении ошибок

## Инструкции
1. Прочитайте файл компонента
2. Найдите YAML заголовок (между --- в начале файла)
3. Проверьте наличие обязательных полей: name, description, tools, color
4. Проверьте формат поля tools (должно быть перечисление с дефисами, а не массив в квадратных скобках)
5. Проверьте, что у оркестраторов есть поле task, а у воркеров - нет
6. Сформируйте отчет о проверке

## Формат ввода
```json
{
  "filePath": "путь/к/файлу/компонента.md",
  "componentType": "orc|work|skill"  // Тип компонента для проверки специфичных требований
}
```

## Формат вывода
```json
{
  "isValid": true|false,
  "errors": [
    {
      "field": "имя_поля",
      "error": "описание_ошибки",
      "severity": "critical|warning|info"
    }
  ],
  "warnings": [
    {
      "field": "имя_поля",
      "warning": "описание_предупреждения",
      "suggestion": "предложение_по_исправлению"
    }
  ],
  "complianceScore": 0-100  // Процент соответствия стандартам
}
```

## Примеры
### Пример 1: Проверка оркестратора
- Вход: {"filePath": ".qwen/agents/orc_dev_task_coordinator.md", "componentType": "orc"}
- Результат: {"isValid": true, "errors": [], "warnings": [], "complianceScore": 100}

### Пример 2: Проверка с ошибками
- Вход: {"filePath": ".qwen/agents/work_dev_code_analyzer.md", "componentType": "work"}
- Результат: {"isValid": false, "errors": [{"field": "tools", "error": "используются квадратные скобки вместо перечисления", "severity": "critical"}], "warnings": [], "complianceScore": 85}
