# Шаблон отчета агента

## Структура унифицированного отчета

Для обеспечения согласованности и возможности автоматической обработки все агенты должны использовать следующую структуру отчета:

```markdown
# {ТипОтчета} Report: {Версия}

**Статус**: ✅ УСПЕШНО | ⚠️ ЧАСТИЧНО | ❌ НЕУДАЧНО
**Продолжительность**: {время выполнения}
**Агент**: {имя-агента}
**Фаза**: {номер-фазы}

---

## Итоговое резюме
{Краткое изложение проделанной работы и ключевых результатов}

## Выполненная работа
- Задача 1: Статус (Выполнено/Неудачно/Частично)
- Задача 2: Статус (Выполнено/Неудачно/Частично)

## Внесенные изменения
- Измененные файлы: {список с количеством}
- Созданные файлы: {список с количеством}
- Удаленные файлы: {список с количеством}

## Результаты проверки
- Команда: Результат (УСПЕШНО/НЕУДАЧНО)
- Детали: {конкретные результаты проверки}

## Метрики
- Продолжительность: {время}
- Выполненные задачи: {количество}
- Внесенные изменения: {количество}
- Проверки валидации: {количество}

## Обнаруженные ошибки
- Ошибка 1: Описание и контекст
- Ошибка 2: Описание и контекст

## Следующие шаги
- Для оркестратора: {что должен сделать оркестратор дальше}
- Шаги восстановления при неудаче: {шаги восстановления}

## Артефакты
- Файл плана: {путь}
- Отчет: {путь}
- Дополнительные артефакты: {пути}
```

## Рекомендации по заполнению

1. **Статус**: Используйте эмодзи для быстрого определения статуса
2. **Продолжительность**: Указывайте в формате ЧЧ:ММ:СС или количестве минут
3. **Агент**: Используйте полное имя агента как в файле агента
4. **Фаза**: Указывайте номер текущей фазы выполнения
5. **Итоговое резюме**: Должно давать общее представление о проделанной работе
6. **Выполненная работа**: Перечислите конкретные задачи с их статусами
7. **Внесенные изменения**: Укажите все файлы, с которыми была работа (изменение, создание, удаление)
8. **Результаты проверки**: Включите результаты всех проверок качества
9. **Метрики**: Количественные показатели выполнения
10. **Обнаруженные ошибки**: Все возникшие ошибки с описанием
11. **Следующие шаги**: Рекомендации по дальнейшим действиям
12. **Артефакты**: Укажите все созданные артефакты

## Использование навыков

Для генерации заголовков отчетов используйте навык `generate-report-header`:

```markdown
# Использование навыка
- Используйте `generate-report-header` для создания согласованных заголовков
- Передайте ему тип отчета, статус, продолжительность и другую информацию
- Навык создаст заголовок в стандартизированном формате
```

## Проверка отчетов

Для проверки полноты и формата отчетов используйте навык `validate-report-file`:

```markdown
# Проверка отчета
- Используйте `validate-report-file` для проверки наличия всех обязательных разделов
- Навык проверит формат и полноту отчета
- При необходимости исправьте отчет в соответствии с требованиями
```
