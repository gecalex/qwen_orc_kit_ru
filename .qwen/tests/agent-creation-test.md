# Тестовый сценарий: Проверка создания агентов

## Цель
Проверить, что процесс создания новых агентов работает корректно и создает агентов с правильной структурой и именованием.

## Предварительные условия
- Система Qwen Code CLI установлена и настроена
- Доступны скрипты создания агентов
- Доступен meta-agent для создания агентов

## Тестовые сценарии

### Сценарий 1: Создание оркестратора с использованием скрипта

**Шаги:**
1. Запустить скрипт создания агента:
   ```
   ./scripts/agent-creation/create-agent.sh orc test_new_coordinator
   ```
2. Проверить, что файл агента создан в `.qwen/agents/orc_test_new_coordinator.md`
3. Проверить содержимое файла на соответствие шаблону
4. Проверить YAML заголовок на корректность

**Ожидаемый результат:**
- Файл агента успешно создан
- Файл содержит корректный YAML заголовок
- Файл соответствует структуре шаблона агента
- Имя файла соответствует стандарту именования

### Сценарий 2: Создание воркера с использованием скрипта

**Шаги:**
1. Запустить скрипт создания агента:
   ```
   ./scripts/agent-creation/create-agent.sh work test_new_processor
   ```
2. Проверить, что файл агента создан в `.qwen/agents/work_test_new_processor.md`
3. Проверить содержимое файла на соответствие шаблону воркера
4. Проверить YAML заголовок на корректность

**Ожидаемый результат:**
- Файл агента успешно создан
- Файл содержит корректный YAML заголовок
- Файл соответствует структуре шаблона воркера
- Имя файла соответствует стандарту именования

### Сценарий 3: Проверка уникальности имени агента

**Шаги:**
1. Создать агента с определенным именем
2. Попытаться создать второго агента с тем же именем
3. Проверить, что система предотвращает создание дубликата

**Ожидаемый результат:**
- При попытке создать агента с уже существующим именем, процесс прерывается с ошибкой
- Система сообщает о существующем агенте с таким именем

### Сценарий 4: Создание агента с использованием meta-agent

**Шаги:**
1. Вызвать meta-agent для создания нового агента
2. Указать тип агента (оркестратор или воркер), домен и функциональность
3. Проверить созданный файл агента на корректность структуры

**Ожидаемый результат:**
- Агент создан с правильной структурой
- Агент соответствует указанному типу и функциональности
- Использованы правильные шаблоны и стандарты

## Критерии успеха
- Все тестовые сценарии выполняются без ошибок
- Созданные агенты соответствуют шаблонам и стандартам
- Процесс создания агентов автоматизирован и упрощен
- Имена агентов соответствуют системе именования
- Проверка уникальности имени работает корректно

## Проверяемые аспекты
- Корректность YAML заголовков
- Соответствие структуре шаблона
- Правильность именования файлов
- Функциональность скриптов создания
- Работа проверки уникальности имен
- Интеграция с MCP серверами
- Использование стандартных навыков
