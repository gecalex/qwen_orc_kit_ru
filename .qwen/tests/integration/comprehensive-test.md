# Комплексный тестовый сценарий

## Цель
Проверить интеграцию всех новых компонентов системы Qwen Code Orchestrator Kit.

## Подготовка
1. Убедитесь, что все компоненты системы установлены и настроены
2. Проверьте, что скрипты в `scripts/orchestration-tools/` исполняемы
3. Убедитесь, что все агенты и команды доступны

## Тестовые сценарии

### Сценарий 1: Проверка системы безопасности
**Цель**: Проверить функциональность агента безопасности и связанных компонентов

**Шаги**:
1. Запустите команду `/health-security`
2. Проверьте, что агент `security-analyzer` запускается
3. Убедитесь, что генерируется отчет о безопасности
4. Проверьте, что скрипт `generate-security-report.sh` работает

**Ожидаемый результат**: 
- Команда выполняется без ошибок
- Агент анализирует код на наличие уязвимостей
- Генерируется отчет о безопасности

### Сценарий 2: Проверка системы анализа зависимостей
**Цель**: Проверить функциональность агента анализа зависимостей и связанных компонентов

**Шаги**:
1. Запустите команду `/health-deps`
2. Проверьте, что агент `dependency-analyzer` запускается
3. Убедитесь, что генерируется отчет об анализе зависимостей
4. Проверьте, что скрипт `update-dependencies.sh` работает

**Ожидаемый результат**: 
- Команда выполняется без ошибок
- Агент анализирует зависимости на наличие проблем
- Генерируется отчет о состоянии зависимостей

### Сценарий 3: Проверка системы обнаружения мертвого кода
**Цель**: Проверить функциональность агента обнаружения мертвого кода и связанных компонентов

**Шаги**:
1. Запустите команду `/health-cleanup`
2. Проверьте, что агент `dead-code-detector` запускается
3. Убедитесь, что генерируется отчет о мертвом коде
4. Проверьте, что скрипт `remove-dead-code.sh` работает

**Ожидаемый результат**: 
- Команда выполняется без ошибок
- Агент анализирует код на наличие мертвого кода
- Генерируется отчет о найденном мертвом коде

### Сценарий 4: Проверка контрольных точек качества
**Цель**: Проверить функциональность системы контрольных точек качества

**Шаги**:
1. Запустите команду `/run-quality-gate`
2. Проверьте, что запускаются все скрипты проверки:
   - `check-coverage.sh`
   - `check-security.sh`
   - `check-bundle-size.sh`
3. Убедитесь, что генерируется сводный отчет

**Ожидаемый результат**: 
- Команда выполняется без ошибок
- Все проверки качества проходят
- Генерируется сводный отчет о качестве

### Сценарий 5: Проверка системы MCP-конфигураций
**Цель**: Проверить функциональность системы переключения MCP-конфигураций

**Шаги**:
1. Запустите скрипт `scripts/orchestration-tools/switch-mcp.sh`
2. Выберите одну из доступных конфигураций
3. Убедитесь, что конфигурация применяется
4. Проверьте, что создается файл `.mcp.json`

**Ожидаемый результат**: 
- Скрипт запускается без ошибок
- Конфигурация успешно применяется
- Создается или обновляется файл `.mcp.json`

### Сценарий 6: Проверка системы worktree
**Цель**: Проверить функциональность системы Git worktree

**Шаги**:
1. Запустите команду `/worktree-create test-feature`
2. Убедитесь, что создается новый worktree
3. Проверьте, что можно переключиться в worktree
4. Запустите команду `/worktree-list`
5. Запустите команду `/worktree-remove test-feature`

**Ожидаемый результат**: 
- Worktree успешно создается
- Список worktree отображается корректно
- Worktree успешно удаляется

### Сценарий 7: Проверка системы вебхуков
**Цель**: Проверить функциональность системы вебхуков

**Шаги**:
1. Проверьте наличие файла `.qwen/settings.local.json.example`
2. Запустите скрипт `scripts/orchestration-tools/handle-webhooks.sh` с параметром "Stop"
3. Убедитесь, что обрабатываются настроенные вебхуки
4. Проверьте логи в `logs/webhook_events.log`

**Ожидаемый результат**: 
- Скрипт запускается без ошибок
- Вебхуки обрабатываются согласно конфигурации
- События логируются в файл

## Комплексный сценарий интеграции
**Цель**: Проверить взаимодействие всех компонентов системы

**Шаги**:
1. Создайте новый worktree для тестовой фичи
2. Внесите изменения в код (добавьте тестовый файл)
3. Запустите проверку безопасности
4. Запустите проверку зависимостей
5. Запустите проверку мертвого кода
6. Запустите комплексную проверку качества
7. Удалите worktree

**Ожидаемый результат**:
- Все компоненты системы работают согласованно
- Все проверки проходят успешно
- Нет конфликтов между компонентами

## Критерии успеха
- Все тестовые сценарии проходят без ошибок
- Все компоненты системы функционируют корректно
- Интеграция между компонентами работает должным образом
- Нет конфликтов и ошибок взаимодействия
