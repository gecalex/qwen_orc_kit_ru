# Тесты для Qwen Code Orchestrator Kit

## Функциональные тесты

### Тест 1: Проверка наличия агентов
- Цель: Убедиться, что все базовые агенты существуют
- Шаги:
  1. Проверить наличие файлов в директории `.qwen/agents/`
  2. Убедиться, что каждый файл содержит корректный YAML frontmatter
- Ожидаемый результат: Все 5 базовых агентов присутствуют и имеют корректный формат

### Тест 2: Проверка скриптов контрольных точек
- Цель: Убедиться, что скрипты контрольных точек качества работают
- Шаги:
  1. Запустить `./scripts/run_quality_gate.sh 1`
  2. Проверить вывод скрипта
- Ожидаемый результат: Скрипт выполняется без ошибок

### Тест 3: Проверка шаблонов
- Цель: Убедиться, что шаблоны существуют и имеют корректный формат
- Шаги:
  1. Проверить наличие файла `.qwen/specify/templates/spec-template.md`
  2. Убедиться, что шаблон содержит все необходимые секции
- Ожидаемый результат: Шаблон спецификации существует и полный

## Интеграционные тесты

### Тест 4: Полный цикл использования агента
- Цель: Проверить возможность запуска агента через систему
- Шаги:
  1. Создать файл задачи в `state/test_task.json`
  2. Запустить один из агентов с тестовой задачей
  3. Проверить результат выполнения
- Ожидаемый результат: Агент успешно выполняет задачу

### Тест 5: Проверка контрольных точек качества
- Цель: Убедиться, что контрольные точки корректно проверяют код
- Шаги:
  1. Создать тестовый файл с намеренно внесенными проблемами
  2. Запустить соответствующую контрольную точку
  3. Проверить, что проблемы были обнаружены
- Ожидаемый результат: Контрольная точка обнаруживает проблемы

## Модульные тесты

### Тест 6: Проверка bash-скриптов
- Цель: Убедиться, что bash-скрипты корректно обрабатывают разные сценарии
- Шаги:
  1. Запустить скрипты с разными параметрами
  2. Проверить корректность обработки ошибок
  3. Проверить возвращаемые коды
- Ожидаемый результат: Скрипты корректно обрабатывают нормальные и крайние случаи

## Тесты документации

### Тест 7: Проверка актуальности документации
- Цель: Убедиться, что документация соответствует реализации
- Шаги:
  1. Проверить, что все упомянутые файлы и директории существуют
  2. Убедиться, что описанные команды работают
  3. Проверить примеры на работоспособность
- Ожидаемый результат: Документация полностью соответствует реализации
