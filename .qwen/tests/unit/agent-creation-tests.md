# Тесты для проверки корректности создаваемых агентов

## Модульные тесты

### Тест 1: Проверка формата имени агента
- Проверяет, что имя агента соответствует формату `{типы}_{домен}_{имя}`
- Проверяет, что имя использует kebab-case
- Проверяет, что тип агента - либо `orc_`, либо `work_`

### Тест 2: Проверка YAML заголовка
- Проверяет наличие обязательных полей: `name`, `description`, `color`
- Для оркестраторов проверяет наличие поля `color`
- Проверяет корректность формата поля `tools`

### Тест 3: Проверка структуры файла агента
- Проверяет наличие обязательных разделов: `## Назначение`, `## Инструкции`
- Проверяет, что раздел `Назначение` не пустой
- Проверяет наличие разделов `## Формат файла плана` и `## Стандартизированная отчетность`

### Тест 4: Проверка уникальности имени
- Проверяет, что создаваемый агент имеет уникальное имя в системе
- Проверяет, что не происходит перезаписи существующих агентов без подтверждения

## Интеграционные тесты

### Тест 5: Создание агента через мета-агента
- Запускает мета-агента с параметрами
- Проверяет, что файл агента создан
- Проверяет корректность содержимого файла

### Тест 6: Создание агента через скрипт
- Вызывает скрипт `generate-from-template.sh` с параметрами
- Проверяет, что файл агента создан
- Проверяет корректность содержимого файла

### Тест 7: Проверка всех доменов
- Создает агенты во всех поддерживаемых доменах: `dev`, `frontend`, `backend`, `testing`, `research`, `security`
- Проверяет корректность каждого созданного агента

### Тест 8: Проверка всех типов агентов
- Создает оркестраторы и воркеры
- Проверяет корректность каждого типа агента

## Тесты производительности

### Тест 9: Время создания агента
- Измеряет время, необходимое для создания агента
- Проверяет, что время укладывается в допустимые рамки

## Тесты безопасности

### Тест 10: Проверка на валидацию входных данных
- Проверяет, что система корректно обрабатывает некорректные входные данные
- Проверяет защиту от инъекций в именах и описаниях

## Сценарии тестирования

### Сценарий 1: Создание стандартного воркера
1. Вызвать мета-агента с параметрами: тип `work`, домен `dev`, имя `code-analyzer`, описание "Агент для анализа кода"
2. Проверить создание файла `.qwen/agents/work_dev_code-analyzer.md`
3. Проверить корректность YAML заголовка
4. Проверить наличие всех обязательных разделов

### Сценарий 2: Создание оркестратора
1. Вызвать мета-агента с параметрами: тип `orc`, домен `testing`, имя `quality-assurer`, описание "Оркестратор для обеспечения качества"
2. Проверить создание файла `.qwen/agents/orc_testing_quality-assurer.md`
3. Проверить корректность YAML заголовка
4. Проверить наличие всех обязательных разделов

### Сценарий 3: Проверка уникальности
1. Создать агента с определенным именем
2. Попытаться создать второго агента с тем же именем
3. Проверить, что система корректно обрабатывает конфликт имен

## Скрипт автоматического тестирования

Для автоматизации тестирования создайте скрипт `tests/run-agent-creation-tests.sh`, который будет:
1. Создавать тестовые агенты
2. Проверять их корректность
3. Удалять временные файлы после тестирования
4. Генерировать отчет о результатах тестирования
