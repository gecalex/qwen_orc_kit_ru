# Установка Qwen Code Orchestrator Kit

Данный документ описывает процесс установки и начальной настройки Qwen Code Orchestrator Kit.

## Требования

- Git
- Python 3.11 или выше
- Node.js (по желанию, для JavaScript/TypeScript проектов)
- Установленный Qwen Code CLI

## Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/ваш-репозиторий/qwen_orc_kit_ru.git
cd qwen_orc_kit_ru
```

### 2. Установка зависимостей

Для Python проектов:

```bash
# Установка uv (если не установлен)
pip install uv

# Установка зависимостей проекта
uv sync --dev
```

Для JavaScript/TypeScript проектов:

```bash
# Установка зависимостей
npm install
# или
pnpm install
```

### 3. Настройка Qwen CLI

1. Убедитесь, что Qwen Code CLI установлен в вашей системе
2. Установите MCP-сервера, если они требуются для вашего проекта
3. Настройте `.qwen/settings.json` с необходимыми параметрами

### 4. Проверка установки

Для проверки корректности установки выполните:

```bash
# Проверка доступных агентов
ls .qwen/agents/
```

Вы должны увидеть следующие файлы агентов:
- `bug-fixer.md`
- `bug-hunter.md`
- `code-quality-checker.md`
- `security-orchestrator.md`
- `tech-translator-ru.md`

## Запуск первого задания

После установки вы можете запустить первое задание через оркестратор:

```bash
# Пример запуска агента
task '{
  "subagent_type": "bug-hunter",
  "description": "Поиск потенциальных проблем в коде",
  "prompt": "Проанализируй файлы в директории src/ на предмет потенциальных багов и уязвимостей."
}'
```

## Использование экспериментальных навыков

Для использования экспериментальных навыков:

1. Убедитесь, что в `.qwen/settings.json` включена опция экспериментальных навыков:

```json
{
  "tools": {
    "experimental": {
      "skills": true
    }
  }
}
```

2. Запустите Qwen Code с флагом `--experimental-skills`:

```bash
qwen --experimental-skills
```

3. Модель будет автоматически использовать доступные навыки на основе вашего запроса, или вы можете явно вызвать навык командой:

```
/skills <имя-навыка>
```

Навыки находятся в директории `.qwen/skills/` и расширяют возможности агентов за счет модульных возможностей.

## Использование процесса разработки на основе спецификаций

Для использования процесса разработки на основе спецификаций:

1. Убедитесь, что установлены все необходимые зависимости
2. Используйте команды `speckit.*` для работы с процессом:
   - `speckit.specify` - создание спецификации
   - `speckit.clarify` - уточнение спецификации
   - `speckit.plan` - создание плана реализации
   - `speckit.tasks` - генерация задач
   - `speckit.implement` - выполнение задач

3. Спецификации хранятся в директории `specs/`
4. Для получения более подробной информации см. `docs/architecture/specification-driven-development.md`

## Устранение неполадок

### Проблемы с запуском агентов

Если агенты не запускаются, проверьте:

1. Корректность YAML frontmatter в файлах агентов
2. Наличие всех необходимых инструментов в системе
3. Правильность синтаксиса в файле задачи

### Проблемы с доступом к файлам

Убедитесь, что:

1. У вас есть права на чтение/запись в директории проекта
2. Все пути к файлам указаны корректно
3. Файлы агентов находятся в директории `.qwen/agents/`

## Инструменты оркестрации

### Скрипты анализа проекта

В проекте доступны скрипты для анализа состояния проекта:

- `scripts/orchestration-tools/analyze-project-state.sh` - определение состояния проекта (пустой, с кодом, с частичными/полными спецификациями)
- `scripts/orchestration-tools/orchestrate-project.sh` - адаптивное управление проектом на основе его состояния

Эти инструменты позволяют оркестратору адаптировать свое поведение в зависимости от текущего состояния проекта.
