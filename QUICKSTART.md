# Быстрый старт с Qwen Code Orchestrator Kit

Данный документ поможет вам быстро начать работу с Qwen Code Orchestrator Kit.

## Подготовка к работе

### 1. Установка необходимых инструментов

Убедитесь, что у вас установлены:

- Git
- Python 3.11+
- Qwen Orchestrator Kit

### 2. Клонирование репозитория

```bash
git clone https://github.com/ваш-репозиторий/qwen_orc_kit_ru.git
cd qwen_orc_kit_ru
```

## Запуск первого задания

### 1. Проверка доступных агентов

Проверьте, что все базовые агенты установлены:

```bash
ls .qwen/agents/
```

Вы должны увидеть:
- `bug-fixer.md`
- `bug-hunter.md`
- `code-quality-checker.md`
- `security-orchestrator.md`
- `tech-translator-ru.md`

### 2. Запуск простого задания

Давайте запустим `bug-hunter` для анализа кода:

```bash
task '{
  "subagent_type": "bug-hunter",
  "description": "Анализ кода на наличие потенциальных проблем",
  "prompt": "Проанализируй файлы в директории .specify/templates/ на предмет потенциальных багов и уязвимостей."
}'
```

### 3. Проверка результата

После выполнения задания проверьте файлы в директории `state/`:

```bash
ls -la state/
```

## Работа с агентами

### Bug Hunter (охотник за багами)

Используется для поиска потенциальных ошибок и уязвимостей:

```bash
task '{
  "subagent_type": "bug-hunter",
  "description": "Поиск уязвимостей безопасности",
  "prompt": "Проанализируй файлы в src/ на предмет возможных уязвимостей безопасности."
}'
```

### Code Quality Checker (проверка качества кода)

Используется для проверки соответствия кода стандартам:

```bash
task '{
  "subagent_type": "code-quality-checker",
  "description": "Проверка качества кода",
  "prompt": "Проверь файлы в директории .qwen/.qwen/docs/architecture/ на соответствие стилю документации и лучшим практикам."
}'
```

### Bug Fixer (исправление ошибок)

Используется для исправления найденных ошибок:

```bash
task '{
  "subagent_type": "bug-fixer",
  "description": "Исправление ошибки в коде",
  "prompt": "Исправь найденную уязвимость в файле example.py, связанную с неправильной обработкой пользовательского ввода."
}'
```

## Работа с контрольными точками качества

Система включает в себя несколько контрольных точек качества (quality gates):

### 1. Pre-Execution Gate

Проверяет корректность задачи перед выполнением:

```bash
./.qwen/scripts/run_quality_gate.sh 1
```

### 2. Post-Execution Gate

Проверяет результат выполнения задачи:

```bash
./.qwen/scripts/run_quality_gate.sh 2
```

### 3. Pre-Commit Gate

Проверяет код перед коммитом:

```bash
./.qwen/scripts/run_quality_gate.sh 3
```

## Создание спецификации

Для создания спецификации функциональности используйте шаблон:

```bash
# Создайте новый файл спецификации на основе шаблона
cp .specify/templates/spec-template.md specs/new_feature_spec.md
```

Затем заполните спецификацию в соответствии с шаблоном.

## Пример полного цикла разработки

1. **Создание спецификации**:
   ```bash
   cp .specify/templates/spec-template.md specs/my_new_feature.md
   # Заполните спецификацию
   ```

2. **Анализ кода**:
   ```bash
   task '{
     "subagent_type": "bug-hunter",
     "description": "Анализ существующего кода",
     "prompt": "Проанализируй текущую реализацию в src/ перед добавлением новой функциональности."
   }'
   ```

3. **Проверка качества**:
   ```bash
   task '{
     "subagent_type": "code-quality-checker",
     "description": "Проверка качества архитектуры",
     "prompt": "Проверь архитектурные решения в спецификации specs/my_new_feature.md на соответствие лучшим практикам."
   }'
   ```

4. **Реализация**:
   - Реализуйте функциональность согласно спецификации

5. **Проверка после реализации**:
   ```bash
   ./.qwen/scripts/run_quality_gate.sh 2
   ```

6. **Коммит изменений**:
   ```bash
   ./.qwen/scripts/run_quality_gate.sh 3
   git add .
   git commit -m "feat: добавить новую функциональность согласно specs/my_new_feature.md"
   ```

## Полезные команды

- `ls .qwen/agents/` - просмотр доступных агентов
- `ls .qwen/commands/` - просмотр доступных команд
- `./.qwen/scripts/run_quality_gate.sh [1|2|3]` - запуск контрольной точки качества
- `tree specs/` - просмотр структуры спецификаций (если установлена утилита tree)

## Следующие шаги

После освоения базового функционала ознакомьтесь с:

- Подробной документацией в `.qwen/.qwen/docs/architecture/`
- Примерами использования в директории `examples/` (если существует)
- Руководством по созданию собственных агентов

## Использование экспериментальных навыков агента

В проекте также доступна экспериментальная функция навыков агента. Для её включения:

1. Убедитесь, что файл `.qwen/settings.json` содержит настройки для экспериментальных навыков:

```json
{
  "tools": {
    "experimental": {
      "skills": true
    }
  }
}
```

2. Запускайте Qwen Code с флагом `--experimental-skills`:

```bash
qwen --experimental-skills
```

Навыки позволяют расширять возможности агентов за счет модульных возможностей, которые упаковывают экспертизу в обнаруживаемые возможности.
