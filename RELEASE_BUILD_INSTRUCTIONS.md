# Инструкции по сборке релизной версии проекта

## Обзор

Этот документ описывает процесс подготовки релизной версии проекта Qwen Code Orchestrator Kit. Релизная версия содержит только те файлы, которые необходимы для работы системы в продакшене, исключая все файлы, относящиеся к разработке.

## Подготовка к релизу

### 1. Проверка целостности проекта

Перед началом процесса подготовки релиза убедитесь, что:

- Все изменения закоммичены в репозиторий
- Все тесты проходят успешно
- Все зависимости обновлены до актуальных версий
- Документация актуализирована

### 2. Определение файлов для исключения

Следующие категории файлов и директорий будут исключены из релизной версии:

#### Файлы разработки:
- `.git/` - директория системы контроля версий
- `specs/` - спецификации функций
- `state/` - состояние системы и артефакты выполнения задач
- `examples/specs/` - примеры спецификаций
- `FEATURE_DIR/` - директория для разработки новых функций
- `workspace/` - рабочая область для разработки
- `backups/` - резервные копии
- `logs/` - логи выполнения
- `.tmp/`, `tmp/` - временные файлы
- `*.tmp`, `*.temp`, `*.draft` - временные файлы

#### Файлы тестирования и отладки:
- `tests/` - тесты
- `test/` - тесты
- `coverage/` - результаты покрытия кода
- `*.log` - лог-файлы
- `reports/` - отчеты анализа
- `*.report.md` - файлы отчетов
- `analysis-*.md` - файлы анализа

#### Временные файлы и кэши:
- `node_modules/` - зависимости Node.js
- `.venv/`, `venv/`, `env/` - виртуальные окружения Python
- `__pycache__/`, `*.pyc` - кэши Python
- `.DS_Store`, `Thumbs.db` - системные файлы ОС
- `*.bak`, `*.backup` - резервные копии файлов

#### Файлы IDE и редакторов:
- `.vscode/` - настройки VS Code
- `.idea/` - настройки IntelliJ IDEA
- `*.swp`, `*.swo` - файлы swap Vim

#### Файлы конфигурации разработки:
- `.qwen/commands/*` - команды для разработки (оставляем .qwen/agents/, .qwen/skills/, .qwen/templates/)
- `.specify/memory/*` - память для агентов
- `.mcp.*.json` - все файлы конфигурации MCP (кроме .mcp.json)

### 3. Файлы, которые должны остаться в релизе

Следующие файлы и директории являются необходимыми для работы системы в продакшене:

#### Основные компоненты:
- `.qwen/agents/` - агенты системы
- `.qwen/skills/` - навыки агентов
- `.qwen/templates/` - шаблоны для создания новых компонентов
- `.qwen/config/` - конфигурации для MCP и других сервисов

#### Документация:
- `.qwen/docs/architecture/` - архитектурная документация
- `docs/help/` - справочная документация
- `docs/next_step/` - документация для следующих шагов

#### Скрипты:
- `.qwen/scripts/orchestration-tools/` - скрипты оркестрации
- `.qwen/scripts/specification-tools/` - скрипты для работы со спецификациями
- `.qwen/scripts/release-tools/` - скрипты для подготовки релиза
- `.qwen/scripts/quality-gates/` - скрипты контрольных точек качества

#### Основные файлы проекта:
- `QWEN.md` - основной файл с поведенческой парадигмой
- `README.md` - файл с описанием проекта
- `CONTRIBUTING.md` - инструкции по внесению вклада
- `INSTALLATION.md` - инструкции по установке
- `LICENSE` - лицензия проекта
- `.gitignore` - обновленный файл игнорирования
- `release-config.toml` - конфигурация релиза

## Процесс подготовки релиза

### 1. Использование скрипта очистки

Для автоматической подготовки релизной версии используйте скрипт:

```bash
bash .qwen/scripts/release-tools/clean-for-template.sh
```

Этот скрипт:
- Удаляет все файлы, специфичные для текущего проекта
- Заменяет специфичные имена проекта на плейсхолдеры
- Создает обобщенный .gitignore
- Подготавливает проект к использованию в качестве шаблона

### 2. Ручная проверка

После выполнения скрипта выполните ручную проверку:

1. Убедитесь, что все необходимые файлы для работы системы в продакшене остались
2. Проверьте, что все файлы разработки удалены
3. Убедитесь, что документация актуальна
4. Проверьте, что все зависимости указаны корректно

### 3. Создание артефакта релиза

Для создания артефакта релиза выполните:

```bash
bash .qwen/scripts/release-tools/create-release-artifact.sh <версия-релиза>
```

Например:
```bash
bash .qwen/scripts/release-tools/create-release-artifact.sh 1.0.0
```

Этот скрипт создаст архив в директории `releases/` с именем `qwen-orc-kit-template-<версия>.tar.gz`.

## Проверка релизной версии

### 1. Проверка целостности архива

Убедитесь, что архив можно распаковать без ошибок:

```bash
tar -tzf releases/qwen-orc-kit-template-<версия>.tar.gz
```

### 2. Тестирование шаблона

Распакуйте архив и протестируйте его использование как шаблона для нового проекта:

```bash
mkdir test-release
cd test-release
tar -xzf ../releases/qwen-orc-kit-template-<версия>.tar.gz
# Проверьте содержимое и попробуйте запустить базовую функциональность
```

### 3. Проверка документации

Убедитесь, что вся документация в архиве актуальна и полезна для пользователей, создающих новые проекты на основе шаблона.

## Заключение

После выполнения всех шагов вы получите релизную версию проекта Qwen Code Orchestrator Kit, содержащую только те файлы, которые необходимы для работы системы в продакшене. Все файлы разработки, временные файлы и специфичные для проекта артефакты будут исключены из релиза.
