# Использование шаблона Qwen Code Orchestrator Kit

Данный архив содержит шаблон проекта Qwen Code Orchestrator Kit с полной поддержкой адаптивного оркестратора и процесса Speckit.

## Структура шаблона

```
{{PROJECT_TEMPLATE_NAME}}/
├── .specify/                 # Шаблоны и скрипты для спецификаций
│   ├── memory/              # Постоянная память для агентов
│   ├── .qwen/scripts/             # Bash-скрипты для автоматизации
│   └── templates/           # Шаблоны для различных документов
├── .qwen/                   # Конфигурация Qwen CLI
│   ├── agents/              # Агенты с ограниченными привилегиями
│   └── commands/            # Команды
├── .qwen/docs/                    # Документация проекта
├── specs/                   # Спецификации функциональности
├── .qwen/scripts/                 # Скрипты для контрольных точек качества
├── state/                   # Состояние системы и артефакты выполнения задач
├── src/                     # Исходный код проекта (пустая директория)
├── tests/                   # Тесты проекта (шаблоны)
├── README.md                # Основное руководство
├── CONTRIBUTING.md          # Руководство по участию в проекте
├── INSTALLATION.md          # Инструкция по установке
└── QUICKSTART.md            # Быстрый старт
```

## Начало работы с шаблоном

### 1. Распаковка шаблона

```bash
tar -xzf qwen-orc-kit-template-1.0.0.tar.gz
cd qwen-orc-kit-template-1.0.0
mv qwen-orc-kit-template-1.0.0 your-project-name
cd your-project-name
```

### 2. Инициализация Git

```bash
git init
git add .
git commit -m "feat: initial commit from Qwen Code Orchestrator Kit template"
```

### 3. Адаптация к вашему проекту

1. Обновите `README.md` с информацией о вашем проекте
2. Обновите `.specify/memory/constitution.md` с принципами вашего проекта
3. Настройте файлы конфигурации под ваш стек технологий

## Адаптивное поведение оркестратора

Шаблон включает адаптивный оркестратор, который автоматически определяет состояние проекта:

### Пустой проект
Если проект только начинается, оркестратор:
- Предложит создать конституцию проекта
- Поможет определить основные принципы
- Предложит создать первую спецификацию

### Существующий код без спецификаций
Если в проекте есть код, но нет спецификаций, оркестратор:
- Проведет анализ существующего кода
- Предложит реверс-инжиниринг спецификаций
- Поможет интегрировать процесс Speckit

### Частичные спецификации
Если проект имеет частичные спецификации, оркестратор:
- Поможет доработать и заполнить пробелы
- Проверит соответствие реализации спецификациям
- Предложит пути улучшения

### Полные спецификации
Если проект имеет полные спецификации, оркестратор:
- Следует стандартному процессу Speckit
- Проверяет соответствие реализации спецификациям
- Использует контрольные точки качества

## Основные команды Speckit

- `speckit.constitution` - создание конституции проекта
- `speckit.specify` - создание спецификации функции
- `speckit.clarify` - уточнение спецификации
- `speckit.plan` - создание плана реализации
- `speckit.tasks` - генерация задач для реализации
- `speckit.implement` - выполнение задач реализации
- `speckit.validate` - проверка соответствия реализации спецификациям

## Безопасность и контрольные точки

Шаблон включает 5 контрольных точек качества:
1. Pre-Execution Gate: Проверка корректности задачи
2. Post-Execution Gate: Верификация результата под-агента
3. Pre-Commit Gate: Проверки перед коммитом
4. Pre-Merge Gate: Проверки перед вливанием
5. Pre-Implementation Gate: Проверка качества спецификаций перед реализацией

## Поддержка TDD

Шаблон включает инструменты для поддержки разработки через тестирование:
- `.qwen/scripts/specification-tools/generate-tests-from-spec.sh` - генерация тестов из спецификаций
- `.qwen/scripts/specification-tools/assign-agents-to-tasks.sh` - автоматическое назначение агентов задачам

## Интеграция с системой агентов

Шаблон включает агентов с принципом минимальных привилегий:
- `bug-fixer` - исправление ошибок
- `bug-hunter` - поиск ошибок
- `code-quality-checker` - проверка качества кода
- `security-orchestrator` - обеспечение безопасности
- `tech-translator-ru` - перевод документации
- `specification-analyst` - анализ спецификаций
- `specification-compliance-checker` - проверка соответствия реализации

Для каждого агента предоставлены только минимально необходимые инструменты для выполнения задачи.
