# Инструменты автоматизации

## Обзор

Инструменты автоматизации в Qwen Orchestrator Kit предназначены для:
- Автоматизации проверок качества
- Обеспечения соответствия стандартам
- Упрощения процесса разработки
- Снижения количества ручных операций

## Контрольные точки качества

### Типы контрольных точек

#### Блокирующие (Blocking)
- Останавливают выполнение при неудачном результате
- Примеры: проверки безопасности, проверки типизации

#### Неблокирующие (Non-blocking)
- Генерируют предупреждения при неудачном результате
- Примеры: проверки стиля кода, проверки покрытия тестами

### Категории контрольных точек

1. **Проверки безопасности** (`scripts/quality-gates/check-security.sh`) - блокирующие
2. **Проверки покрытия тестами** (`scripts/quality-gates/check-coverage.sh`) - неблокирующие
3. **Проверки размера бандла** (`scripts/quality-gates/check-bundle-size.sh`) - неблокирующие
4. **Проверки типизации** (`scripts/quality-gates/check-typing.sh`) - блокирующие
5. **Проверки линтинга** (`scripts/quality-gates/check-linting.sh`) - неблокирующие

### Навык выполнения контрольных точек

Используйте навык `run-quality-gate` для выполнения проверок качества:

```json
{
  "command": "scripts/quality-gates/check-security.sh",
  "isBlocking": true,
  "expectedResult": "No critical vulnerabilities found",
  "timeout": 300
}
```

## Автоматические проверки

### При создании агента
- Проверка YAML заголовка
- Проверка структуры агента
- Проверка соответствия шаблону
- Проверка именования

### При обновлении агента
- Проверка обратной совместимости
- Проверка соответствия стандартам
- Проверка влияния на другие компоненты

### При коммите
- Проверка формата файлов
- Запуск тестов
- Проверка типизации
- Проверка линтером

## Скрипты автоматизации

### Скрипты проверки качества
- `scripts/quality-gates/check-security.sh` - проверка безопасности
- `scripts/quality-gates/check-coverage.sh` - проверка покрытия тестами
- `scripts/quality-gates/check-bundle-size.sh` - проверка размера бандла
- `scripts/quality-gates/check-typing.sh` - проверка типизации
- `scripts/quality-gates/check-linting.sh` - проверка линтером

### Скрипты проверки соответствия
- `scripts/validation/check-yaml-headers.sh` - проверка YAML заголовков
- `scripts/validation/check-agent-structure.sh` - проверка структуры агентов
- `scripts/validation/check-naming-convention.sh` - проверка именования

### Скрипты обновления
- `scripts/maintenance/update-templates.sh` - обновление шаблонов
- `scripts/maintenance/standardize-agents.sh` - стандартизация агентов
- `scripts/maintenance/fix-yaml-headers.sh` - исправление YAML заголовков

## Интеграция с CI/CD

### В процессе сборки
- Запуск всех блокирующих проверок
- Проверка безопасности
- Проверка типизации
- Проверка линтером

### В процессе тестирования
- Проверка покрытия тестами
- Проверка производительности
- Проверка функциональности

### В процессе деплоя
- Финальная проверка качества
- Проверка соответствия спецификациям
- Проверка документации

## Автоматизация с помощью агентов

### Агент-специалист по Qwen Orchestrator Kit
- Отслеживает актуальность документации
- Проверяет соответствие стандартам
- Обновляет устаревшие компоненты
- Генерирует отчеты о несоответствиях

### Оркестраторы проверок
- Запускают комплексные проверки
- Интегрируют результаты из разных источников
- Генерируют сводные отчеты
- Управляют процессом исправления ошибок

## Настройка автоматизации

### Конфигурация проверок
- `.qwen/config/quality-gates.json` - настройки контрольных точек
- `.qwen/config/validation-rules.json` - правила валидации
- `.qwen/config/automation-rules.json` - правила автоматизации

### Планирование проверок
- Проверки при каждом коммите
- Проверки при создании MR
- Периодические проверки соответствия
- Проверки при релизе

## Мониторинг и отчетность

### Метрики качества
- Количество ошибок в агентах
- Время на исправление ошибок
- Покрытие тестами
- Соответствие стандартам
- Частота рефакторинга

### Отчеты
- Ежедневные отчеты о качестве
- Отчеты о несоответствиях
- Отчеты о производительности
- Отчеты о безопасности

## Лучшие практики

### Для настройки:
- Настройте все блокирующие проверки
- Обеспечьте быстрое выполнение проверок
- Настройте уведомления о критических ошибках
- Регулярно обновляйте правила проверок

### Для поддержки:
- Регулярно анализируйте результаты проверок
- Обновляйте стандарты на основе обратной связи
- Обучайте команду использованию автоматизации
- Поддерживайте актуальность инструментов
