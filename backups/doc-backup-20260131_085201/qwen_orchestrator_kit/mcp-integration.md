# Интеграция с Model Context Protocol (MCP) серверами

## Обзор MCP

Model Context Protocol (MCP) - это протокол для интеграции внешних сервисов и инструментов с агентами Qwen Orchestrator Kit. MCP позволяет агентам использовать внешние ресурсы, такие как документация библиотек, базы знаний, специализированные инструменты и т.д.

## Доступные MCP серверы

### Context7 (`mcp__context7__*`)
- **Назначение**: Доступ к документации библиотек и фреймворков
- **Использование**: Проверка актуальных паттернов и API
- **Триггер**: Перед реализацией кода с внешними библиотеками
- **Ключевые инструменты**: `mcp__context7__resolve-library-id`, `mcp__context7__get-library-docs`

### Supabase (`mcp__supabase__*`)
- **Назначение**: Работа с базами данных Supabase
- **Использование**: Операции с базами данных и миграции
- **Триггер**: При работе с базами данных

### Sequential Thinking (`mcp__sequential_thinking__*`)
- **Назначение**: Помощь в сложных рассуждениях
- **Использование**: Анализ архитектурных решений
- **Триггер**: При решении сложных многоступенчатых задач

### Playwright (`mcp__playwright__*`)
- **Назначение**: Автоматизация браузерных тестов
- **Использование**: Тестирование UI компонентов
- **Триггер**: При тестировании пользовательских сценариев

### shadcn (`mcp__shadcn__*`)
- **Назначение**: Работа с компонентами shadcn
- **Использование**: При реализации UI компонентов
- **Требует**: Полной конфигурации (`.mcp.full.json`)

### n8n (`mcp__n8n_mcp__*`)
- **Назначение**: Управление рабочими процессами
- **Использование**: При автоматизации процессов
- **Триггер**: При работе с n8n интеграциями

## Использование MCP в агентах

### Рекомендации по использованию

Используйте навык `select-mcp-server` для определения подходящих MCP серверов:

```json
{
  "taskType": "bug-fixing",
  "domain": "react",
  "requirements": ["documentation", "pattern-validation"]
}
```

### Включение рекомендаций в планы задач

Включайте рекомендации MCP в планы задач:

```json
{
  "phase": 2,
  "config": {
    "priority": "critical",
    "scope": ["src/", "lib/"]
  },
  "validation": {
    "required": ["type-check", "build"],
    "optional": ["tests"]
  },
  "mcpGuidance": {
    "recommended": ["mcp__context7__*"],
    "library": "react",
    "reason": "Check current React patterns before implementing fixes"
  },
  "nextAgent": "work_react_specialist"
}
```

## Практические советы

### При использовании MCP:
- Используйте только при необходимости
- Проверяйте доступность MCP серверов перед использованием
- Учитывайте время ответа MCP серверов
- Обрабатывайте ошибки при недоступности MCP серверов
- Следуйте рекомендациям по использованию для каждого типа задач

### Интеграция в планы:
- Включайте рекомендации MCP в файлы плана
- Указывайте конкретные библиотеки или технологии
- Объясняйте причину использования конкретного сервера
- Учитывайте возможность отключения MCP в окружении

## Конфигурация MCP

### Файлы конфигурации:
- `.mcp.json` - базовая конфигурация
- `.mcp.full.json` - полная конфигурация с расширенными возможностями
- `.mcp.database.json` - конфигурация для баз данных
- `.mcp.frontend.json` - конфигурация для фронтенда

### Проверка конфигурации:
Используйте команду для проверки конфигурации MCP:
```
mcp check-config
```

## Лучшие практики

### Для разработчиков:
- Документируйте использование MCP в агентах
- Обеспечьте резервные пути при недоступности MCP
- Тестируйте агентов как с MCP, так и без него
- Обновляйте рекомендации MCP при изменении требований

### Для архитекторов:
- Определяйте, какие функции должны использовать MCP
- Создавайте политики использования MCP
- Обеспечьте безопасность при использовании внешних сервисов
- Мониторьте использование MCP в системе

## Отладка MCP

### При возникновении проблем:
- Проверьте конфигурацию MCP
- Убедитесь в доступности серверов
- Проверьте формат запросов к MCP
- Используйте логирование для отладки

### Инструменты диагностики:
- `mcp status` - проверка состояния подключений
- `mcp logs` - просмотр логов взаимодействия
- `mcp test {server}` - тестирование конкретного сервера
