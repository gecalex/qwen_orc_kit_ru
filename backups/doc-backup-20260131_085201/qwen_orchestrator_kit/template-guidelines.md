# Рекомендации по созданию шаблонов

## Цель шаблонов

Шаблоны в Qwen Orchestrator Kit предназначены для:
- Ускорения создания новых агентов и навыков
- Обеспечения единообразия в структуре компонентов
- Снижения порога входа для новых разработчиков
- Поддержания соответствия стандартам

## Структура шаблонов

Шаблоны находятся в директории `.qwen/templates/` и включают:
- `orchestrator-template.md` - шаблон для оркестраторов
- `worker-template.md` - шаблон для воркеров
- `skill-template.md` - шаблон для навыков
- `report-template.md` - шаблон для форматирования отчетов

## Шаблон оркестратора

Оркестратор должен включать следующие разделы:

1. **YAML заголовок** - с обязательными полями
2. **Назначение** - описание роли и ответственности
3. **Использование сервера MCP** - рекомендации по использованию внешних сервисов
4. **Инструкции** - пошаговое описание логики работы
5. **Рекомендации MCP в файлах плана** - пример структуры плана
6. **Шаблон возврата управления** - описание процесса завершения работы
7. **Стандартизированная отчетность** - формат отчетов
8. **Интеграция навыков** - список используемых навыков

## Шаблон воркера

Воркер должен включать следующие разделы:

1. **YAML заголовок** - с обязательными полями
2. **Назначение** - описание роли и ответственности
3. **Использование сервера MCP** - рекомендации по использованию внешних сервисов
4. **Инструкции** - пошаговое описание логики работы (обычно 5 фаз)
5. **Формат файла плана** - пример структуры плана
6. **Шаблон возврата управления** - описание процесса завершения работы
7. **Стандартизированная отчетность** - формат отчетов
8. **Интеграция навыков** - список используемых навыков

## Шаблон навыка

Навык должен включать следующие разделы:

1. **YAML заголовок** - с полями `name` и `description`
2. **Когда использовать** - сценарии применения
3. **Инструкции** - пошаговое описание логики
4. **Формат ввода** - структура входных данных
5. **Формат вывода** - структура выходных данных
6. **Примеры** - практические примеры использования

## Переменные в шаблонах

Используйте следующие переменные в шаблонах:
- `{domain}` - домен агента (например, `dev`, `backend`, `frontend`)
- `{feature_name}` - имя функции или задачи
- `{workflow}` - имя рабочего процесса
- `{current-phase}` - текущая фаза выполнения

## Параметры шаблонов

Шаблоны должны быть гибкими и легко адаптируемыми:
- Используйте комментарии для пояснения назначения разделов
- Обеспечьте возможность легкой замены переменных
- Включите примеры, которые легко модифицировать
- Документируйте необязательные разделы

## Проверка шаблонов

Перед использованием шаблона убедитесь, что:
- Все обязательные поля присутствуют
- Структура соответствует стандартам
- Примеры актуальны и работоспособны
- Описания ясны и понятны

## Обновление шаблонов

Шаблоны должны регулярно обновляться:
- При изменении стандартов Qwen Orchestrator Kit
- При появлении новых возможностей
- При получении обратной связи от разработчиков
- При выявлении частых ошибок в создаваемых компонентах

## Автоматизация

Для автоматизации создания компонентов на основе шаблонов:
- Используйте навык `template-agent` для генерации новых агентов
- Интегрируйте проверки соответствия шаблону в процесс разработки
- Создайте скрипты для массового обновления компонентов
- Обеспечьте уведомления о несоответствии шаблону
