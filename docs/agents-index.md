# Индекс агентов Qwen Orchestrator Kit

## Обзор

Этот файл содержит индекс всех агентов в системе Qwen Orchestrator Kit, сгруппированных по функциональности и доменам. Он предназначен для улучшения навигации и поиска подходящих агентов для выполнения задач.

## Структура индекса

Агенты в этом индексе сгруппированы по следующим категориям:
- Оркестраторы (Orchestrators) - координируют выполнение задач между другими агентами
- Воркеры (Workers) - выполняют конкретные задачи
- Специализированные агенты - выполняют специфические функции

Каждый агент в индексе имеет:
- Имя файла
- Тип (оркестратор/воркер)
- Домен (dev, frontend, backend, testing, research, security)
- Краткое описание функций
- Основные инструменты, к которым имеет доступ

## Оркестраторы

### Оркестраторы Dev (Разработка)
- **orc_dev_task_coordinator.md** - Оркестратор для координации задач разработки в области dev. Реализует итерационную логику и шаблон возврата управления. Следует стандартизированному формату отчетности.
- **orc_planning_task_analyzer.md** - Оркестратор для координации автоматизированных рабочих процессов фазы планирования. Анализирует задачи, определяет требуемые типы агентов, назначает исполнителей и создает планы выполнения.

### Оркестраторы Backend (Бэкенд)
- **orc_backend_api_coordinator.md** - Оркестратор для координации задач разработки API в области бэкенда. Реализует итерационную логику и шаблон возврата управления.

### Оркестраторы Frontend (Фронтенд)
- **orc_frontend_ui_coordinator.md** - Оркестратор для координации задач UI/UX в области фронтенда. Реализует итерационную логику и шаблон возврата управления.

### Оркестраторы Testing (Тестирование)
- **orc_testing_quality_assurer.md** - Оркестратор для координации задач обеспечения качества в области тестирования. Реализует итерационную логику и шаблон возврата управления.

### Оркестраторы Research (Исследования)
- **orc_research_data_analyzer.md** - Оркестратор для координации задач анализа данных в области исследований. Реализует итерационную логику и шаблон возврата управления.

### Оркестраторы Security (Безопасность)
- **orc_security_security_orchestrator.md** - Оркестратор для координации многофазных рабочих процессов безопасности с итеративной логику и шаблоном возврата управления.

## Воркеры

### Воркеры Dev (Разработка)
- **work_dev_code_analyzer.md** - Воркер для выполнения задач анализа кода из файлов плана в области разработки.
- **work_dev_meta_agent.md** - Воркер для автоматического создания новых агентов (оркестраторов и воркеров) на основе заданных параметров.
- **work_dev_test-agent.md** - Воркер для выполнения задач из файлов плана в области dev.

### Воркеры Backend (Бэкенд)
- **work_backend_api_validator.md** - Воркер для проверки реализаций API из файлов плана в области бэкенда.

### Воркеры Frontend (Фронтенд)
- **work_frontend_component_generator.md** - Воркер для генерации компонентов интерфейса из файлов плана.

### Воркеры Testing (Тестирование)
- **work_testing_test_generator.md** - Воркер для генерации тестовых случаев из файлов плана в области тестирования.

### Воркеры Planning (Планирование)
- **work_planning_task_classifier.md** - Воркер для анализа и классификации задач из файлов плана в области планирования.
- **work_planning_agent_requirer.md** - Воркер для определения требуемых типов агентов из файлов плана в области планирования.
- **work_planning_executor_assigner.md** - Воркер для назначения исполнителей задачам из файлов плана в области планирования.

### Воркеры Research (Исследования)
- **work_research_trend_tracker.md** - Воркер для отслеживания тенденций и анализа данных из файлов плана в области исследований.

## Специализированные агенты

- **bug-orchestrator.md** - Автономный оркестратор L1 для автоматизированного обнаружения и исправления ошибок.
- **security-orchestrator.md** - Автономный оркестратор L1 для автоматизированного обнаружения и исправления уязвимостей.
- **dependency-analyzer.md** - Агент для анализа зависимостей проекта и выявления проблем.
- **dead-code-detector.md** - Агент для обнаружения мертвого кода и неиспользуемых компонентов.
- **bug-fixer.md** - Агент для исправления ошибок в коде на основе отчетов и анализа.
- **bug-hunter.md** - Агент для поиска ошибок, уязвимостей и проблем в коде.
- **security-analyzer.md** - Агент для проверки безопасности кода на наличие уязвимостей.
- **code-quality-checker.md** - Агент для проверки качества кода, стиля, безопасности и соответствия стандартам.
- **tech-translator-ru.md** - Агент для перевода технической документации, комментариев и сообщений с английского на русский.
- **qwen-code-cli-specialist.md** - Агент-специалист по Qwen Code CLI, отвечает за создание и поддержание документации, шаблонов, спецификаций и автоматизацию контроля соответствия официальному руководству Qwen Code CLI.

## Поиск агентов

Для поиска агента по функциональности используйте следующие критерии:
1. Определите тип задачи (анализ, генерация, валидация, координация и т.д.)
2. Определите домен задачи (dev, frontend, backend, testing, research, security)
3. Выберите соответствующий тип агента (оркестратор для координации, воркер для выполнения конкретных задач)
4. Ознакомьтесь с описанием агента в соответствующем файле

## Обновление индекса

Этот индекс автоматически обновляется при добавлении новых агентов в систему. Для ручного обновления используйте скрипт `scripts/monitoring/update-agents-index.sh`.
