# Стандартизированный формат отчетов агентов

## Обзор

Этот документ описывает унифицированный формат отчетов, которые должны генерировать все агенты в системе Qwen Code CLI. Стандартизация отчетов обеспечивает согласованность, упрощает автоматическую обработку результатов и улучшает взаимодействие между агентами и оркестраторами.

## Назначение

- Обеспечение согласованности формата отчетов от всех агентов
- Упрощение автоматической обработки и анализа результатов
- Повышение прозрачности и отслеживаемости выполнения задач
- Облегчение интеграции с другими компонентами системы

## Структура стандартизированного отчета

Каждый отчет агента должен содержать следующие обязательные разделы:

### 1. Заголовок

```markdown
# {ТипОтчета} Report: {Версия}

**Status**: ✅ PASSED | ⚠️ PARTIAL | ❌ FAILED
**Duration**: {время выполнения}
**Agent**: {имя-агента}
**Phase**: {номер-фазы}

---
```

### 2. Исполнительное резюме

Краткое изложение проделанной работы и ключевых результатов.

### 3. Выполненная работа

Перечень задач с их статусами (Complete/Failed/Partial).

### 4. Внесенные изменения

Список файлов, которые были изменены, созданы или удалены, с указанием количества.

### 5. Результаты валидации

Результаты проверок качества с деталями и статусами (PASSED/FAILED).

### 6. Метрики

Количественные показатели выполнения: продолжительность, количество задач, изменений, проверок и т.д.

### 7. Обнаруженные ошибки

Описание всех ошибок с контекстом и описанием.

### 8. Следующие шаги

Рекомендации по дальнейшим действиям и шаги восстановления при ошибках.

### 9. Артефакты

Перечень созданных артефактов: файлы плана, отчеты, дополнительные файлы.

## Пример полного отчета

```markdown
# Bug Fix Report: v1.0.0

**Status**: ✅ PASSED
**Duration**: 00:15:30
**Agent**: work_health_bug_fixer
**Phase**: 2

---

## Executive Summary
Исправлены 3 критические ошибки, включая null pointer exception и логическую ошибку в калькуляторе скидок.

## Work Performed
- Исправление null pointer exception в src/auth/service.py: Complete
- Исправление логической ошибки в src/pricing/calculator.py: Complete
- Обновление документации: Partial

## Changes Made
- Файлы изменены: src/auth/service.py, src/pricing/calculator.py, docs/errors.md (3 файла)
- Файлы созданы: (0 файлов)
- Файлы удалены: (0 файлов)

## Validation Results
- Типизация: PASSED
- Сборка: PASSED
- Тесты: PASSED
- Детали: Все проверки пройдены успешно

## Metrics
- Продолжительность: 00:15:30
- Задач выполнено: 3
- Изменений внесено: 15
- Проверок выполнено: 3

## Errors Encountered
- Ошибка 1: Временная ошибка подключения к базе данных (устранена)
- Ошибка 2: Нет

## Next Steps
- Для оркестратора: Запустить тестирование исправлений
- Шаги восстановления при ошибке: Откатить изменения в src/auth/service.py

## Artifacts
- Файл плана: .tmp/current/plans/bug-fix-plan.json
- Отчет: .tmp/current/reports/bug-fix-report.md
- Дополнительные артефакты: docs/errors.md
```

## Использование навыков

### Генерация заголовков

Для генерации согласованных заголовков отчетов используйте навык `generate-report-header`:

```json
{
  "reportType": "Bug Fix",
  "version": "v1.0.0",
  "status": "PASSED",
  "duration": "00:15:30",
  "agent": "work_health_bug_fixer",
  "phase": 2
}
```

### Проверка отчетов

Для проверки полноты и формата отчетов используйте навык `validate-report-file`:

```json
{
  "reportFilePath": ".tmp/current/reports/bug-fix-report.md",
  "requiredSections": ["Header", "Executive Summary", "Work Performed", "Validation Results"],
  "strictFormat": true
}
```

## Интеграция с агентами

Все агенты должны:

1. Генерировать отчеты в конце выполнения задачи
2. Использовать стандартизированный формат
3. Включать все обязательные разделы
4. Использовать соответствующие навыки для генерации и проверки отчетов
5. Сохранять отчеты в указанное место для последующей обработки

## Обновление агентов

При создании новых агентов или обновлении существующих:

1. Обязательно включайте генерацию стандартизированных отчетов
2. Используйте шаблоны из `.qwen/templates/report-template.md`
3. Интегрируйте навыки `generate-report-header` и `validate-report-file`
4. Следуйте структуре, описанной в этом документе
