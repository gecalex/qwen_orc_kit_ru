# Навыки (Skills) в системе Qwen Code CLI

## Обзор

Навыки (Skills) - это переиспользуемые функции, которые могут быть вызваны агентами для выполнения часто используемых операций. Они представляют собой модульные, специализированные функции, которые обеспечивают согласованность и упрощают разработку агентов.

## Назначение

- Предоставление стандартных функций для часто используемых операций
- Обеспечение согласованности в подходах к обработке данных
- Упрощение разработки агентов за счет переиспользования функций
- Снижение дублирования кода между агентами

## Типы навыков

### 1. Навыки обработки данных
- `parse-error-logs` - парсинг логов ошибок
- `parse-git-status` - парсинг статуса git
- `format-markdown-table` - форматирование таблиц markdown
- `format-todo-list` - форматирование списков задач

### 2. Навыки валидации
- `run-quality-gate` - выполнение контрольных точек качества
- `validate-plan-file` - валидация файлов плана
- `validate-report-file` - валидация файлов отчетов
- `validate-schema` - валидация по JSON схеме

### 3. Навыки форматирования
- `generate-report-header` - генерация заголовков отчетов
- `format-commit-message` - форматирование сообщений коммитов
- `format-markdown-table` - форматирование таблиц markdown
- `render-template` - рендеринг шаблонов

### 4. Навыки анализа
- `calculate-priority-score` - расчет приоритетных баллов
- `parse-package-json` - извлечение метаданных из package.json
- `extract-version` - извлечение версий из строк
- `parse-error-logs` - анализ логов ошибок

## Структура навыка

Каждый навык находится в директории `.qwen/skills/{название-навыка}/SKILL.md` и содержит:

1. **YAML заголовок** с именем, описанием и разрешенными инструментами
2. **Раздел "When to Use"** - описывает, когда использовать навык
3. **Раздел "Instructions"** - инструкции по использованию
4. **Форматы ввода и вывода** - структуры данных для входных и выходных данных
5. **Примеры** - примеры использования с входными и выходными данными

## Создание новых навыков

Для создания нового навыка:

1. Создайте директорию `.qwen/skills/{название-навыка}/`
2. Внутри создайте файл `SKILL.md` с соответствующей структурой
3. Определите форматы ввода и вывода
4. Добавьте примеры использования
5. Укажите разрешенные инструменты (если требуются)

## Использование навыков в агентах

Агенты могут вызывать навыки с помощью инструмента `Skill`:

```
task '{
  "subagent_type": "skill",
  "description": "Parse error logs from build output",
  "prompt": "Execute parse-error-logs skill with parameters: {logType: \\"typescript\\", logContent: \\"...\\", maxErrors: 5}"
}'
```

## Лучшие практики

1. **Малый размер**: Навыки должны быть компактными (<100 строк логики)
2. **Одна задача**: Каждый навык должен решать одну конкретную задачу
3. **Переиспользуемость**: Навыки должны быть применимы в различных контекстах
4. **Ясный интерфейс**: Входные и выходные форматы должны быть четко определены
5. **Документация**: Каждый навык должен быть хорошо документирован с примерами

## Интеграция с агентами

- Воркеры используют навыки для валидации (`run-quality-gate`), генерации отчетов (`generate-report-header`)
- Оркестраторы используют навыки для валидации планов (`validate-plan-file`), отчетов (`validate-report-file`)
- Все агенты могут использовать утилитарные навыки для парсинга, форматирования и вычислений

## Стандартные навыки системы

Следующие навыки являются стандартными для системы:

1. `generate-report-header` - создание заголовков отчетов
2. `validate-report-file` - проверка полноты отчетов
3. `validate-plan-file` - проверка файлов плана
4. `run-quality-gate` - выполнение проверок качества
5. `parse-error-logs` - парсинг логов ошибок
6. `format-markdown-table` - форматирование таблиц
7. `calculate-priority-score` - расчет приоритетов
