# Документация по навигации по агентам Qwen Orchestrator Kit

## Обзор

В проекте Qwen Orchestrator Kit реализована улучшенная система навигации по агентам, которая позволяет эффективно находить и использовать агентов по их функциональности. В связи с ограничениями платформы на добавление новых полей в YAML-заголовки агентов, навигация осуществляется через альтернативные механизмы.

## Способы навигации по агентам

### 1. Индекс агентов

Файл `docs/agents-index.md` содержит автоматически обновляемый список всех агентов с кратким описанием их функций. Индекс обновляется с помощью скрипта `scripts/monitoring/update-agents-index.sh`.

### 2. Категории агентов

Файл `docs/agent-categories.md` группирует агентов по функциональности:
- Агенты разработки
- Агенты тестирования
- Агенты безопасности
- Агенты исследований
- Агенты мониторинга и качества
- Агенты планирования

### 3. Поиск по ключевым словам

Для поиска агентов по функциональности используйте скрипт `scripts/search/agent-search.sh`:

```bash
bash scripts/search/agent-search.sh "security" "authentication" "authorization"
```

Скрипт ищет совпадения в описаниях и функциональности агентов и создает отчет с результатами поиска.

### 4. Анализ агентов

Для получения подробной информации обо всех агентах используйте скрипт `scripts/analysis/agent-analyzer.sh`:

```bash
bash scripts/analysis/agent-analyzer.sh
```

Скрипт создает отчет с информацией о каждом агенте и общей статистикой.

## Структура именования агентов

Агенты следуют стандартной схеме именования: `{типы}_{домен}_{имя}`, где:

- `{типы}` - тип агента: `orc` (оркестратор) или `work` (воркер)
- `{домен}` - область применения: `dev`, `frontend`, `backend`, `testing`, `research`, `security`, `planning`
- `{имя}` - конкретное имя агента в формате kebab-case

Примеры:
- `orc_dev_task_coordinator` - оркестратор задач разработки
- `work_testing_test_generator` - воркер генерации тестов
- `orc_security_security_orchestrator` - оркестратор безопасности

## Домены агентов

### dev (Разработка)
Агенты, занимающиеся общей разработкой, созданием инструментов и архитектурными задачами.

### frontend (Фронтенд)
Агенты, специализирующиеся на задачах фронтенд-разработки.

### backend (Бэкенд)
Агенты, специализирующиеся на задачах бэкенд-разработки.

### testing (Тестирование)
Агенты, отвечающие за тестирование и проверку качества.

### research (Исследования)
Агенты, занимающиеся анализом и исследованием новых возможностей.

### security (Безопасность)
Агенты, специализирующиеся на задачах безопасности и анализе уязвимостей.

### planning (Планирование)
Агенты, участвующие в фазе планирования и анализе задач.

## Рекомендации по использованию

1. **Для поиска агента по функциональности**: Используйте скрипт поиска с ключевыми словами
2. **Для общего обзора**: Обратитесь к индексу агентов
3. **Для понимания категорий**: Ознакомьтесь с файлом категорий агентов
4. **Для анализа**: Запустите скрипт анализа агентов для получения полной информации

## Автоматизация навигации

Система включает автоматические инструменты для поддержания актуальности информации:
- Скрипт обновления индекса агентов
- Скрипт анализа агентов
- Скрипт поиска по ключевым словам

Эти инструменты обеспечивают согласованность и актуальность информации о доступных агентах.
