# Введение в Qwen Orchestrator Kit

## Что такое Qwen Orchestrator Kit?

Qwen Orchestrator Kit - это командная строка, управляемая агентами система для автоматизации задач разработки. Она позволяет создавать, управлять и координировать агентов для выполнения различных задач программирования.

## Основные компоненты

### Агенты
Агенты - это основные исполнительные единицы в системе. Они бывают двух типов:
- **Оркестраторы** (`orc_*`) - координируют выполнение сложных многопроходных рабочих процессов
- **Воркеры** (`work_*`) - выполняют конкретные задачи из файлов плана

### Навыки
Навыки (`skill`) - это переиспользуемые функции, которые агенты могут вызывать для выполнения часто используемых операций.

## Структура проекта

```
.qwen/
├── agents/          # Директория для агентов
│   ├── orc_{domain}_{name}.md    # Оркестраторы
│   └── work_{domain}_{name}.md   # Воркеры
├── skills/          # Директория для навыков
│   └── {skill-name}/
│       └── SKILL.md
├── templates/       # Шаблоны для создания новых компонентов
└── configs/         # Конфигурации для MCP и других сервисов
```

## Создание агента

Для создания нового агента:
1. Выберите подходящий шаблон из `.qwen/templates/`
2. Назовите файл по шаблону: `{типа}_{домена}_{имя}.md`
3. Убедитесь, что YAML заголовок содержит все необходимые поля
4. Реализуйте логику агента в соответствии с шаблоном

## YAML заголовок агента

Каждый агент должен иметь YAML заголовок со следующими полями:

```yaml
---
name: {имя-агента}
description: Краткое описание функций агента
tools:
 - read_file
 - write_file
 - edit
 - run_shell_command
 - grep_search
 - glob
 - web_search
 - web_fetch
 - read_many_files
 - todo_write
 - skill
 - task
color: {цвет}
---
```

## Стандарты именования

- Используйте kebab-case для имен агентов и навыков
- Для оркестраторов используйте префикс `orc_`
- Для воркеров используйте префикс `work_`
- Укажите домен агента в имени (например, `dev`, `backend`, `frontend`, `testing`)
