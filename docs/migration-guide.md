# Руководство по миграции для пользователей Qwen Code CLI

## Обзор изменений

В рамках проекта "Реализация решений на основе анализа различий между Qwen Code CLI и Claude Code Orchestrator Kit" были внедрены значительные улучшения в архитектуру и функциональность системы. Это руководство поможет вам понять изменения и адаптировать ваш рабочий процесс.

## Основные изменения

### 1. Система именования агентов

#### Было:
- Агенты находились в плоской структуре без четкой классификации
- Назначение агента не всегда было очевидно из его имени

#### Стало:
- Внедрена система именования: `orc_{domain}_имя_агента.md` для оркестраторов и `work_{domain}_имя_агента.md` для воркеров
- Примеры:
  - `orc_dev_task_coordinator.md` - оркестратор для координации задач в домене разработки
  - `work_backend_api_validator.md` - воркер для валидации API в бэкенд домене

#### Что нужно изменить:
- Переименуйте существующие агенты в соответствии с новой системой именования
- Обновите ссылки на агентов в ваших скриптах и документации

### 2. Фаза планирования (Phase 0)

#### Новое:
- Внедрена автоматизированная фаза планирования
- Анализ задач и определение требуемых типов агентов
- Назначение исполнителей для всех задач
- Решение исследовательских задач

#### Как использовать:
- При запуске процесса реализации теперь автоматически выполняется фаза планирования
- Система анализирует задачи и создает план выполнения
- При необходимости создаются новые агенты через meta-agent-v3

### 3. Стандартизированные отчеты

#### Было:
- Формат отчетов мог варьироваться от агента к агенту

#### Стало:
- Внедрен унифицированный формат отчетов
- Все агенты генерируют отчеты в едином формате
- Обязательные разделы: заголовок, исполнительное резюме, выполненная работа, изменения, результаты валидации, метрики, ошибки, следующие шаги, артефакты

#### Что нужно изменить:
- Обновите свои агенты для использования стандартизированного формата отчетов
- Используйте навык `generate-report-header` для создания заголовков отчетов

### 4. Контрольные точки качества

#### Новое:
- Внедрена система контрольных точек качества
- Блокирующие и неблокирующие проверки
- Интеграция с оркестраторами
- Навык `run-quality-gate` для выполнения проверок

#### Как использовать:
- Включайте контрольные точки в ваши рабочие процессы
- Используйте навык `run-quality-gate` для выполнения проверок
- Настройте блокирующие проверки для критических аспектов

### 5. Навыки (Skills)

#### Новое:
- Расширенный набор навыков для часто используемых операций
- Навыки для парсинга логов, форматирования данных, валидации и т.д.
- Централизованное хранение в `.qwen/skills/`

#### Как использовать:
- Используйте существующие навыки в ваших агентах
- Создавайте новые навыки для часто используемых операций
- Навыки вызываются через инструмент `Skill`

### 6. Интеграция с MCP

#### Улучшено:
- Расширенная интеграция с MCP серверами
- Рекомендации по использованию MCP в зависимости от типа задачи
- Включение MCP в плановые файлы

#### Как использовать:
- Используйте рекомендации по MCP в ваших агентах
- Включайте MCP в плановые файлы для направления воркеров к подходящим серверам

### 7. Оркестрация

#### Улучшено:
- Поддержка итеративной логики
- Паттерн возвращения управления
- Расширенные возможности оркестраторов

#### Как использовать:
- Обновите свои оркестраторы для использования новых возможностей
- Реализуйте итеративную логику при необходимости
- Используйте паттерн возвращения управления

## Процесс миграции

### Шаг 1: Оценка текущего состояния

1. Проведите инвентаризацию существующих агентов
2. Определите, какие агенты нуждаются в переименовании
3. Оцените, какие агенты нужно обновить для использования новых функций

### Шаг 2: Переименование агентов

1. Переименуйте агенты в соответствии с новой системой именования
2. Обновите все ссылки на агенты
3. Проверьте работоспособность после переименования

### Шаг 3: Обновление агентов

1. Обновите агенты для использования стандартизированных отчетов
2. Интегрируйте навыки в агенты
3. Добавьте использование MCP серверов при необходимости

### Шаг 4: Тестирование

1. Проверьте работу всех обновленных агентов
2. Убедитесь, что отчеты генерируются в правильном формате
3. Проверьте работу контрольных точек качества
4. Протестируйте фазу планирования

### Шаг 5: Обновление документации

1. Обновите документацию в соответствии с новыми стандартами
2. Добавьте информацию о новых компонентах
3. Обновите примеры использования

## Потенциальные проблемы и решения

### Проблема: Конфликты имен после переименования
**Решение**: Используйте постепенный подход с созданием алиасов и поэтапной заменой

### Проблема: Несовместимость старых агентов с новыми стандартами
**Решение**: Создайте адаптеры или постепенно обновляйте агенты

### Проблема: Непонимание новых паттернов
**Решение**: Обратитесь к документации в `docs/architecture/` и примерам в `.qwen/templates/`
