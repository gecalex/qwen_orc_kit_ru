# Руководство по миграции для существующих пользователей

## Обзор изменений

В этой версии Qwen Code Orchestrator Kit были добавлены значительные улучшения и новые функции, направленные на повышение безопасности, качества кода и эффективности разработки. Ниже приведено руководство по миграции для существующих пользователей.

## Новые компоненты

### 1. Система агентов безопасности
- **Агент анализа безопасности** (`security-analyzer`): сканирует код на наличие уязвимостей
- **Команда `/health-security`**: запускает комплексную проверку безопасности
- **Навык `security-scanner`**: проверяет код на наличие известных паттернов уязвимостей
- **Скрипт `generate-security-report.sh`**: генерирует отчеты о найденных уязвимостях

### 2. Система анализа зависимостей
- **Агент анализа зависимостей** (`dependency-analyzer`): проверяет зависимости на уязвимости и устаревание
- **Команда `/health-deps`**: запускает аудит зависимостей
- **Навык `dependency-auditor`**: проверяет зависимости на наличие проблем
- **Скрипт `update-dependencies.sh`**: анализирует и предлагает обновления зависимостей

### 3. Система обнаружения мертвого кода
- **Агент обнаружения мертвого кода** (`dead-code-detector`): находит неиспользуемые компоненты
- **Команда `/health-cleanup`**: запускает процесс очистки мертвого кода
- **Навык `code-cleaner`**: помогает удалять мертвый код
- **Скрипт `remove-dead-code.sh`**: анализирует и отмечает потенциальный мертвый код

### 4. Контрольные точки качества
- **Команда `/run-quality-gate`**: запускает комплексную проверку качества
- **Скрипт `check-coverage.sh`**: проверяет покрытие кода тестами
- **Скрипт `check-security.sh`**: проверяет проект на уязвимости
- **Скрипт `check-bundle-size.sh`**: проверяет размер собранного проекта

### 5. Система MCP-конфигураций
- **Скрипт `switch-mcp.sh`**: позволяет динамически переключаться между конфигурациями
- **Конфигурации**: BASE, DATABASE, FRONTEND, FULL для оптимизации использования токенов

### 6. Система Git worktree
- **Команда `/worktree-create`**: создает новый worktree для изолированной разработки
- **Команда `/worktree-list`**: показывает список всех worktree
- **Команда `/worktree-remove`**: удаляет указанный worktree
- **Скрипт `manage-worktree.sh`**: управляет worktree для параллельной разработки

### 7. Система вебхуков
- **Навык `webhook-sender`**: отправляет уведомления о событиях
- **Команда `/configure-webhooks`**: помогает настроить систему уведомлений
- **Скрипт `handle-webhooks.sh`**: обрабатывает события и отправляет уведомления

## Шаги миграции

### Шаг 1: Обновление структуры проекта
1. Убедитесь, что у вас есть директории:
   - `.qwen/skills/` (если не существует, создайте)
   - `scripts/orchestration-tools/` (если не существует, создайте)
   - `docs/architecture/` (если не существует, создайте)
   - `tests/integration/` (если не существует, создайте)

2. Проверьте, что у вас есть файлы конфигурации:
   - `.qwen/settings.local.json.example`

### Шаг 2: Установка новых агентов
1. Скопируйте новые агенты в `.qwen/agents/`:
   - `security-analyzer.md`
   - `dependency-analyzer.md`
   - `dead-code-detector.md`

2. Убедитесь, что у вас есть все необходимые команды в `.qwen/commands/`:
   - `health-security.md`
   - `health-deps.md`
   - `health-cleanup.md`
   - `run-quality-gate.md`
   - `worktree-create.md`
   - `worktree-list.md`
   - `worktree-remove.md`
   - `configure-webhooks.md`

### Шаг 3: Установка новых навыков
1. Скопируйте новые навыки в `.qwen/skills/`:
   - `security-scanner/SKILL.md`
   - `dependency-auditor/SKILL.md`
   - `code-cleaner/SKILL.md`
   - `webhook-sender/SKILL.md`

### Шаг 4: Установка новых скриптов
1. Скопируйте новые скрипты в `scripts/orchestration-tools/`:
   - `switch-mcp.sh`
   - `manage-worktree.sh`
   - `handle-webhooks.sh`
   - `generate-security-report.sh`
   - `update-dependencies.sh`
   - `remove-dead-code.sh`
   - `check-coverage.sh`
   - `check-security.sh`
   - `check-bundle-size.sh`
   - `base-quality-check.sh`

2. Убедитесь, что скрипты исполняемые:
   ```bash
   chmod +x scripts/orchestration-tools/*.sh
   ```

### Шаг 5: Настройка MCP-конфигураций
1. Скопируйте файлы конфигурации в корень проекта:
   - `.mcp.base.json`
   - `.mcp.database.json`
   - `.mcp.frontend.json`
   - `.mcp.full.json`

### Шаг 6: Обновление документации
1. Обновите документацию в `docs/architecture/`:
   - `security-guidelines.md`
   - `dependency-management.md`
   - `code-maintenance.md`
   - `quality-standards.md`
   - `mcp-configurations.md`
   - `worktree-guidelines.md`
   - `notification-system.md`
   - `standards.md`

### Шаг 7: Настройка системы уведомлений
1. Создайте файл конфигурации уведомлений:
   ```bash
   cp .qwen/settings.local.json.example .qwen/settings.local.json
   ```
2. Отредактируйте `.qwen/settings.local.json`, добавив свои URL вебхуков

### Шаг 8: Обновление README.md
1. Обновите ваш `README.md`, добавив информацию о новых возможностях

## Проверка миграции

После выполнения всех шагов миграции, выполните комплексный тест:

1. Запустите базовую проверку качества:
   ```bash
   bash scripts/orchestration-tools/base-quality-check.sh
   ```

2. Проверьте, что все новые команды доступны:
   - `/health-security`
   - `/health-deps`
   - `/health-cleanup`
   - `/run-quality-gate`
   - `/worktree-create`
   - `/worktree-list`
   - `/worktree-remove`
   - `/configure-webhooks`

3. Проверьте, что все скрипты работают:
   - `scripts/orchestration-tools/switch-mcp.sh`
   - `scripts/orchestration-tools/manage-worktree.sh`
   - `scripts/orchestration-tools/handle-webhooks.sh`

## Рекомендации по использованию

### Для команд разработки
1. Начните использовать команды проверки качества в вашем процессе разработки:
   - Запускайте `/health-security` перед слиянием кода
   - Используйте `/health-deps` для регулярного аудита зависимостей
   - Применяйте `/health-cleanup` для поддержания чистоты кода

2. Используйте систему worktree для параллельной разработки:
   - Создавайте отдельные worktree для каждой фичи
   - Используйте `/worktree-create` для изоляции изменений

3. Настройте систему уведомлений для автоматического информирования команды:
   - Настройте вебхуки для уведомлений о завершении задач
   - Используйте разные каналы для разных типов событий

### Для архитекторов
1. Используйте контрольные точки качества как часть процесса проверки:
   - Запускайте `/run-quality-gate` перед релизами
   - Внедрите автоматические проверки в CI/CD

2. Оптимизируйте использование токенов с помощью системы MCP-конфигураций:
   - Используйте `switch-mcp.sh` для переключения между конфигурациями
   - Выбирайте подходящую конфигурацию для каждого типа задачи

## Возможные проблемы и решения

### Проблема: Команда не найдена
**Решение**: Убедитесь, что файл команды находится в правильной директории `.qwen/commands/` и имеет правильное имя.

### Проблема: Скрипт не исполняется
**Решение**: Проверьте права доступа к файлу и установите их при необходимости:
```bash
chmod +x путь/к/скрипту.sh
```

### Проблема: Ошибка при запуске агента
**Решение**: Проверьте, что файл агента находится в `.qwen/agents/` и содержит правильную YAML-спецификацию в начале файла.
