# Команда /translate-doc

## Синтаксис

/translate-doc {путь_к_файлу}

## Ваша задача

Вы переводите техническую документацию с английского на русский для разработчиков.

### Шаг 1: Чтение исходного файла

Прочитайте файл документации на английском, указанный пользователем:
- Используйте инструмент Read с предоставленным путем_к_файлу
- Проверьте, существует ли файл и доступен ли он для чтения
- Если файл не найден, сообщите об ошибке и остановитесь

### Шаг 2: Перевод на русский

Переведите содержимое, сохраняя структуру и точность:

**Сохранить**:
- Форматирование Markdown (заголовки, списки, таблицы, блоки кода)
- Блоки кода БЕЗ ИЗМЕНЕНИЙ (не переводить код)
- Точность технических терминов (orchestrator, worker, quality gate)
- Ссылки и URL БЕЗ ИЗМЕНЕНИЙ
- Пути к файлам и команды БЕЗ ИЗМЕНЕНИЙ
- YAML frontmatter (если присутствует)
- Примеры JSON БЕЗ ИЗМЕНЕНИЙ

**Перевести**:
- Всё текстовое содержимое (абзацы, описания)
- Заголовки и названия
- Элементы списков (кроме кода)
- Содержимое таблиц (кроме столбцов с кодом)

**Руководящие принципы перевода**:
- Использовать профессиональный технический русский
- Поддерживать последовательность с существующей русской документацией
- Сохранять терминологию Claude Code: agent, orchestrator, worker, quality gate
- Сохранять тон и стиль (формальная техническая документация)

### Шаг 3: Добавить заголовок перевода

В начале переведенного документа (после YAML frontmatter, если присутствует), добавьте:

> Предупреждение: Автоматический перевод
> Переведено с английского: {путь_к_исходному_файлу}
> Дата перевода: {ГГГГ-ММ-ДД}
> Оригинал может содержать более актуальную информацию. В случае расхождений используйте английскую версию.

### Шаг 4: Сохранить переведенный файл

Сохраните переведенное содержимое:
- Сгенерируйте имя файла: {оригинальное_имя_файла}.ru.md
- Сохраните в том же каталоге, что и исходный файл
- Используйте инструмент Write с полным путем

Пример:
- Исходник: docs/Agents Ecosystem/ARCHITECTURE.md
- Вывод: docs/Agents Ecosystem/ARCHITECTURE.ru.md

### Шаг 5: Сообщить о завершении

Сообщите пользователю:

Перевод завершен

Источник: {путь_к_исходному_файлу}
Вывод: {путь_к_файлу_вывода}

Перевод включает:
- N разделов переведено
- M блоков кода сохранено
- Заголовок перевода добавлен

Примечание: Проверьте перевод на техническую точность, особенно специфичные для домена термины.

## Примеры

### Пример 1: Базовое использование

Пользователь: /translate-doc docs/Agents Ecosystem/ARCHITECTURE.md
Ассистент: Перевожу документ...
Вывод: docs/Agents Ecosystem/ARCHITECTURE.ru.md создан

### Пример 2: Обработка ошибок

Пользователь: /translate-doc docs/nonexistent.md
Ассистент: Ошибка: Файл не найден в docs/nonexistent.md
