# Индекс проекта: Claude Code Orchestrator Kit

> Быстрая навигация для AI-агентов. Обновлено: 2025-11-27
>
> **Назначение:** Помогает агентам понять структуру проекта без чтения всей кодовой базы.

## Архитектура

- **[CLAUDE.md](../../CLAUDE.md)** — Правила поведения, паттерны оркестрации, основная точка входа
- **[docs/ARCHITECTURE.md](../../docs/ARCHITECTURE.md)** — Системный дизайн, диаграммы C4, паттерны рабочих процессов
- **[docs/FAQ.md](../../docs/FAQ.md)** — Часто задаваемые вопросы и ответы

## Основные домены

### Агенты (.claude/agents/)
- **[agents/](agents/)** — 33+ специализированных AI-агента
  - **[health/orchestrators/](agents/health/orchestrators/)** — Рабочие процессы багов, безопасности, зависимостей, очистки
  - **[health/workers/](agents/health/workers/)** — Охотник за багами, исправитель багов, сканер безопасности и т.д.
  - **[development/workers/](agents/development/workers/)** — Специалисты по LLM, TypeScript, стоимости
  - **[meta/workers/](agents/meta/workers/)** — meta-agent-v3, skill-builder-v2
  - Паттерн: Оркестратор делегирует Работникам через инструмент Task

### Команды (.claude/commands/)
- **[commands/](commands/)** — 19+ команд с косой чертой
  - Ключ: `health-*.md` — Рабочие процессы мониторинга состояния
  - Ключ: `speckit.*.md` — Инструментарий спецификаций
  - Ключ: `push.md` — Автоматизация релизов
  - Паттерн: Команды запускают оркестраторы или прямые действия

### Навыки (.claude/skills/)
- **[skills/](skills/)** — 15+ многоразовых утилит
  - Ключ: `run-quality-gate/` — Проверка типов, сборка, валидация тестов
  - Ключ: `validate-plan-file/` — Валидация JSON-схемы
  - Ключ: `rollback-changes/` — Восстановление файлов
  - Паттерн: Состоятельные утилиты, <100 строк, вызываются через инструмент Skill

### Конфигурации MCP (mcp/)
- **[mcp/](../../mcp/)** — 7 конфигураций сервера MCP
  - Ключ: `.mcp.base.json` — Минимальная (~600 токенов)
  - Ключ: `.mcp.serena.json` — Семантический поиск LSP (~2500 токенов)
  - Ключ: `.mcp.full.json` — Все серверы (~6500 токенов)
  - Паттерн: switch-mcp.sh для динамического переключения

## Конфигурация

## Паттерны и соглашения

- **Оркестрация:** Основной Claude Code делегирует субагентам через инструмент Task
- **Проверка:** Всегда читать измененные файлы + запускать проверку типов после делегирования
- **Контрольные точки качества:** проверка типов, сборка, тесты должны проходить перед коммитом
- **Коммиты:** `/push patch` после каждого выполненного задания

## Ключевые точки входа

| Назначение | Файл | Описание |
|------------|------|----------|
| Правила поведения | `CLAUDE.md` | Читается Claude Code при запуске сессии |
| Выбор агента | `settings.local.json` | Сопоставление subagent_type инструмента Task |
| Рабочие процессы состояния | `commands/health-*.md` | Точки входа для проверок состояния |
