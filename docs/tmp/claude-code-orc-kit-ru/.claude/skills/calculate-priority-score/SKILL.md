# Расчет приоритетного балла

Расчет числового приоритетного балла и категории для проблем на основе нескольких факторов.

## Когда использовать

- Определение приоритетности ошибок
- Оценка рисков уязвимостей безопасности
- Упорядочивание задач
- Принятие решений о распределении ресурсов

## Инструкции

### Шаг 1: Получение атрибутов проблемы

Принять атрибуты проблемы в качестве входных данных.

**Ожидаемый ввод**:
```json
{
  "severity": "critical|high|medium|low",
  "impact": "breaking|major|minor|none",
  "likelihood": "certain|likely|possible|unlikely"
}
```

### Шаг 2: Загрузка матрицы оценки

Использовать матрицу оценки для назначения баллов.

**Баллы серьезности**:
- critical: 10
- high: 7
- medium: 5
- low: 2

**Баллы влияния**:
- breaking: 10
- major: 7
- minor: 3
- none: 0

**Баллы вероятности**:
- certain: 10
- likely: 7
- possible: 5
- unlikely: 2

### Шаг 3: Расчет общего балла

Суммировать все факторные баллы.

**Формула**: `score = severity + impact + likelihood`

**Диапазон**: 0-30

### Шаг 4: Определение категории приоритета

Сопоставить балл с категорией приоритета.

**Категории приоритета**:
- **P0 (Критический)**: 25-30
  - Метка: "Критический - Требуется немедленное действие"
  - Действие: Прекратить все, исправить немедленно

- **P1 (Высокий)**: 19-24
  - Метка: "Высокий - Исправить в этом спринте"
  - Действие: Приоритизировать в текущем спринте

- **P2 (Средний)**: 12-18
  - Метка: "Средний - Запланировать на следующий спринт"
  - Действие: Включить в бэклог, исправить в ближайшее время

- **P3 (Низкий)**: 5-11
  - Метка: "Низкий - Запланировать по возможности"
  - Действие: Было бы неплохо, низкий приоритет

- **P4 (Минимальный)**: 0-4
  - Метка: "Минимальный - Рассмотреть закрытие"
  - Действие: Возможно, не стоит исправлять

### Шаг 5: Возврат оцененного результата

Вернуть полную оценку приоритета.

**Ожидаемый вывод**:
```json
{
  "score": 27,
  "category": "P0",
  "label": "Critical - Immediate Action Required",
  "factors": {
    "severity": 10,
    "impact": 10,
    "likelihood": 7
  },
  "recommendation": "Drop everything, fix now"
}
```

## Обработка ошибок

- **Недопустимая серьезность**: Вернуть ошибку с перечислением допустимых значений
- **Недопустимое влияние**: Вернуть ошибку с перечислением допустимых значений
- **Недопустимая вероятность**: Вернуть ошибку с перечислением допустимых значений
- **Отсутствующий фактор**: Вернуть ошибку с запросом всех факторов

## Примеры

### Пример 1: Критическая ошибка

**Ввод**:
```json
{
  "severity": "critical",
  "impact": "breaking",
  "likelihood": "certain"
}
```

**Вывод**:
```json
{
  "score": 30,
  "category": "P0",
  "label": "Critical - Immediate Action Required",
  "factors": {
    "severity": 10,
    "impact": 10,
    "likelihood": 10
  },
  "recommendation": "Drop everything, fix now"
}
```

### Пример 2: Проблема среднего приоритета

**Ввод**:
```json
{
  "severity": "medium",
  "impact": "minor",
  "likelihood": "likely"
}
```

**Вывод**:
```json
{
  "score": 15,
  "category": "P2",
  "label": "Medium - Schedule for Next Sprint",
  "factors": {
    "severity": 5,
    "impact": 3,
    "likelihood": 7
  },
  "recommendation": "Include in backlog, address soon"
}
```

### Пример 3: Улучшение низкого приоритета

**Ввод**:
```json
{
  "severity": "low",
  "impact": "minor",
  "likelihood": "unlikely"
}
```

**Вывод**:
```json
{
  "score": 7,
  "category": "P3",
  "label": "Low - Schedule When Convenient",
  "factors": {
    "severity": 2,
    "impact": 3,
    "likelihood": 2
  },
  "recommendation": "Nice to have, low priority"
}
```

### Пример 4: Высокое влияние, но маловероятно

**Ввод**:
```json
{
  "severity": "high",
  "impact": "breaking",
  "likelihood": "unlikely"
}
```

**Вывод**:
```json
{
  "score": 19,
  "category": "P1",
  "label": "High - Fix This Sprint",
  "factors": {
    "severity": 7,
    "impact": 10,
    "likelihood": 2
  },
  "recommendation": "Prioritize in current sprint"
}
```

- [ ] Обрабатывает все допустимые значения факторов
- [ ] Возвращает четкие рекомендации
- [ ] Проверяет входные факторы
