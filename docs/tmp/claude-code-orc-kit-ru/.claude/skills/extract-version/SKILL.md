# Извлечение версии

Анализ строк семантической версии и проверка соответствия semver.

## Когда использовать

- Извлечение версии из смешанного текста
- Проверка формата версии
- Анализ компонентов версии
- Сравнение номеров версий
- Нормализация строк версии

## Инструкции

### Шаг 1: Получение входных данных версии

Принять строку версии в различных форматах.

**Ожидаемый ввод**:
- `versionString`: Строка (например, "v0.7.0", "0.7.0", "version: 0.7.0", "Ver. 1.2.3-beta.1")

### Шаг 2: Извлечение шаблона версии

Использовать регулярное выражение для извлечения шаблона semver.

**Шаблоны для сопоставления**:
- `X.Y.Z`: Базовый semver
- `vX.Y.Z`: С префиксом 'v'
- `X.Y.Z-prerelease`: С тегом предварительной версии
- `X.Y.Z+build`: С метаданными сборки
- Полный semver: `X.Y.Z-prerelease+build`

**Регулярное выражение**:
```
(\d+)\.(\d+)\.(\d+)(?:-([a-zA-Z0-9.-]+))?(?:\+([a-zA-Z0-9.-]+))?
```

### Шаг 3: Анализ компонентов

Извлечь основные, второстепенные, патч и дополнительные компоненты.

**Компоненты**:
- `major`: Первое число
- `minor`: Второе число
- `patch`: Третье число
- `prerelease`: Дополнительный тег предварительной версии (после -)
- `build`: Дополнительные метаданные сборки (после +)

### Шаг 4: Проверка Semver

Проверить, следует ли версия спецификации семантического версионирования 2.0.0.

**Правила проверки**:
- Основной, второстепенный и патч должны быть неотрицательными целыми числами
- Предварительная версия и сборка должны соответствовать разрешенным символам
- Нет ведущих нулей в числовых идентификаторах (кроме самого 0)

### Шаг 5: Возврат проанализированных данных

Вернуть структурированный объект версии.

**Ожидаемый вывод**:
```json
{
  "major": 0,
  "minor": 7,
  "patch": 0,
  "prerelease": null,
  "build": null,
  "original": "v0.7.0",
  "normalized": "0.7.0",
  "valid": true,
  "semver": "0.7.0"
}
```

## Обработка ошибок

- **Версия не найдена**: Вернуть ошибку, указывающую на отсутствие совпадения с шаблоном версии
- **Неверный формат**: Вернуть ошибку с неверной строкой версии
- **Неверный компонент**: Описать, какой компонент недействителен

## Примеры

### Пример 1: Базовая версия

**Ввод**:
```
versionString: "0.7.0"
```

**Вывод**:
```json
{
  "major": 0,
  "minor": 7,
  "patch": 0,
  "prerelease": null,
  "build": null,
  "original": "0.7.0",
  "normalized": "0.7.0",
  "valid": true,
  "semver": "0.7.0"
}
```

### Пример 2: Версия с префиксом

**Ввод**:
```
versionString: "v1.2.3"
```

**Вывод**:
```json
{
  "major": 1,
  "minor": 2,
  "patch": 3,
  "prerelease": null,
  "build": null,
  "original": "v1.2.3",
  "normalized": "1.2.3",
  "valid": true,
  "semver": "1.2.3"
}
```

### Пример 3: Предварительная версия

**Ввод**:
```
versionString: "2.0.0-beta.1"
```

**Вывод**:
```json
{
  "major": 2,
  "minor": 0,
  "patch": 0,
  "prerelease": "beta.1",
  "build": null,
  "original": "2.0.0-beta.1",
  "normalized": "2.0.0-beta.1",
  "valid": true,
  "semver": "2.0.0-beta.1"
}
```

### Пример 4: Полный Semver с сборкой

**Ввод**:
```
versionString: "1.0.0-alpha.1+001"
```

**Вывод**:
```json
{
  "major": 1,
  "minor": 0,
  "patch": 0,
  "prerelease": "alpha.1",
  "build": "001",
  "original": "1.0.0-alpha.1+001",
  "normalized": "1.0.0-alpha.1+001",
  "valid": true,
  "semver": "1.0.0-alpha.1+001"
}
```

### Пример 5: Извлечение из текста

**Ввод**:
```
versionString: "Version: 0.8.0 released on 2025-10-17"
```

**Вывод**:
```json
{
  "major": 0,
  "minor": 8,
  "patch": 0,
  "prerelease": null,
  "build": null,
  "original": "Version: 0.8.0 released on 2025-10-17",
  "normalized": "0.8.0",
  "valid": true,
  "semver": "0.8.0"
}
```

### Пример 6: Неверная версия

**Ввод**:
```
versionString: "1.2.a"
```

**Вывод**:
```json
{
  "valid": false,
  "error": "Invalid version format: patch must be numeric",
  "original": "1.2.a"
}
```

## Валидация

- [ ] Разбирает базовый semver (X.Y.Z)
- [ ] Правильно обрабатывает префикс 'v'
- [ ] Извлекает теги предварительной версии
- [ ] Извлекает метаданные сборки
- [ ] Проверяет соответствие semver
- [ ] Извлекает версию из смешанного текста
- [ ] Возвращает нормализованную версию

## Вспомогательные файлы

Не требуются - чистая логика разбора с регулярным выражением.
