# Форматирование списка задач

Создание правильно структурированных списков задач для инструмента TodoWrite с правильными полями статуса и activeForm.

## Когда использовать

- Создание начальных контрольных списков фаз в оркестраторах
- Обновление статуса списка задач
- Обеспечение последовательного форматирования задач
- Генерация списков отслеживания прогресса

## Инструкции

### Шаг 1: Получение элементов задач

Принять массив элементов задач со статусом.

**Ожидаемый ввод**:
```json
{
  "todos": [
    {
      "content": "Фаза 1: Обнаружение",
      "status": "completed"
    },
    {
      "content": "Фаза 2: Реализация",
      "status": "in_progress"
    },
    {
      "content": "Фаза 3: Проверка",
      "status": "pending"
    }
  ]
}
```

### Шаг 2: Генерация активных форм

Создать activeForm для каждой задачи на основе содержимого.

**Правила**:
- Преобразовать императив в настоящее продолженное время
- "Запустить X" → "Запускаю X"
- "Создать X" → "Создаю X"
- "Исправить X" → "Исправляю X"
- "Проверить X" → "Проверяю X"

### Шаг 3: Проверка статуса

Убедиться, что статус является допустимым значением TodoWrite.

**Допустимые статусы**:
- `pending`: Не начато
- `in_progress`: В настоящее время активно
- `completed`: Завершено

### Шаг 4: Форматирование для TodoWrite

Структурировать данные для инструмента TodoWrite.

**Ожидаемый вывод**:
```json
{
  "todos": [
    {
      "content": "Фаза 1: Обнаружение",
      "status": "completed",
      "activeForm": "Обнаруживаю и анализирую"
    },
    {
      "content": "Фаза 2: Реализация",
      "status": "in_progress",
      "activeForm": "Реализую изменения"
    },
    {
      "content": "Фаза 3: Проверка",
      "status": "pending",
      "activeForm": "Проверяю результаты"
    }
  ]
}
```

## Обработка ошибок

- **Неверный статус**: Вернуть ошибку с перечислением допустимых статусов
- **Отсутствует содержимое**: Вернуть ошибку для пустого содержимого
- **Пустой массив**: Вернуть ошибку с запросом хотя бы одной задачи

## Примеры

### Пример 1: Контрольный список фаз

**Ввод**:
```json
{
  "todos": [
    {"content": "Фаза 0: Предварительная проверка", "status": "completed"},
    {"content": "Фаза 1: Обнаружение ошибок", "status": "in_progress"},
    {"content": "Фаза 2: Исправление ошибок", "status": "pending"},
    {"content": "Фаза 3: Проверка", "status": "pending"}
  ]
}
```

**Вывод**:
```json
{
  "todos": [
    {
      "content": "Фаза 0: Предварительная проверка",
      "status": "completed",
      "activeForm": "Проверяю предварительные проверки"
    },
    {
      "content": "Фаза 1: Обнаружение ошибок",
      "status": "in_progress",
      "activeForm": "Обнаруживаю ошибки"
    },
    {
      "content": "Фаза 2: Исправление ошибок",
      "status": "pending",
      "activeForm": "Исправляю ошибки"
    },
    {
      "content": "Фаза 3: Проверка",
      "status": "pending",
      "activeForm": "Проверяю исправления"
    }
  ]
}
```

### Пример 2: Простой список задач

**Ввод**:

{

    {
      "content": "Запустить проверку типов",
      "status": "completed",
      "activeForm": "Запускаю проверку типов"
    },
    {
      "content": "Запустить сборку",
      "status": "completed",
      "activeForm": "Запускаю сборку"
    },
    {
      "content": "Запустить тесты",
      "status": "in_progress",
      "activeForm": "Запускаю тесты"
    }
  ]
}
```

      "status": "completed",

      "content": "Исправить ошибки среднего приоритета",
      "status": "pending",
      "activeForm": "Исправляю ошибки среднего приоритета"
    }
  ]
}
```
