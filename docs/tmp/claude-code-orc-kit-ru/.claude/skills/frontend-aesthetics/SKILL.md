# Эстетика фронтенда

Создание уникальных, креативных дизайнов фронтенда, избегающих стандартных эстетических решений, генерируемых ИИ, и шаблонных паттернов.

## Когда использовать

- Проектирование новых компонентов UI или макетов
- Выбор типографики и сочетаний шрифтов
- Выбор цветовых схем и тем
- Реализация анимаций и микровзаимодействий
- Проверка дизайнов фронтенда на наличие стандартных паттернов
- Принятие решений по дизайну для лендингов, дашбордов или веб-приложений
- Предоставление рекомендаций по дизайну агентам, ориентированным на фронтенд

## Инструкции

### Шаг 1: Оценка контекста дизайна

Понимать идентичность бренда проекта, его цель и целевую эстетику перед принятием дизайнерских решений.

**Ключевые вопросы**:
- Какова личность бренда проекта? (игривый, профессиональный, технический, редакционный и т.д.)
- Кто целевая аудитория?
- Какую эмоциональную реакцию должен вызывать дизайн?
- Что делает этот проект уникальным?

### Шаг 2: Выбор типографики

Выбирать красивые, уникальные и интересные шрифты, соответствующие характеру проекта.

**ИЗБЕГАТЬ этих стандартных шрифтов**:
- Inter (чрезмерно используется в интерфейсах, генерируемых ИИ)
- Roboto
- Arial
- Системные шрифты (-apple-system, BlinkMacSystemFont)

**Рекомендуемые категории шрифтов**:

**Код/Техническая эстетика**:
- JetBrains Mono
- Fira Code
- Cascadia Code
- Victor Mono

**Редакционная/Изысканная**:
- Playfair Display
- Crimson Pro
- Spectral
- Lora

**Современная/Чистая**:
- Space Grotesk (использовать скупо - становится все более распространенным)
- DM Sans
- Outfit
- Plus Jakarta Sans

**Критично**: Варьировать выбор шрифтов в разных проектах. Не сходиться на одних и тех же выборах (например, Space Grotesk) во всех генерациях.

### Шаг 3: Дизайн цвета и темы

Создавать связные цветовые системы с использованием CSS-переменных с доминирующими цветами и яркими акцентами.

**Принципы**:
- Доминирующие цвета с яркими акцентами > робкие, равномерно распределенные палитры
- Следовать связной эстетике с использованием CSS-переменных
- Брать вдохновение из тем IDE (Dracula, Nord, Tokyo Night, Monokai и т.д.)
- Учитывать культурную эстетику, релевантную контексту проекта

**ИЗБЕГАТЬ**:
- Фиолетовые градиенты на белом фоне (клише эстетики ИИ)
- Стандартные сочетания синего/серого
- Предсказуемые радужные палитры с равным весом
- Безопасные, корпоративные цветовые схемы, когда они неуместны в контексте

**Подход**:
- Выбрать 1-2 доминирующих цвета, определяющих бренд
- Добавить 1-2 ярких акцентных цвета для призывов к действию и выделений
- Использовать пользовательские CSS-свойства для темизации
- Рассмотреть варианты как светлого, так и темного режима

### Шаг 4: Движение и анимация

Использовать анимации стратегически для моментов с высоким воздействием и приятных микровзаимодействий.

**Приоритеты анимации**:
1. **Моменты с высоким воздействием**: Оркестрованные загрузки страниц с пошаговыми открытиями
2. **Микровзаимодействия**: Наведение на кнопки, переходы, изменения состояния
3. **Контекстные эффекты**: Анимации, запускаемые прокруткой, параллакс

**Руководящие принципы реализации**:
- **Для HTML/Vanilla JS**: Приоритет решений только на CSS (переходы, анимации, @keyframes)
- **Для React**: Использовать библиотеку Motion (Framer Motion) при наличии
- **Сосредоточиться на оркестровке**: Одна хорошо оркестрованная последовательность > разбросанные микровзаимодействия
- **Использовать animation-delay**: Создавать пошаговые открытия для связанных элементов

**Пример паттерна**:
```css
.stagger-item {
  animation: fadeInUp 0.6s ease-out forwards;
  opacity: 0;
}
.stagger-item:nth-child(1) { animation-delay: 0.1s; }
.stagger-item:nth-child(2) { animation-delay: 0.2s; }
.stagger-item:nth-child(3) { animation-delay: 0.3s; }
```

### Шаг 5: Фон и атмосфера

Создавать глубину и атмосферу через слоистые фоны и контекстные эффекты.

**ИЗБЕГАТЬ**:
- По умолчанию использовать сплошные цвета
- Простые белые или серые фоны
- Плоские, безжизненные поверхности

**Рекомендуемые подходы**:
- Слоистые CSS-градиенты для глубины
- Использовать геометрические узоры (полосы, сетки, точки)
- Добавлять едва заметные текстуры шума
- Реализовывать контекстные эффекты (свечение, размытие, тени)
- Сопоставлять сложность фона общей эстетике

**Примеры паттернов**:
```css
/* Слоистый градиент */
background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);

/* Геометрический узор */
background-image:
  repeating-linear-gradient(45deg, transparent, transparent 10px, rgba(0,0,0,.05) 10px, rgba(0,0,0,.05) 20px);

/* Едва заметная текстура шума */
background-image: url('data:image/svg+xml,...'), linear-gradient(...);
```

### Шаг 6: Проверка на анти-паттерны

Проверять дизайн на распространенные ловушки UI, генерируемого ИИ.

**Контрольный список анти-паттернов**:
- [ ] Не использовать шрифты Inter, Roboto, Arial или системные шрифты
- [ ] Нет фиолетовых градиентов на белом фоне
- [ ] Цветовая палитра имеет четкую иерархию (доминирующий + акцент, не равномерное распределение)
- [ ] Анимации оркестрованы и целенаправленны (не разбросаны)
- [ ] Фоны имеют глубину и атмосферу (не сплошные цвета)
- [ ] Макет креативен и соответствует контексту (не шаблонный)
- [ ] Типографика уникальна и соответствует личности бренда
- [ ] Дизайнерские решения различаются от предыдущих проектов (не повторяются)

### Шаг 7: Возврат рекомендаций по дизайну

Предоставлять конкретные, реализуемые рекомендации по дизайну.

## Обработка ошибок

- **Отсутствует контекст**: Задавать уточняющие вопросы об идентичности бренда и цели проекта
- **Обнаружены стандартные выборы**: Явно предупреждать и предлагать уникальные альтернативы
- **Повторяющиеся паттерны**: Отмечать сходимость на ранее использованные эстетики
- **Непонятные требования**: Запрашивать конкретные цели или ограничения дизайна

## Валидация

- [ ] Рекомендует уникальные шрифты, соответствующие характеру проекта
- [ ] Явно предупреждает против стандартных шрифтов (Inter, Roboto, Arial, системные)
- [ ] Создает цветовые палитры с четкой иерархией (доминирующий + акцент)
- [ ] Избегает клишированных цветовых схем (фиолетовые градиенты)
- [ ] Предлагает оркестрованные анимации для моментов с высоким воздействием
- [ ] Рекомендует фоны с глубиной и атмосферой
- [ ] Проверяет на анти-паттерны
- [ ] Варьирует эстетические рекомендации в разных контекстах
- [ ] Предоставляет обоснование, специфичное для контекста, для каждого выбора

## Интеграция с агентами

### Агенты, ориентированные на фронтенд

Перед реализацией компонентов UI вызывать навык frontend-aesthetics для получения рекомендаций по дизайну:

```markdown
## Шаг 1: Рекомендации по дизайну

Использовать навык frontend-aesthetics для получения рекомендаций по типографике, цвету, анимации и фону.

Ввод: Контекст проекта, идентичность бренда, целевая эстетика
Вывод: Комплексные рекомендации по дизайну с конкретными рекомендациями

Проверить вывод на анти-паттерны перед продолжением.
```

### Агенты код-ревью

Использовать навык frontend-aesthetics для оценки существующего кода фронтенда на наличие стандартных паттернов:

```markdown
## Шаг 3: Обзор эстетики фронтенда

Использовать навык frontend-aesthetics в режиме проверки:
- Извлечь текущий выбор шрифтов из CSS
- Определить цветовую палитру из CSS-переменных
- Проверить реализацию анимации
- Проверить сложность фона

Отметить любые обнаруженные анти-паттерны.
```

### Интеграция оркестратора

Включить проверку frontend-aesthetics в контрольные точки качества:

```markdown
## Фаза 2: Обзор дизайна

Использовать навык frontend-aesthetics для проверки дизайнерских решений:
1. Проверить выбор типографики
2. Проверить цветовую схему
3. Проверить подход к анимации
4. Оценить глубину фона

Если anti_pattern_validation.passed == false:
  Делегировать улучшения дизайна фронтенд-работнику
```

## Примечания

- **Бюджет токенов**: Этот навык намеренно сфокусирован (~500 токенов) для использования в качестве справочника по дизайну без перегрузки контекста агента
- **Вариативность критична**: Агенты должны избегать схождения на одних и тех же дизайнерских выборах (например, Space Grotesk) во всех проектах
- **Контекст имеет значение**: Рекомендации по дизайну всегда должны соответствовать идентичности бренда проекта и его цели
- **Креативность превыше шаблонов**: Поощрять неожиданные, уникальные выборы, а не безопасные, предсказуемые паттерны
- **Соображения производительности**: Анимации только на CSS предпочтительны для HTML; Framer Motion подходящ для React, когда доступен
- **Лицензирование шрифтов**: Убедиться, что рекомендуемые шрифты доступны через Google Fonts или другие доступные источники
- **Доступность**: Уникальная эстетика не должна компрометировать читаемость или соответствие WCAG
- **Итерация**: Дизайнерские решения могут быть уточнены на основе обратной связи пользователей и тестирования
