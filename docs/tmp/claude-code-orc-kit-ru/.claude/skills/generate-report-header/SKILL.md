# Генерация заголовка отчета

Создание последовательных, хорошо отформатированных заголовков для всех отчетов, генерируемых агентом, с правильными метаданными.

## Когда использовать

- В начале всех отчетов, генерируемых рабочим
- Сводные документы
- Отчеты о проверке
- Отчеты об аудите
- Любые стандартизированные выводы, требующие заголовка

## Инструкции

### Шаг 1: Сбор информации заголовка

Собрать необходимую информацию для заголовка.

**Ожидаемый ввод**:
- `reportType`: Строка (например, "Bug Hunting", "Security Audit", "Version Update")
- `version`: Строка (например, "0.8.0", "2025-10-17", "final")
- `status`: Строка (success|partial|failed|in_progress)
- `timestamp`: Строка (необязательно, формат ISO-8601, по умолчанию текущее время)
- `duration`: Строка (необязательно, например, "3m 45s", "1h 12m")
- `workflow`: Строка (необязательно, например, "bugs", "security", "dead-code", "dependencies")
- `phase`: Строка (необязательно, например, "detection", "fixing", "verification")
- `additionalMetadata`: Объект (необязательно, дополнительные поля)

### Шаг 2: Форматирование временной метки

Преобразовать временную метку в читаемый формат при необходимости.

**Формат**: "YYYY-MM-DD HH:mm:ss UTC"

**Пример**: "2025-10-17 14:30:00 UTC"

### Шаг 3: Определение эмодзи статуса

Сопоставить статус с соответствующим эмодзи.

**Сопоставление статуса**:
- `success`: ✅
- `partial`: ⚠️
- `failed`: ❌
- `in_progress`: 🔄

### Шаг 4: Генерация заголовка

Создать отформатированный заголовок markdown.

**Ожидаемый вывод**:
```markdown
# {Тип отчета} Report: {Версия}

**Generated**: {Временная метка}
**Status**: {Эмодзи статуса} {Статус}
**Version**: {Версия}
**Duration**: {Продолжительность} (если предоставлено)
**Workflow**: {Рабочий процесс} (если предоставлено)
**Phase**: {Фаза} (если предоставлено)

---

## Executive Summary
```

**Стандартные метрики** (включать при наличии):
- Временная метка (ISO-8601)
- Продолжительность (человекочитаемый формат)
- Рабочий процесс (домен: ошибки, безопасность, мертвый код, зависимости)
- Фаза (обнаружение, исправление, проверка)
- Статус проверки (✅ ПРОЙДЕНО, ⛔ НЕ ПРОЙДЕНО, ⚠️ ЧАСТИЧНО)

### Шаг 5: Добавление дополнительных метаданных

Включить дополнительные поля метаданных, если они предоставлены.

**Необязательные поля**:
- Имя агента
- Продолжительность
- Количество файлов
- Количество проблем
- Любые пользовательские поля

## Обработка ошибок

- **Отсутствует тип отчета**: Вернуть ошибку с запросом типа отчета
- **Неверный статус**: Вернуть ошибку с перечислением допустимых статусов
- **Неверная временная метка**: Использовать текущее время и выдать предупреждение

## Примеры

### Пример 1: Отчет по поиску ошибок

**Ввод**:
```json
{
  "reportType": "Bug Hunting",
  "version": "2025-10-17",
  "status": "success",
  "additionalMetadata": {
    "agent": "bug-hunter",
    "filesScanned": 147,
    "bugsFound": 23
  }
}
```

**Вывод**:
```markdown
# Bug Hunting Report: 2025-10-17

**Generated**: 2025-10-17 14:30:00 UTC
**Status**: ✅ success
**Version**: 2025-10-17
**Agent**: bug-hunter
**Files Scanned**: 147
**Bugs Found**: 23

---

## Executive Summary
```

### Пример 2: Отчет об обновлении версии

**Ввод**:
```json
{
  "reportType": "Version Update",
  "version": "0.7.0 → 0.8.0",
  "status": "success"
}
```

**Вывод**:
```markdown
# Version Update Report: 0.7.0 → 0.8.0

**Generated**: 2025-10-17 14:30:00 UTC
**Status**: ✅ success
**Version**: 0.7.0 → 0.8.0

---

## Executive Summary
```

### Пример 3: Аудит безопасности (частичный)

**Ввод**:
```json
{
  "reportType": "Security Audit",
  "version": "final",
  "status": "partial",
  "timestamp": "2025-10-17T14:30:00Z",
  "additionalMetadata": {
    "criticalIssues": 2,
    "highIssues": 5,
    "fixedIssues": 5
  }
}
```

**Вывод**:
```markdown
# Security Audit Report: final

**Generated**: 2025-10-17 14:30:00 UTC
**Status**: ⚠️ partial
**Version**: final
**Critical Issues**: 2
**High Issues**: 5
**Fixed Issues**: 5

---

## Executive Summary
```

### Пример 4: Неудачное обновление зависимостей

**Ввод**:
```json
{
  "reportType": "Dependency Update",
  "version": "2025-10-17",
  "status": "failed",
  "additionalMetadata": {
    "error": "npm install failed",
    "failedPackages": ["package-a", "package-b"]
  }
}
```

**Вывод**:
```markdown
# Dependency Update Report: 2025-10-17

**Generated**: 2025-10-17 14:30:00 UTC
**Status**: ❌ failed
**Version**: 2025-10-17
**Error**: npm install failed
**Failed Packages**: package-a, package-b

---

## Executive Summary
```

## Валидация

- [ ] Генерирует заголовок со всеми обязательными полями
- [ ] Правильно форматирует временную метку
- [ ] Сопоставляет статус с правильным эмодзи
- [ ] Включает дополнительные метаданные при наличии
- [ ] Проверяет значения статуса
- [ ] Использует текущее время, если временная метка отсутствует
