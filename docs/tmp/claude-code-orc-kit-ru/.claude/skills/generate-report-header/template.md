# Шаблон заголовка отчета

## Стандартный формат

```markdown
# {ReportType} Отчет: {Version}

**Сформирован**: {Timestamp}
**Статус**: {Emoji} {Status}
**Версия**: {Version}

---

## Итоги
```

## Значения статуса

- **success** (✅): Операция завершена успешно
- **partial** (⚠️): Операция завершена с предупреждениями или частичными сбоями
- **failed** (❌): Операция критически не удалась
- **in_progress** (🔄): Операция в настоящее время выполняется

## Формат времени

ISO-8601 или удобочитаемый: "YYYY-MM-DD HH:mm:ss UTC"

## Дополнительные метаданные

Общие дополнительные поля:
- **Agent**: Имя агента-работника
- **Duration**: Время выполнения
- **Files Processed**: Количество файлов
- **Issues Found**: Количество найденных проблем
- **Error**: Сообщение об ошибке (для неудачного статуса)

## Типы отчетов

- Отчет по охоте за багами
- Отчет по аудиту безопасности
- Отчет по мертвому коду
- Отчет по аудиту зависимостей
- Отчет по обновлению версии
- Отчет по здоровью кода
- Отчет по верификации

## Пример полного заголовка

```markdown
# Отчет по охоте за багами: 2025-10-17

**Сформирован**: 2025-10-17 14:30:00 UTC
**Статус**: ✅ success
**Версия**: 2025-10-17
**Agent**: bug-hunter
**Duration**: 3m 45s
**Files Scanned**: 147
**Bugs Found**: 23

---

## Итоги

Этот отчет документирует результаты автоматического обнаружения багов в кодовой базе.
Критические: 3 | Высокие: 8 | Средние: 12
```
