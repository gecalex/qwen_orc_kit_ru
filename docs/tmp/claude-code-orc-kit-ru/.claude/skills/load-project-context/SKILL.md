# Загрузка контекста проекта

Загружает файл индекса проекта для понимания структуры проекта без чтения всей кодовой базы.

## Когда использовать

- Перед изучением незнакомых частей кодовой базы
- При планировании сложных изменений в нескольких файлах
- Когда нужно понять, где находится конкретная функциональность
- В начале сложных рабочих процессов оркестратора

## Когда НЕ использовать

- Для простых задач с одним файлом
- Когда вы уже знаете расположение файлов
- Для тривиальных исправлений ошибок

## Алгоритм

1. Проверить, существует ли `.claude/project-index.md`
2. Если существует:
   - Прочитать файл
   - Вернуть содержимое для использования агентом
3. Если не существует:
   - Сообщить, что индекс проекта не настроен
   - Предложить создать его из шаблона

## Реализация

```markdown
### Шаг 1: Проверка наличия индекса проекта

Прочитать файл `.claude/project-index.md`

### Шаг 2: Если файл существует

Вернуть полное содержимое. Агент должен использовать это для:
- Понимания структуры проекта
- Определения ключевых каталогов для задачи
- Поиска соответствующих паттернов и соглашений

### Шаг 3: Если файл не существует

Вернуть это сообщение:
"Индекс проекта не найден. Чтобы создать его:
1. Скопируйте `.claude/templates/project-index.template.md` в `.claude/project-index.md`
2. Настройте под структуру вашего проекта
3. Оставьте менее 150 строк для эффективности токенов"
```

## Формат вывода

Навык возвращает содержимое индекса проекта напрямую, которое включает:
- Обзор архитектуры со ссылками
- Основные домены с ключевыми каталогами
- Паттерны и соглашения
- Недавние изменения

## Стоимость токенов

- Чтение файла: ~100-200 токенов (если индекс хорошо поддерживается менее 150 строк)
- Никаких дополнительных накладных расходов обработки

## Пример использования

```
Агент: Мне нужно понять, где в этом проекте обрабатывается аутентификация.

[Вызывает навык load-project-context]

Навык возвращает содержимое project-index.md, показывающее:
- Паттерн аутентификации: "Supabase Auth + RLS policies"
- Ключевой каталог: "packages/api/src/middleware/" для middleware аутентификации

Теперь агент точно знает, где искать, без сканирования всей кодовой базы.
```

## Сопровождение

Индекс проекта должен обновляться:
- Когда происходят крупные архитектурные изменения
- Еженедельно для раздела "Недавние изменения"
- Когда добавляются новые домены/пакеты
