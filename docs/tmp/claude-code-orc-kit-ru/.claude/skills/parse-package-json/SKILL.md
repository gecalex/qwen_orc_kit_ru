# Разбор Package JSON

Извлечение структурированных данных из файлов package.json для управления версиями, аудита зависимостей и получения метаданных.

## Когда использовать

- Получение текущей версии проекта для рабочих процессов релиза
- Извлечение списка зависимостей для аудита
- Чтение метаданных пакета для отчетов
- Проверка структуры package.json

## Инструкции

### Шаг 1: Чтение package.json

Использовать инструмент Read для загрузки файла package.json.

**Ожидаемый ввод**:
- `file_path`: Путь к package.json (по умолчанию: `./package.json`)

**Используемые инструменты**: Read

### Шаг 2: Разбор JSON

Разбор содержимого JSON и проверка структуры.

**Проверка**:
- Файл должен быть действительным JSON
- Должен содержать поля `name` и `version` как минимум
- Зависимости должны быть объектами (если присутствуют)

### Шаг 3: Извлечение полей

Извлечение соответствующих полей в структурированный вывод.

**Поля для извлечения**:
- `version`: Строковое значение семантической версии
- `name`: Название пакета
- `description`: Описание пакета (необязательно)
- `dependencies`: Объект производственных зависимостей (необязательно)
- `devDependencies`: Объект зависимостей разработки (необязательно)
- `scripts`: Доступные npm-скрипты (необязательно)
- `main`: Файл точки входа (необязательно)
- `private`: Флаг приватности (необязательно)

### Шаг 4: Возврат структурированных данных

Вернуть извлеченные данные в виде объекта JSON.

**Ожидаемый вывод**:
```json
{
  "version": "0.7.0",
  "name": "megacampus2",
  "description": "Project description",
  "dependencies": {
    "react": "^18.2.0"
  },
  "devDependencies": {
    "typescript": "^5.0.0"
  },
  "scripts": {
    "build": "tsc",
    "test": "jest"
  },
  "main": "dist/index.js",
  "private": true
}
```

## Обработка ошибок

- **Файл не найден**: Вернуть ошибку с сообщением "package.json not found at {path}"
- **Неверный JSON**: Вернуть ошибку с деталями разбора
- **Отсутствуют обязательные поля**: Вернуть ошибку с перечислением отсутствующих полей (name, version)
- **Неверная структура**: Вернуть ошибку с описанием проблемы структуры

## Примеры

### Пример 1: Извлечение версии

**Ввод**:
```
file_path: ./package.json
```

**Вывод**:
```json
{
  "version": "0.7.0",
  "name": "megacampus2"
}
```

### Пример 2: Полная информация о пакете

**Ввод**:
```
file_path: ./packages/client/package.json
```

**Вывод**:
```json
{
  "version": "1.2.3",
  "name": "@megacampus/client",
  "dependencies": {
    "react": "^18.2.0",
    "react-dom": "^18.2.0"
  },
  "scripts": {
    "dev": "vite",
    "build": "vite build"
  }
}
```

### Пример 3: Ошибка - Файл не найден

**Ввод**:
```
file_path: ./nonexistent.json
```

**Вывод**:
```json
{
  "error": "package.json not found at ./nonexistent.json",
  "success": false
}
```

## Валидация

- [ ] Может читать package.json из пользовательского пути
- [ ] Извлекает все указанные поля правильно
- [ ] Корректно обрабатывает отсутствующие необязательные поля
- [ ] Возвращает ошибку при отсутствии файла
- [ ] Возвращает ошибку при неверном JSON
- [ ] Возвращает ошибку при отсутствии обязательных полей
