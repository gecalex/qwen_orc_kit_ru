# Отображение шаблона

Отображение шаблонов с подстановкой переменных, поддержка различных форматов синтаксиса и вспомогательных функций.

## Когда использовать

- Форматирование сообщений коммитов с динамическими данными
- Генерация разделов отчетов с переменными
- Создание форматированного вывода из шаблонов
- Замена заполнителей в текстовых шаблонах
- Консолидация логики форматирования в навыках

## Инструкции

### Шаг 1: Получение шаблона и данных

Принять строку шаблона и данные переменных.

**Ожидаемый ввод**:
- `template`: Строка (шаблон с заполнителями)
- `variables`: Объект (пары ключ-значение для подстановки)
- `syntax`: Строка (необязательно: mustache|shell|mixed, по умолчанию: mustache)

**Поддерживаемый синтаксис**:
- **Mustache**: `{{variable}}`, `{{object.property}}`
- **Shell**: `${variable}`, `${VARIABLE}`
- **Mixed**: Поддерживает оба

### Шаг 2: Разбор шаблона

Определить все заполнители в шаблоне.

**Шаблоны заполнителей**:
- `{{variable}}`: Простая переменная
- `{{object.property}}`: Вложенное свойство
- `{{#if condition}}...{{/if}}`: Условие (необязательно)
- `{{#each array}}...{{/each}}`: Цикл (необязательно)

### Шаг 3: Разрешение переменных

Заменить заполнители значениями из объекта переменных.

**Правила разрешения**:
- Прямое совпадение: `{{name}}` → `variables.name`
- Вложенное свойство: `{{user.name}}` → `variables.user.name`
- Отсутствующая переменная: Заменить пустой строкой или оставить заполнитель (настраивается)
- Неопределенная: Заменить пустой строкой

### Шаг 4: Применение фильтров (необязательно)

Поддержка простых фильтров, если указано.

**Общие фильтры**:
- `{{variable | uppercase}}`: Преобразовать в верхний регистр
- `{{variable | lowercase}}`: Преобразовать в нижний регистр
- `{{variable | capitalize}}`: Заглавная первая буква
- `{{date | format:YYYY-MM-DD}}`: Формат даты

### Шаг 5: Возврат отображенного вывода

Вернуть окончательную отображенную строку.

**Ожидаемый вывод**: Строка (шаблон с замененными переменными)

## Обработка ошибок

- **Отсутствующие переменные**: Заменить пустой строкой или выдать предупреждение (настраивается)
- **Неверный синтаксис шаблона**: Вернуть ошибку с описанием проблемы синтаксиса
- **Циклические ссылки**: Обнаружить и вернуть ошибку
- **Несоответствие типов**: Автоматически преобразовать в строку

## Примеры

### Пример 1: Простая подстановка переменных

**Ввод**:
```json
{
  "template": "Hello {{name}}, your score is {{score}}!",
  "variables": {
    "name": "Alice",
    "score": 95
  },
  "syntax": "mustache"
}
```

**Вывод**:
```
Hello Alice, your score is 95!
```

### Пример 2: Вложенные свойства

**Ввод**:
```json
{
  "template": "User: {{user.name}} ({{user.email}})\nRole: {{user.role}}",
  "variables": {
    "user": {
      "name": "Bob",
      "email": "bob@example.com",
      "role": "admin"
    }
  }
}
```

**Вывод**:
```
User: Bob (bob@example.com)
Role: admin
```

### Пример 3: Синтаксис Shell

**Ввод**:
```json
{
  "template": "Version: ${VERSION}\nDate: ${RELEASE_DATE}",
  "variables": {
    "VERSION": "0.8.0",
    "RELEASE_DATE": "2025-10-17"
  },
  "syntax": "shell"
}
```

**Вывод**:
```
Version: 0.8.0
Date: 2025-10-17
```

### Пример 4: Шаблон сообщения коммита

**Ввод**:
```json
{
  "template": "{{type}}({{scope}}): {{description}}\n\n{{body}}\n\n🤖 Generated with Claude Code\nCo-Authored-By: Claude <noreply@anthropic.com>",
  "variables": {
    "type": "feat",
    "scope": "auth",
    "description": "add OAuth2 support",
    "body": "Implemented OAuth2 flow with token refresh.\nSupports Google and GitHub providers."
  }
}
```

**Вывод**:
```
feat(auth): add OAuth2 support

Implemented OAuth2 flow with token refresh.
Supports Google and GitHub providers.

🤖 Generated with Claude Code
Co-Authored-By: Claude <noreply@anthropic.com>
```

### Пример 5: Отсутствующие переменные

**Ввод**:
```json
{
  "template": "Name: {{name}}\nAge: {{age}}\nCity: {{city}}",
  "variables": {
    "name": "Alice"
  }
}
```

**Вывод (с заменой пустой строкой)**:
```
Name: Alice
Age:
City:
```

**Вывод (с сохранением заполнителя)**:
```
Name: Alice
Age: {{age}}
City: {{city}}
```

### Пример 6: Шаблон заголовка отчета

**Ввод**:
```json
{
  "template": "# {{reportType}} Report: {{version}}\n\n**Generated**: {{timestamp}}\n**Status**: {{statusEmoji}} {{status}}\n**Version**: {{version}}",
  "variables": {
    "reportType": "Bug Hunting",
    "version": "2025-10-17",
    "timestamp": "2025-10-17 14:30:00 UTC",
    "statusEmoji": "✅",
    "status": "success"
  }
}
```

**Вывод**:
```markdown
# Bug Hunting Report: 2025-10-17

**Generated**: 2025-10-17 14:30:00 UTC
**Status**: ✅ success
**Version**: 2025-10-17
```

## Валидация

- [ ] Правильно заменяет простые переменные
- [ ] Обрабатывает вложенные свойства
- [ ] Поддерживает несколько форматов синтаксиса
- [ ] Корректно обрабатывает отсутствующие переменные
- [ ] Преобразует типы в строки
- [ ] Сохраняет форматирование (переводы строк, отступы)
- [ ] Обнаруживает циклические ссылки
- [ ] Возвращает четкие сообщения об ошибках

## Вспомогательные файлы

Не требуются - чистая логика отображения шаблона.

## Интеграция с другими навыками

Этот навык консолидирует логику форматирования, используемую:
- **format-commit-message**: Использовать render-template для форматирования сообщений
- **generate-report-header**: Использовать render-template для генерации заголовков
- **format-todo-list**: Использовать render-template для генерации activeForm
- **generate-changelog**: Использовать render-template для форматирования разделов

**Пример интеграции**:
```
Вместо: Жестко закодированное форматирование строк в каждом навыке
Использовать: Навык render-template с предопределенными шаблонами
Результат: Последовательное форматирование, легче поддерживать
```
