# Возобновление сессии

Проверяет и загружает контекст предыдущей сохраненной сессии, обеспечивая бесшовное продолжение рабочего процесса.

## Когда использовать

- В начале рабочих процессов оркестратора проверки (автоматически)
- Когда пользователь явно просит продолжить предыдущую работу
- После перезапуска сессии или переключения контекста

## Когда НЕ использовать

- Для новых рабочих процессов без предыдущего контекста
- Когда пользователь явно хочет начать с чистого листа

## Алгоритм

### Шаг 1: Проверка существующей сессии

Проверить, существует ли `.tmp/current/session/context.md`.

### Шаг 2: Проверка актуальности

Если файл существует, проверить метку времени "Updated":
- Если < 24 часов: Сессия действительна
- Если > 24 часов: Сессия устарела, спросить пользователя

### Шаг 3: Загрузка контекста

Прочитать `.tmp/current/session/context.md` и извлечь:
- Текущий рабочий процесс и фаза
- Последнее действие и следующие шаги
- Состояние Git
- Статус контрольной точки качества

### Шаг 4: Загрузка недавних записей журнала

Прочитать последние 10 записей из `.tmp/current/session/log.md`:
- Недавние решения
- Обнаруженные проблемы
- Выводы

### Шаг 5: Представление опции возобновления

Если найдена действительная сессия, показать пользователю:
```
Найдена предыдущая сессия:
- Рабочий процесс: health-bugs
- Фаза: 3/7 (Staged Fixing)
- Последнее действие: Исправлена проверка аутентификации
- Возраст: 2 часа назад

Опции:
1. Возобновить с места, где вы остановились
2. Начать сначала (архивировать текущую сессию)
```

### Шаг 6: При возобновлении

Вернуть объединенный контекст для продолжения оркестратором:
```markdown
## Возобновление сессии

### Текущее состояние
[Содержимое из context.md]

### Недавние решения
[Последние 5 записей из log.md]

### Следующие действия
[Из раздела Next Steps в context.md]
```

### Шаг 7: При начале с чистого листа

Если пользователь выбирает начало с чистого листа:
1. Переместить текущие файлы сессии в `.tmp/archive/session-[timestamp]/`
2. Вернуть пустой контекст
3. Оркестратор начинает Фазу 1

## Реализация

```markdown
### Проверить существование сессии

Прочитать `.tmp/current/session/context.md`

Если не найден:
  Вернуть "Предыдущая сессия не найдена. Начинаем с чистого листа."

### Проверить метку времени

Извлечь "Updated: YYYY-MM-DDTHH:MM:SS" из файла
Вычислить возраст в часах

Если возраст > 24:
  Спросить пользователя: "Сессия [X] часов. Возобновить или начать с чистого листа?"

### Загрузить и вернуть

Полностью прочитать context.md
Прочитать последние 500 строк log.md (охватывает ~10 записей)
Вернуть объединенное содержимое
```

## Формат вывода

### Когда сессия найдена:
```
Предыдущая сессия найдена:
- Рабочий процесс: health-bugs
- Фаза: 3/7 (Staged Fixing)
- Возраст: 2ч 15м
- Последнее: Исправлена проверка аутентификации в src/auth.ts

Возобновить? [Y/n]
```

### Когда нет сессии:
```
Предыдущая сессия не найдена. Начинаем новый рабочий процесс.
```

### Когда устарела:
```
Найдена устаревшая сессия (26 часов):
- Рабочий процесс: health-security
- Фаза: 2/5

[R]esume в любом случае, [A]рхивировать и начать с чистого листа, или [C]ancel?
```

## Стоимость токенов

- Чтение контекста: ~100-150 токенов
- Чтение журнала (последние 10 записей): ~200-300 токенов
- Всего: ~300-450 токенов (только при возобновлении)

## Связанные навыки

- `save-session-context` — Сохранить текущее состояние для дальнейшего использования
- `load-project-context` — Загрузить структуру проекта

## Интеграция с оркестраторами

Оркестраторы проверки должны вызывать этот навык в Фазе 0:

```markdown
## Фаза 0: Проверка сессии

1. Вызвать навык `resume-session`
2. Если действительная сессия:
   - Спросить пользователя: Возобновить или с чистого листа?
   - Если возобновить: Перейти к сохраненной фазе
3. Если нет сессии:
   - Перейти к Фазе 1
4. Создать новый контекст сессии
```
