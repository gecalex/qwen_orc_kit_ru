# Сохранение контекста сессии

Сохраняет текущее состояние рабочего процесса для обеспечения бесшовного возобновления сессии.

## Когда использовать

- После завершения каждой фазы в рабочих процессах оркестратора проверки
- Перед завершением длительной сессии
- При переключении контекста на другую задачу
- После значительного прогресса, который следует сохранить

## Когда НЕ использовать

- Для простых, быстрых задач, которые не нуждаются в возобновлении
- Когда рабочий процесс полностью завершен (вместо этого очистка)

## Алгоритм

### Шаг 1: Сбор текущего состояния

Собрать следующую информацию:
- Имя и фаза текущего рабочего процесса
- Уровень приоритета и количество завершенных
- Последнее завершенное действие
- Список измененных файлов (из git status)

### Шаг 2: Получение состояния Git

Запустить `git status` и `git log -1` для захвата:
- Текущей ветки
- Количество не зафиксированных файлов
- Хэш и сообщение последнего коммита

### Шаг 3: Определение следующих шагов

На основе текущей фазы и рабочего процесса, определить:
- Непосредственно следующее действие
- Следующие 2-3 действия

### Шаг 4: Запись файла контекста

Записать в `.tmp/current/session/context.md`:

```markdown
# Контекст сессии

> Автоматически сгенерирован навыком save-session-context. Обновлено: [ISO timestamp]

## Текущее состояние

- **Рабочий процесс:** [workflow-name]
- **Фаза:** [current]/[total] ([phase-name])
- **Приоритет:** [priority-level] ([completed] из [total] исправлено)
- **Последнее действие:** [description]

## Активные файлы

[Список из git status]

## Следующие шаги

1. [Следующее действие]
2. [Следующее действие]
3. [Третье действие]

## Состояние Git

- **Ветка:** [branch-name]
- **Не зафиксировано:** [count] файлов
- **Последний коммит:** [hash] "[message]"

## Статус контрольной точки качества

- **Проверка типов:** [status]
- **Сборка:** [status]
- **Тесты:** [status]

## Инструкции по возобновлению

Для продолжения этой сессии:
1. Прочитать этот файл, чтобы понять текущее состояние
2. Прочитать `session-log.md` для недавних решений
3. Продолжить из раздела "Следующие шаги"
```

### Шаг 5: Подтверждение сохранения

Вывести сообщение подтверждения с путем к файлу.

## Заметки по реализации

```bash
# Получить состояние git
git status --porcelain | wc -l  # Количество не зафиксированных
git log -1 --format="%h %s"     # Последний коммит
git branch --show-current       # Текущая ветка
```

## Вывод

Сообщение подтверждения:
```
Контекст сессии сохранен в .tmp/current/session/context.md
- Рабочий процесс: health-bugs
- Фаза: 3/7 (Staged Fixing)
- Не зафиксировано: 2 файла
```

## Стоимость токенов

- Выполнение: ~50 токенов для команд git
- Вывод: 0 (операция только для записи)
