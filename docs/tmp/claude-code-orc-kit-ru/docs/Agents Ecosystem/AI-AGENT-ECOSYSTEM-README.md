# Экосистема AI агентов для Claude Code
## Готовая к производству система оркестрации с навыками, работниками и контрольными точками качества

**Версия**: 1.0.0
**Статус**: Готово к производству
**Лицензия**: MIT
**Платформа**: Claude Code CLI

---

## 🎯 Что это?

**Полная, готовая к производству экосистема AI агентов** для Claude Code, которая реализует:

- ✅ **Оркестраторы** - Чистые координаторы для сложных рабочих процессов
- ✅ **Работники** - Специализированные агенты выполнения
- ✅ **Навыки** - Легковесные, повторно используемые утилиты
- ✅ **Контрольные точки качества** - Контрольные точки проверки с блокировкой/предупреждением
- ✅ **Система состояния** - Автоматическое исправление ошибок, сканирование безопасности, удаление мертвого кода
- ✅ **Система релизов** - Автоматическое обновление версий с помощью ИИ

**Основано на исследованиях**: Шаблоны из исследовательской системы многопользовательского ИИ Anthropic, Typhren, vanzan01, zhsama и официальной документации Claude Code.

**Эффективность токенов**: Двухуровневая конфигурация MCP экономит ~600-3000 токенов за разговор.

## 📦 Что включено

### 1. Оркестраторы (5)

**Область состояния**:
- `bug-orchestrator` - Поиск и исправление ошибок с проверкой
- `security-orchestrator` - Аудит безопасности и устранение уязвимостей
- `dead-code-orchestrator` - Обнаружение и очистка мертвого кода
- `dependency-orchestrator` - Аудит зависимостей и обновления
- `code-health-orchestrator` - Параллельное выполнение всех областей состояния

**Область релизов**:
- `release-orchestrator` - Автоматизированный рабочий процесс релиза

**Расположение**: `.claude/agents/health/orchestrators/`, `.claude/agents/release/`

### 2. Работники (10+)

**Работники состояния**:
- `bug-hunter` - Обнаружение ошибок через проверку типов и сборку
- `bug-fixer` - Исправление ошибок по одной с проверкой
- `security-scanner` - OWASP Top 10, проверка политик RLS
- `vulnerability-fixer` - Исправление уязвимостей безопасности
- `dead-code-hunter` - Обнаружение неиспользуемого кода
- `dead-code-remover` - Безопасное удаление мертвого кода
- `dependency-auditor` - Аудит зависимостей на уязвимости
- `dependency-updater` - Безопасное обновление зависимостей

**Работники релизов**:
- `version-updater` - Интеллектуальное обновление всех ссылок на версию

**Расположение**: `.claude/agents/health/workers/`, `.claude/agents/release/`

### 3. Навыки (10)

Легковесные утилиты для общих задач:

1. `parse-package-json` - Извлечение версии и метаданных
2. `validate-plan-file` - Проверка файлов плана оркестратора
3. `format-commit-message` - Генерация стандартизированных коммитов
4. `generate-report-header` - Создание согласованных заголовков отчетов
5. `parse-git-status` - Разбор вывода статуса git
6. `extract-version` - Разбор строк семантической версии
7. `format-todo-list` - Генерация списков TodoWrite
8. `validate-report-file` - Проверка полноты отчета
9. `calculate-priority-score` - Расчет приоритета ошибки/проблемы
10. `format-markdown-table` - Генерация таблиц markdown

**Расположение**: `.claude/skills/`

### 4. Команды (2)

**Команда состояния**: `/health [quick|full|bugs|security|cleanup|deps]`
- `quick`: Ошибки + Безопасность (15-30 мин)
- `full`: Все области параллельно (30-60 мин)
- `bugs`: Только обнаружение и исправление ошибок
- `security`: Только аудит безопасности и устранение
- `cleanup`: Только обнаружение и удаление мертвого кода
- `deps`: Только аудит зависимостей и обновления

**Команда релиза**: `/push [patch|minor|major|--skip-ai]`
- Автоматическое увеличение версии
- Обновление ссылок на версию с помощью ИИ
- Операции Git (коммит, тег, пуш)
- Генерация сводки релиза

**Расположение**: `.claude/commands/`

### 5. Контрольные точки качества (12)

Контрольные точки проверки для каждой фазы рабочего процесса:

**Область ошибок** (3 точки):
- Обнаружение завершено
- Исправления применены
- Проверка

**Область безопасности** (3 точки):
- Аудит завершен
- Критические исправления применены
- Проверка

**Область мертвого кода** (3 точки):
- Обнаружение завершено
- Очистка применена
- Проверка

**Область зависимостей** (3 точки):
- Аудит завершен
- Обновления применены
- Проверка

**Спецификация**: `docs/QUALITY-GATES-SPECIFICATION.md`

## 🏗️ Архитектура

### Шаблон оркестрации

```
Запрос пользователя
    ↓
Оркестратор (Координатор)
    ↓
Создает файл плана
    ↓
Сигнализирует готовность
    ↓
Возвращает управление
    ↓
Основная сессия Claude
    ↓
Авто-вызывает работника (на основе контекста)
    ↓
Работник выполняет
    ↓
Генерирует отчет
    ↓
Возвращается к оркестратору
    ↓
Оркестратор проверяет (Контрольная точка качества)
    ↓
Следующая фаза или завершение
```

### Ключевые шаблоны

**Шаблон возврата управления**:
- Оркестраторы не вызывают работников напрямую
- Они создают файлы плана и сигнализируют готовность
- Основная сессия Claude явно вызывает работников через инструмент Task
- Работники завершают и возвращаются
- Оркестраторы возобновляются для проверки

**Стратегия MCP**:
- По умолчанию: `.mcp.json` (context7, sequential-thinking) - минимальное количество токенов
- Расширенный: `.mcp.full.json` (добавляет supabase, playwright, n8n, shadcn)
- Переключение конфигураций для специализированных задач, возврат к обычной работе

**Шина и спицы**:
- Центральный оркестратор направляет работу
- Работники не общаются друг с другом
- Предотвращает хаос координации

**Контрольные точки качества**:
- Блокирующие критерии (⛔ ОСТАНОВИТЬ, если не прошло)
- Неблокирующие критерии (⚠️ предупредить, но продолжить)
- Переопределение пользователем с подтверждением
- Записывается в отчеты

## 📚 Примеры использования

### Пример 1: Быстрая проверка состояния

```bash
# Запустить быструю проверку состояния (ошибки + безопасность)
claude "/health quick"

# Ожидаемый поток:
# 1. code-health-orchestrator запускается
# 2. bug-orchestrator работает параллельно с security-orchestrator
# 3. bug-hunter обнаруживает ошибки → bug-fixer исправляет → проверка
# 4. security-scanner проверяет → vulnerability-fixer исправляет → проверка
# 5. Сводный отчет генерируется
```

### Пример 2: Выпуск новой версии

```bash
# Выпустить патч-версию (0.8.0 → 0.8.1)
claude "/push patch"

# Ожидаемый поток:
# 1. Предварительная проверка (чистый репозиторий, тесты проходят)
# 2. Увеличение версии в package.json
# 3. version-updater находит все ссылки на версию
# 4. Коммит Git, тег, пуш
# 5. Сводка релиза генерируется
```

### Пример 3: Использование навыков в вашем коде

```typescript
// Агент или команда могут ссылаться на навыки

// Шаг 1: Получить текущую версию
// Используйте навык parse-package-json для извлечения версии
// Ожидаемый вывод: { "version": "0.7.0" }

// Шаг 2: Форматировать сообщение коммита
// Используйте навык format-commit-message
// Входные данные: { type: "feat", scope: "api", description: "add user endpoint" }
// Вывод: "feat(api): add user endpoint\n\n🤖 Generated with Claude Code..."
```

### Пример 4: Пользовательский оркестратор

```markdown
---
name: my-custom-orchestrator
description: Custom workflow coordinator for [your domain]
tools: Read, Write, Grep, Glob, TodoWrite
---

# Мой пользовательский оркестратор

## Фаза 1: Планирование
1. Создать файл плана

## Фаза 2: Контрольная точка качества

- Проверить, существует ли отчет
- Проверить статус проверки

Сгенерировать сводку
```

## 📊 Метрики качества

### Проверки состояния кода

| Область | Обнаружение | Исправление | Проверка |
|--------|-----------|--------|--------------|
| Ошибки | проверка типов, сборка | По одной с проверкой | Ноль критических ошибок |
| Безопасность | OWASP Top 10, проверка RLS | Критические уязвимости | Ноль критических CVE |
| Мертвый код | Анализ AST, grep | Безопасное удаление | Сборка все еще работает |
| Зависимости | npm audit, устаревшие | Только критические CVE | npm audit чист

### Контрольные точки качества

- **12 точек** по 4 областям
- **Блокирующие**: проверка типов, сборка, критические ошибки/CVE
- **Неблокирующие**: тесты, покрытие, лучшие практики
- **Переопределение**: Требуется подтверждение пользователя

### Уровень успеха

- ✅ Проверка типов: 100% должны пройти
- ✅ Сборка: 100% должны пройти
- ⚠️ Тесты: Рекомендуется (90%+)
- ⚠️ Покрытие: Рекомендуется (80%+)

### Добавление новых агентов

1. **Создать файл агента**:
   ```bash
   touch .claude/agents/domain/my-agent.md
   ```

2. **Следовать шаблону**:
   ```markdown
   ---
   name: my-agent
   description: Use proactively for [specific task]
   tools: Read, Write, Bash
   ---

   # My Agent

   [Instructions following architecture guide patterns]
   ```

3. **Тестировать**:
   ```bash
   claude "Use my-agent to [task]"
   ```

### Добавление новых навыков

1. **Создать каталог навыка**:
   ```bash
   mkdir .claude/skills/my-skill
   ```

2. **Создать SKILL.md**:
   ```markdown
   ---
   name: My Skill
   description: [What it does and when to use it]
   allowed-tools: Read
   ---

   # My Skill

   ## Instructions
   [Step-by-step instructions]
   ```

3. **Тестировать**:
   ```bash
   claude "Use the my-skill Skill to [task]"
   ```
