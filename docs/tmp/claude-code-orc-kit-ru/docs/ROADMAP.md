## Стратегия версионности

### Семантическое версионирование

**Формат**: MAJOR.MINOR.PATCH

- **MAJOR**: Критические изменения (например, изменение основных директивив CLAUDE.md)
- **MINOR**: Новые возможности, обратная совместимость (например, новые агенты, команды)
- **PATCH**: Исправления ошибок, улучшения производительности

**Примеры**:
- `1.0.0` → `1.0.1`: Исправление ошибки (патч)
- `1.0.0` → `1.1.0`: Новые агенты (минор)
- `1.0.0` → `2.0.0`: Критические поведенческие изменения (мажор)

### График выпуска

- **Патч-релизы**: По мере необходимости (исправления ошибок)
- **Минорные релизы**: Ежемесячно (новые возможности)
- **Мажорные релизы**: Ежеквартально (крупные изменения)

## Политика устаревания

### Обратная совместимость

**Обещание**: Мы поддерживаем обратную совместимость в пределах MAJOR-версий.

**Пример**:
- Агенты v1.0 работают с v1.5
- v1.x → v2.0 может потребовать обновления

### Процесс устаревания

1. **Объявление** (за 3 месяца до удаления)
   - Пометить функцию как устаревшую в документации
   - Добавить предупреждающие сообщения
   - Предоставить руководство по миграции

2. **Период устаревания** (3 месяца)
   - Функция по-прежнему работает
   - Пользователи переходят к новому подходу
   - Поддержка доступна

3. **Удаление** (Следующая мажор-версия)
   - Функция удалена
   - Критическое изменение задокументировано
   - Руководство по миграции завершено

### Автономная платформа разработки

**Цель**: Позволить AI-агентам управлять всем жизненным циклом разработки программного обеспечения автономно.
