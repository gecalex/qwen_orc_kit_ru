# Пример использования навыков агента

Данный документ демонстрирует использование экспериментальных навыков агента в Qwen Code Orchestrator Kit.

## Сценарий: Комплексный анализ и улучшение кода

Представим, что у нас есть веб-приложение, которое требует комплексного анализа и улучшения. Мы будем использовать несколько навыков для решения задачи.

### Шаг 1: Поиск уязвимостей с помощью навыка security-analyzer

При наличии включенных навыков, можно запросить анализ безопасности:

```
Пожалуйста, проанализируй этот код на предмет уязвимостей безопасности.
```

Модель автоматически определит, что использовать навык `security-analyzer`, и выполнит анализ.

### Шаг 2: Проверка качества кода с помощью навыка code-quality-checker

Для проверки соответствия кода стандартам:

```
Проверь, соответствует ли этот код лучшим практикам и стандартам кодирования.
```

Модель вызовет навык `code-quality-checker`.

### Шаг 3: Поиск багов с помощью навыка bug-hunter

Для поиска потенциальных ошибок:

```
Найди возможные баги и логические ошибки в этом коде.
```

Модель вызовет навык `bug-hunter`.

### Шаг 4: Использование нескольких навыков

Для комплексного анализа можно использовать запрос, который задействует несколько навыков:

```
Выполни полный аудит этого кода: проверь на уязвимости, ошибки и качество кода.
```

Модель может последовательно вызвать несколько навыков: `security-analyzer`, `bug-hunter` и `code-quality-checker`.

### Шаг 5: Явный вызов навыка

Для явного вызова конкретного навыка используйте команду:

```
/skills security-analyzer
```

## Преимущества использования навыков

1. **Модульность**: Каждый навык решает конкретную задачу
2. **Автоматизация**: Модель сама определяет, когда использовать тот или иной навык
3. **Расширяемость**: Можно легко добавлять новые навыки
4. **Совместимость**: Навыки работают вместе с существующей системой агентов

## Создание собственных навыков

Для создания собственного навыка:

1. Создайте директорию в `.qwen/skills/`
2. Добавьте файл `SKILL.md` с описанием и инструкциями
3. Укажите имя и описание навыка в YAML заголовке

Пример структуры:

```
.qwen/skills/my-custom-skill/
└── SKILL.md
```

Пример содержимого SKILL.md:

```markdown
---
name: my-custom-skill
description: Описание того, что делает этот навык и когда его использовать
---

# Название навыка

## Инструкции
Шаги, которые должен выполнить навык.

## Примеры
Примеры использования навыка.
```
