# Пример использования Qwen Code Orchestrator Kit

Данный документ демонстрирует простой пример использования системы с использованием различных агентов.

## Сценарий: Улучшение безопасности веб-приложения

Представим, что у нас есть веб-приложение с потенциальными уязвимостями безопасности. Мы хотим:

1. Найти потенциальные уязвимости
2. Исправить их
3. Проверить качество кода
4. Перевести комментарии на русский язык

## Шаг 1: Поиск уязвимостей

Запустим `bug-hunter` для поиска потенциальных проблем:

```bash
task '{
  "subagent_type": "bug-hunter",
  "description": "Поиск уязвимостей безопасности в коде аутентификации",
  "prompt": "Проанализируй файлы в директории src/auth/ на предмет возможных уязвимостей безопасности, таких как SQL-инъекции, XSS, неправильная обработка сессий и т.д."
}'
```

## Шаг 2: Исправление найденных проблем

После получения отчета от `bug-hunter`, исправим найденные проблемы с помощью `bug-fixer`:

```bash
task '{
  "subagent_type": "bug-fixer",
  "description": "Исправление найденных уязвимостей",
  "prompt": "Исправь уязвимости, найденные в файле src/auth/login_handler.py, связанные с неправильной валидацией пользовательского ввода. Используй параметризованные запросы для защиты от SQL-инъекций."
}'
```

## Шаг 3: Проверка качества кода

После исправления проблем, проверим качество кода с помощью `code-quality-checker`:

```bash
task '{
  "subagent_type": "code-quality-checker",
  "description": "Проверка качества исправленного кода",
  "prompt": "Проверь файлы в директории src/auth/ на соответствие стилю кодирования, лучшим практикам безопасности и производительности."
}'
```

## Шаг 4: Перевод комментариев

Если нужно, переведем комментарии в коде на русский язык с помощью `tech-translator-ru`:

```bash
task '{
  "subagent_type": "tech-translator-ru",
  "description": "Перевод комментариев в файле аутентификации",
  "prompt": "Переведи все комментарии в файле src/auth/login_handler.py с английского на русский, сохранив техническую точность и форматирование."
}'
```

## Шаг 5: Проверка контрольных точек качества

Перед коммитом изменений, пройдем контрольные точки качества:

```bash
# Проверка после выполнения задачи
./scripts/run_quality_gate.sh 2

# Проверка перед коммитом
./scripts/run_quality_gate.sh 3
```

## Шаг 6: Коммит изменений

После прохождения всех проверок, закоммитим изменения:

```bash
git add src/auth/login_handler.py
git commit -m "fix(auth): устранить уязвимости безопасности в обработчике логина"
```

## Заключение

Этот пример показывает, как можно использовать различные агенты системы для улучшения безопасности и качества кода. Каждый агент выполняет свою специфическую задачу, а контрольные точки качества обеспечивают соблюдение стандартов проекта.
