Подготовь спецификацию для релиза проекта Qwen Orchestrator Kit. Включай только функциональные возможности, которые выражают предназначение проекта, исключив все аспекты внутренней разработки.

Сфокусируйся на следующих аспектах проекта:
1. Архитектурные компоненты, видимые для пользователей проекта
2. Функциональные возможности, доступные через команды и интерфейсы
3. Интеграции с внешними системами и инструментами
4. Стандарты и соглашения, применяемые в проекте
5. Документация и руководства для пользователей
6. Система навыков и агентов как часть функциональности проекта
7. Конфигурации MCP как часть функциональности проекта
8. Система уведомлений и интеграций

Не включай в спецификацию:
1. Внутренние процессы разработки
2. Инструменты и скрипты для внутреннего использования
3. Временные файлы и настройки разработки
4. Историю изменений и коммитов
5. Внутреннюю архитектуру планирования и реализации
6. Вспомогательные скрипты и утилиты для разработчиков
7. Файлы и каталоги, относящиеся к внутренним процессам разработки

Спецификация должна описывать проект как готовый продукт для конечных пользователей, а не как процесс его разработки.
