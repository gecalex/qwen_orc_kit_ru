# Промт для формирования спецификаций с помощью speckit.specify

## Контекст
Проект Qwen Orchestrator Kit - это система оркестрации агентов, предназначенная для автоматизации разработки программного обеспечения. Проект включает в себя различные типы агентов (оркестраторы и воркеры), навыки (skills), скрипты и вспомогательные компоненты, которые работают вместе для выполнения задач разработки.

## Цель
Создать спецификации для улучшения архитектуры и функциональности проекта Qwen Orchestrator Kit на основе анализа, представленного в отчете об исследовании проекта.

## Задачи для спецификаций

### 1. Улучшение системы навыков (Skills)
**Описание**: Необходимо реализовать недостающие навыки, такие как `generate-report-header`, `run-quality-gate`, и другие, которые имеют только шаблоны без полной реализации.
**Требования**:
- Полная реализация всех навыков с описанными форматами ввода/вывода
- Обеспечение согласованности между документацией и реализацией
- Тестирование функциональности каждого навыка

### 2. Создание мета-агента для автоматического создания агентов
**Описание**: Реализовать автоматизированное создание новых агентов в рамках плоской структуры.
**Требования**:
- Разработка мета-агента, аналогичного `meta-agent-v3` из Claude Code Orchestrator Kit
- Возможность создания новых агентов за 2-3 минуты
- Поддержка всех типов агентов (оркестраторы и воркеры)
- Интеграция с существующей системой именования

### 3. Улучшение навигации по агентам
**Описание**: Ввести систему категоризации или тегирования агентов для облегчения поиска в условиях плоской структуры.
**Требования**:
- Система тегирования агентов по функциональности
- Улучшенная навигация в рамках ограничений платформы
- Возможность быстрого поиска агентов по функциям
- Интеграция с системой документации

### 4. Внедрение системы индексации проекта
**Описание**: Реализовать `project-index.md` для ленивой загрузки знаний.
**Требования**:
- Создание файла индекса проекта
- Система ленивой загрузки знаний без накладных расходов
- Интеграция с существующими агентами
- Поддержка обновления индекса при изменении проекта

### 5. Разработка системы MCP-конфигураций
**Описание**: Создать несколько специализированных конфигураций для разных сценариев использования.
**Требования**:
- 7 различных конфигураций MCP для разных сценариев (Context7, Supabase, Sequential Thinking, Playwright, shadcn, n8n, и др.)
- Интеграция с существующими агентами
- Возможность выбора подходящей конфигурации для задач
- Документация по использованию каждой конфигурации

### 6. Добавление автоматического анализа задач
**Описание**: Внедрить механизм автоматического определения подходящих агентов для задач.
**Требования**:
- Автоматический анализ задач и определение подходящих агентов
- Интеграция с системой тегирования агентов
- Возможность ручной корректировки назначений
- Поддержка сложных задач, требующих нескольких агентов

### 7. Улучшение системы вебхуков
**Описание**: Добавить уведомления о завершении задач.
**Требования**:
- Система вебхуков для уведомлений о завершении задач
- Поддержка различных каналов уведомлений
- Интеграция с существующими процессами
- Возможность настройки уведомлений

### 8. Создание системы тестирования
**Описание**: Разработать автоматизированные тесты для проверки корректности работы компонентов.
**Требования**:
- Автоматизированные тесты для всех основных компонентов
- Интеграция с системой контрольных точек качества
- Поддержка юнит-тестирования и интеграционного тестирования
- Отчетность о результатах тестирования

## Архитектурные принципы
1. **Согласованность**: Все компоненты должны следовать единой архитектуре и использовать общие навыки
2. **Модульность**: Компоненты должны быть максимально независимыми
3. **Масштабируемость**: Архитектура должна поддерживать рост числа агентов и функций
4. **Автоматизация**: Максимальная автоматизация процессов создания и управления компонентами
5. **Документация**: Все компоненты должны быть должным образом документированы

## Ограничения
1. Плоская структура агентов не может быть изменена из-за ограничений платформы
2. Все изменения должны быть совместимы с существующей архитектурой
3. Необходимо обеспечить обратную совместимость с существующими агентами

## Критерии успеха
1. Все спецификации реализованы и протестированы
2. Улучшена архитектура и функциональность проекта
3. Повышена согласованность между компонентами
4. Улучшена навигация и поиск агентов
5. Повышена автоматизация процессов
6. Улучшена документация проекта
