# Agent Structure Checker

## Когда использовать
- При проверке новых агентов
- В процессе CI/CD для проверки соответствия шаблону
- При рефакторинге существующих агентов
- При автоматическом обновлении компонентов

## Инструкции
1. Прочитайте файл агента
2. Проверьте наличие обязательных разделов: ## Назначение, ## Интеграция навыков (для оркестраторов)
3. Проверьте порядок разделов в соответствии с шаблоном
4. Проверьте наличие примеров использования (при наличии)
5. Сформируйте отчет о проверке

## Формат ввода
```json
{
  "filePath": "путь/к/файлу/агента.md",
  "agentType": "orc|work"  // Тип агента для проверки специфичных требований
}
```

## Формат вывода
```json
{
  "isValid": true|false,
  "errors": [
    {
      "section": "имя_раздела",
      "error": "описание_ошибки",
      "severity": "critical|warning|info"
    }
  ],
  "warnings": [
    {
      "section": "имя_раздела",
      "warning": "описание_предупреждения",
      "suggestion": "предложение_по_исправлению"
    }
  ],
  "complianceScore": 0-100  // Процент соответствия шаблону
}
```

## Примеры
### Пример 1: Проверка оркестратора
- Вход: {"filePath": ".qwen/agents/orc_dev_task_coordinator.md", "agentType": "orc"}
- Результат: {"isValid": true, "errors": [], "warnings": [], "complianceScore": 100}

### Пример 2: Проверка с отсутствующими разделами
- Вход: {"filePath": ".qwen/agents/work_dev_code_analyzer.md", "agentType": "work"}
- Результат: {"isValid": false, "errors": [{"section": "## Интеграция навыков", "error": "раздел обязателен для оркестраторов", "severity": "critical"}], "warnings": [], "complianceScore": 90}
