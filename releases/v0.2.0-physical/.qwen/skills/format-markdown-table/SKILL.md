# Навык форматирования таблиц markdown

## Когда использовать
- При создании таблиц в отчетах
- При генерации сравнительных таблиц
- При форматировании статистических данных
- При создании сводных таблиц
- При представлении табличных данных в документации

## Инструкции

### Фаза 1: Анализ данных
1.1. Определить структуру входных данных
1.2. Определить количество столбцов и строк
1.3. Определить заголовки столбцов
1.4. Определить типы данных в каждом столбце

### Фаза 2: Форматирование таблицы
2.1. Создать заголовок таблицы с правильными разделителями
2.2. Установить выравнивание для каждого столбца
2.3. Форматировать каждую строку данных
2.4. Обеспечить правильное выравнивание и интервалы

### Фаза 3: Валидация формата
3.1. Проверить корректность синтаксиса markdown
3.2. Проверить выравнивание столбцов
3.3. Убедиться, что таблица отображается корректно
3.4. Проверить, что длинные строки не нарушают формат

## Формат ввода
```json
{
  "headers": ["string"],
  "rows": [
    ["string"]
  ],
  "alignments": ["left|center|right"] (optional),
  "title": "string (optional)",
  "caption": "string (optional)"
}
```

## Формат вывода
```markdown
[title (если указан)]

| [header1] | [header2] | ... |
|:---------|:---------:|----:|
| [cell1]   | [cell2]   | ... |
| ...       | ...       | ... |

[caption (если указан)]
```

## Примеры

### Пример 1: Простая таблица
**Вход:**
```json
{
  "headers": ["Имя", "Возраст", "Город"],
  "rows": [
    ["Иван", "30", "Москва"],
    ["Мария", "25", "СПб"],
    ["Петр", "35", "Новосибирск"]
  ]
}
```

**Выход:**
```markdown
| Имя   | Возраст | Город        |
|-------|---------|--------------|
| Иван  | 30      | Москва       |
| Мария | 25      | СПб          |
| Петр  | 35      | Новосибирск  |
```

### Пример 2: Таблица с выравниванием
**Вход:**
```json
{
  "headers": ["Функция", "Статус", "Время выполнения"],
  "rows": [
    ["Создание агента", "Успешно", "2.5s"],
    ["Проверка качества", "Ошибка", "1.2s"],
    ["Тестирование", "Успешно", "3.1s"]
  ],
  "alignments": ["left", "center", "right"]
}
```

**Выход:**
```markdown
| Функция           | Статус   | Время выполнения |
|:------------------|:--------:|-----------------:|
| Создание агента   | Успешно  |             2.5s |
| Проверка качества | Ошибка   |             1.2s |
| Тестирование      | Успешно  |             3.1s |
```

### Пример 3: Таблица с заголовком
**Вход:**
```json
{
  "title": "Статистика по задачам",
  "headers": ["ID", "Название", "Статус", "Приоритет"],
  "rows": [
    ["T001", "Создать агента", "Выполнено", "Высокий"],
    ["T002", "Проверить код", "В работе", "Средний"],
    ["T003", "Написать тесты", "Ожидает", "Низкий"]
  ],
  "alignments": ["left", "left", "center", "right"]
}
```

**Выход:**
```markdown
# Статистика по задачам

| ID   | Название        | Статус    | Приоритет |
|:-----|:----------------|:---------:|----------:|
| T001 | Создать агента  | Выполнено |   Высокий |
| T002 | Проверить код   | В работе  |   Средний |
| T003 | Написать тесты  | Ожидает   |    Низкий |
```

### Пример 4: Сводная таблица с пояснением
**Вход:**
```json
{
  "headers": ["Метрика", "Значение", "Цель"],
  "rows": [
    ["Покрытие тестами", "85%", "90%"],
    ["Время сборки", "4.2s", "<5s"],
    ["Количество багов", "3", "<5"]
  ],
  "caption": "*Таблица показывает текущее состояние метрик качества проекта"
}
```

**Выход:**
```markdown
| Метрика           | Значение | Цель  |
|-------------------|----------|-------|
| Покрытие тестами  | 85%      | 90%   |
| Время сборки      | 4.2s     | <5s   |
| Количество багов  | 3        | <5    |

*Таблица показывает текущее состояние метрик качества проекта
```

## Правила форматирования

1. Заголовки всегда выравниваются по левому краю
2. Разделительная строка определяет выравнивание столбцов:
   - `:---` для левого выравнивания
   - `:---:` для центрирования
   - `---:` для правого выравнивания
3. Все ячейки должны быть отделены символами `|`
4. Между содержимым ячейки и разделителями должен быть пробел
5. Длина строк не должна превышать 120 символов при возможности
