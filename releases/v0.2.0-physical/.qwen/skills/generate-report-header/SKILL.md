# Навык генерации заголовков отчетов

## Когда использовать
- При создании любого отчета агентом
- При формировании заголовков для отчетов об ошибках
- При создании отчетов по аудиту безопасности
- При генерации отчетов о зависимостях
- При создании любых выводов, требующих последовательного форматирования

## Инструкции

### Фаза 1: Сбор метаданных
1.1. Определить тип отчета (ошибка, безопасность, зависимость, выполнение задачи и т.д.)
1.2. Собрать информацию о версии
1.3. Определить имя агента, создавшего отчет
1.4. Зафиксировать дату и время создания отчета

### Фаза 2: Форматирование заголовка
2.1. Выбрать подходящий шаблон заголовка
2.2. Заполнить шаблон собранными метаданными
2.3. Убедиться, что формат соответствует стандартам
2.4. Проверить корректность отображения в разных средах

### Фаза 3: Валидация
3.1. Проверить наличие всех обязательных полей
3.2. Убедиться, что дата и время указаны в правильном формате
3.3. Проверить, что тип отчета указан корректно
3.4. Убедиться, что имя агента указано правильно

## Формат ввода
```json
{
  "reportType": "string",
  "agentName": "string",
  "version": "string",
  "timestamp": "ISO 8601 string",
  "status": "success|error|warning|info",
  "additionalMetadata": {}
}
```

## Формат вывода
```markdown
# {reportType} Отчет: {version}

**Агент**: {agentName}  
**Статус**: {status}  
**Дата создания**: {timestamp}  
**Дополнительные метаданные**: {additionalMetadata}

---

## Исполнительное резюме

[Краткое описание содержания отчета]

---
```

## Примеры

### Пример 1: Отчет об ошибках
**Вход:**
```json
{
  "reportType": "Баг-охотник",
  "agentName": "bug-hunter",
  "version": "1.0.0",
  "timestamp": "2026-01-31T08:30:00Z",
  "status": "success",
  "additionalMetadata": {
    "bugsFound": 5,
    "critical": 1,
    "high": 2,
    "medium": 2
  }
}
```

**Выход:**
```markdown
# Баг-охотник Отчет: 1.0.0

**Агент**: bug-hunter  
**Статус**: success  
**Дата создания**: 2026-01-31T08:30:00Z  
**Дополнительные метаданные**: {"bugsFound": 5, "critical": 1, "high": 2, "medium": 2}

---

## Исполнительное резюме

Отчет о результатах сканирования на наличие ошибок. Найдено 5 потенциальных багов, включая 1 критический и 2 высокоприоритетных.

---
```

### Пример 2: Отчет по безопасности
**Вход:**
```json
{
  "reportType": "Аудит безопасности",
  "agentName": "security-analyzer",
  "version": "2.1.0",
  "timestamp": "2026-01-31T09:15:00Z",
  "status": "warning",
  "additionalMetadata": {
    "vulnerabilitiesFound": 3,
    "critical": 0,
    "high": 1,
    "medium": 2
  }
}
```

**Выход:**
```markdown
# Аудит безопасности Отчет: 2.1.0

**Агент**: security-analyzer  
**Статус**: warning  
**Дата создания**: 2026-01-31T09:15:00Z  
**Дополнительные метаданные**: {"vulnerabilitiesFound": 3, "critical": 0, "high": 1, "medium": 2}

---

## Исполнительное резюме

Отчет о результатах аудита безопасности. Обнаружены 3 уязвимости, включая 1 высокоприоритетную и 2 средней важности.

---
```

### Пример 3: Отчет о выполнении задачи
**Вход:**
```json
{
  "reportType": "Выполнение задачи",
  "agentName": "orc_dev_task_coordinator",
  "version": "1.2.0",
  "timestamp": "2026-01-31T10:00:00Z",
  "status": "success",
  "additionalMetadata": {
    "tasksCompleted": 8,
    "tasksFailed": 0,
    "executionTime": "45.2s"
  }
}
```

**Выход:**
```markdown
# Выполнение задачи Отчет: 1.2.0

**Агент**: orc_dev_task_coordinator  
**Статус**: success  
**Дата создания**: 2026-01-31T10:00:00Z  
**Дополнительные метаданные**: {"tasksCompleted": 8, "tasksFailed": 0, "executionTime": "45.2s"}

---

## Исполнительное резюме

Отчет о выполнении задач координатором. Успешно выполнено 8 задач за 45.2 секунды, без ошибок.

---
```
