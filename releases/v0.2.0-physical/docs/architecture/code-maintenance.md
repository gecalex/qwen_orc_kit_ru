# Руководство по поддержке кода

## Общие принципы поддержки кода

### 1. Чистота кода
- Регулярно удаляйте мертвый код и неиспользуемые компоненты
- Поддерживайте актуальность зависимостей
- Удаляйте закомментированный код, если он не нужен

### 2. Читаемость
- Используйте понятные имена переменных и функций
- Добавляйте комментарии к сложным участкам кода
- Следуйте соглашениям об именовании

### 3. Тестируемость
- Пишите тестируемый код
- Разделяйте зависимости для лучшей тестируемости
- Поддерживайте высокий уровень покрытия тестами

## Типы мертвого кода

### 1. Неиспользуемые функции и методы
- **Описание**: Функции, которые объявлены, но нигде не вызываются
- **Риски**: Увеличение размера кодовой базы, усложнение понимания
- **Обнаружение**: Анализ связей между компонентами

### 2. Неиспользуемые переменные и константы
- **Описание**: Переменные, которые объявлены, но не используются
- **Риски**: Затруднение понимания кода, потенциальные ошибки
- **Обнаружение**: Анализ использования переменных

### 3. Неиспользуемые импорты
- **Описание**: Импорты, которые не используются в файле
- **Риски**: Увеличение времени загрузки, усложнение зависимостей
- **Обнаружение**: Анализ использования импортированных компонентов

### 4. Закомментированный код
- **Описание**: Код, который закомментирован и не выполняется
- **Риски**: Устаревание, усложнение понимания
- **Обнаружение**: Поиск закомментированных блоков кода

### 5. Недостижимый код
- **Описание**: Код, который не может быть выполнен из-за логики
- **Риски**: Потенциальные ошибки, усложнение понимания
- **Обнаружение**: Анализ потока выполнения

## Инструменты поддержки кода

### 1. Агент обнаружения мертвого кода
- Используйте команду `/health-cleanup` для запуска комплексной проверки на мертвый код
- Агент анализирует код на наличие неиспользуемых компонентов
- Генерирует отчет с рекомендациями по очистке кода

### 2. Скрипты проверки
- Используйте скрипт `scripts/orchestration-tools/remove-dead-code.sh` для анализа мертвого кода
- Скрипт автоматически ищет потенциальный мертвый код в проекте
- Создает детализированные отчеты с рекомендациями

### 3. Навыки очистки
- Используйте навык `code-cleaner` для проверки конкретных файлов на наличие мертвого кода
- Навык может быть интегрирован в другие агенты для проверки чистоты кода

## Практики поддержки кода

### 1. Регулярные проверки
- Проводите регулярные проверки на мертвый код
- Используйте автоматизированные инструменты для обнаружения мертвого кода
- Удаляйте мертвый код при обнаружении

### 2. Код-ревью
- Проверяйте наличие мертвого кода при код-ревью
- Используйте инструменты для автоматической проверки
- Обучайте команду принципам чистого кода

### 3. Автоматизация
- Настройте автоматические проверки мертвого кода в CI/CD
- Используйте инструменты статического анализа
- Настройте уведомления о появлении мертвого кода

## Рекомендации по очистке кода

### 1. Безопасность удаления
- Перед удалением кода убедитесь, что он действительно не используется
- Проверяйте зависимости и связи между компонентами
- Используйте систему контроля версий для отката изменений

### 2. Постепенная очистка
- Удаляйте мертвый код постепенно, небольшими частями
- Проверяйте работоспособность после каждого удаления
- Обновляйте тесты при необходимости

### 3. Документирование изменений
- Документируйте причины удаления кода
- Обновляйте документацию после очистки
- Сообщайте команде о произведенных изменениях
