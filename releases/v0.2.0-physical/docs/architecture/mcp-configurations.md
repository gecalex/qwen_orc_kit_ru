# Руководство по MCP-конфигурациям

## Общие принципы MCP-конфигураций

### 1. Динамическое переключение
- Используйте скрипт `scripts/orchestration-tools/switch-mcp.sh` для переключения между конфигурациями
- Переключайтесь в зависимости от текущих задач разработки
- Экономьте токены, используя только необходимые серверы

### 2. Оптимизация токенов
- Каждая конфигурация имеет оценку использования токенов
- Используйте минимально необходимую конфигурацию для задачи
- Отключайте ненужные серверы для экономии токенов

### 3. Специализация
- Разные конфигурации оптимизированы для разных типов задач
- Используйте подходящую конфигурацию для текущего типа работы
- Переключайтесь между конфигурациями по мере необходимости

## Типы MCP-конфигураций

### 1. BASE
- **Назначение**: Минимальная конфигурация для повседневной разработки
- **Серверы**: context7, server-sequential-thinking
- **Оценка токенов**: ~600
- **Использование**: Общие задачи разработки, минимальное использование токенов

### 2. DATABASE
- **Назначение**: Для работы с базами данных
- **Серверы**: context7, server-sequential-thinking, supabase
- **Оценка токенов**: ~2500
- **Использование**: Проектирование БД, политики RLS, SQL-запросы

### 3. FRONTEND
- **Назначение**: Для UI/UX разработки
- **Серверы**: context7, server-sequential-thinking, playwright, shadcn
- **Оценка токенов**: ~2000
- **Использование**: Автоматизация браузера, UI-компоненты

### 4. FULL
- **Назначение**: Полнофункциональная конфигурация
- **Серверы**: Все доступные серверы
- **Оценка токенов**: ~5000
- **Использование**: Максимальные возможности, сложные задачи

## Использование MCP-конфигураций

### 1. Переключение конфигураций
- Используйте скрипт `./scripts/orchestration-tools/switch-mcp.sh` для переключения
- Выберите подходящую конфигурацию из меню
- После переключения перезапустите Qwen Code CLI

### 2. Создание новых конфигураций
- Создайте новый JSON-файл с конфигурацией в корне проекта
- Следуйте формату существующих конфигураций
- Укажите оценку использования токенов
- Добавьте конфигурацию в скрипт переключения

### 3. Управление токенами
- Мониторьте использование токенов в зависимости от конфигурации
- Используйте более легкие конфигурации для простых задач
- Переключайтесь на более тяжелые конфигурации только при необходимости

## Практики использования MCP

### 1. Регулярное переключение
- Переключайтесь между конфигурациями в зависимости от задач
- Используйте BASE для повседневной работы
- Переключайтесь на специализированные конфигурации при необходимости

### 2. Оптимизация расхода токенов
- Следите за оценкой использования токенов для каждой конфигурации
- Используйте только необходимые серверы
- Отключайте ненужные серверы для экономии токенов

### 3. Документирование
- Документируйте использование различных конфигураций
- Обновляйте оценки токенов при изменении конфигураций
- Делитесь лучшими практиками с командой

## Рекомендации по выбору конфигурации

### 1. Тип задачи
- Для общих задач разработки: BASE
- Для работы с базами данных: DATABASE
- Для фронтенд-разработки: FRONTEND
- Для сложных задач с множеством инструментов: FULL

### 2. Ограничения по токенам
- Если бюджет токенов ограничен: BASE
- Если требуется специфичный функционал: соответствующая специализированная конфигурация
- Если бюджет токенов не ограничен: FULL

### 3. Производительность
- Для быстрой работы: BASE
- Для сложных аналитических задач: FULL
- Для специфичных задач: соответствующая специализированная конфигурация
