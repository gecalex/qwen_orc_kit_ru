# Интеграция с MCP (Model Context Protocol) серверами

## Обзор

Интеграция с MCP (Model Context Protocol) серверами позволяет агентам использовать внешние инструменты и сервисы для расширения своих возможностей. Эта система обеспечивает доступ к специализированным библиотекам, API, базам данных и другим ресурсам.

## Назначение

- Расширение возможностей агентов за счет внешних сервисов
- Обеспечение доступа к специализированным инструментам
- Улучшение качества и точности выполнения задач
- Интеграция с внешними системами (Supabase, GitHub, и др.)

## Доступные MCP серверы

### 1. Context7 (`mcp__context7__*`)

**Назначение**: Доступ к документации библиотек и фреймворков
**Использование**: 
- `mcp__context7__resolve-library-id` - определение ID библиотеки
- `mcp__context7__get-library-docs` - получение документации

**Когда использовать**:
- При реализации функций с использованием внешних библиотек
- Для проверки актуальных паттернов и API
- При необходимости получения информации о библиотеке

### 2. Supabase (`mcp__supabase__*`)

**Назначение**: Работа с базами данных Supabase
**Использование**:
- `mcp__supabase__list_tables` - просмотр таблиц
- `mcp__supabase__execute_sql` - выполнение SQL запросов
- `mcp__supabase__list_migrations` - просмотр миграций

**Когда использовать**:
- При работе с базами данных
- Для выполнения миграций
- При реализации функций, связанных с хранилищем данных

### 3. Sequential Thinking (`mcp__sequential_thinking__*`)

**Назначение**: Помощь в сложных многоступенчатых рассуждениях
**Использование**:
- `mcp__sequential_thinking__break_down_problem` - разбиение проблемы на части
- `mcp__sequential_thinking__evaluate_approaches` - оценка подходов

**Когда использовать**:
- При решении сложных многоступенчатых задач
- Для анализа архитектурных решений
- При планировании сложных процессов

### 4. Playwright (`mcp__playwright__*`)

**Назначение**: Автоматизация браузерных тестов
**Использование**:
- `mcp__playwright__run_browser_test` - запуск браузерных тестов
- `mcp__playwright__capture_screenshot` - создание скриншотов

**Когда использовать**:
- При тестировании UI компонентов
- Для автоматизации пользовательских сценариев
- При проверке адаптивности интерфейсов

### 5. shadcn (`mcp__shadcn__*`)

**Назначение**: Работа с компонентами shadcn
**Использование**:
- `mcp__shadcn__list_components` - список доступных компонентов
- `mcp__shadcn__get_component_docs` - документация компонента

**Когда использовать**:
- При реализации UI компонентов
- Для проверки доступных компонентов
- При интеграции с библиотекой shadcn/ui

### 6. n8n (`mcp__n8n_mcp__*`)

**Назначение**: Управление рабочими процессами n8n
**Использование**:
- `mcp__n8n_mcp__list_workflows` - список рабочих процессов
- `mcp__n8n_mcp__execute_workflow` - выполнение рабочего процесса

**Когда использовать**:
- При автоматизации рабочих процессов
- Для интеграции с n8n
- При управлении автоматизированными задачами

## Рекомендации по использованию

### Для разных типов задач

#### 1. Исправление ошибок
- **Рекомендуется**: `mcp__context7__*`
- **Причина**: Проверка актуальных паттернов перед реализацией исправлений

#### 2. Анализ безопасности
- **Рекомендуется**: `mcp__context7__*`, `mcp__supabase__*`
- **Причина**: Проверка безопасности библиотек и политик RLS

#### 3. Разработка UI
- **Рекомендуется**: `mcp__shadcn__*`, `mcp__playwright__*`
- **Причина**: Доступ к компонентам и автоматизация тестирования

#### 4. Работа с базами данных
- **Рекомендуется**: `mcp__supabase__*`
- **Причина**: Прямой доступ к функциям базы данных

#### 5. Автоматизация процессов
- **Рекомендуется**: `mcp__n8n_mcp__*`
- **Причина**: Управление рабочими процессами

## Интеграция в планы задач

Оркестраторы могут включать рекомендации по использованию MCP в плановые файлы:

```json
{
  "phase": 2,
  "config": {
    "priority": "critical",
    "scope": ["src/", "lib/"]
  },
  "validation": {
    "required": ["type-check", "build"],
    "optional": ["tests"]
  },
  "mcpGuidance": {
    "recommended": ["mcp__context7__*"],
    "library": "react",
    "reason": "Check current React patterns before implementing fixes"
  },
  "nextAgent": "work_specialist"
}
```

## Практические советы

1. **Используйте MCP только при необходимости** - не применяйте MCP для простых задач, которые можно выполнить с помощью встроенных инструментов

2. **Проверяйте доступность MCP серверов** - убедитесь, что требуемые MCP серверы настроены и доступны

3. **Учитывайте время ответа** - MCP серверы могут добавлять задержку к выполнению задач

4. **Обрабатывайте ошибки** - предусмотрите резервные пути при недоступности MCP серверов

5. **Следуйте рекомендациям** - используйте рекомендованные MCP серверы для каждого типа задач

## Обновление конфигурации

Конфигурация MCP серверов находится в файле `.mcp.json`. При добавлении новых серверов обновите эту конфигурацию и убедитесь, что все агенты могут получить доступ к новым возможностям.

## Отладка интеграции

Для отладки проблем с MCP интеграцией:

1. Проверьте файл `.mcp.json` на корректность конфигурации
2. Убедитесь, что требуемые пакеты установлены
3. Проверьте сетевое подключение к MCP серверам
4. Просмотрите логи выполнения команд MCP
