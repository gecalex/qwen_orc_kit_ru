# Руководство по работе с worktree

## Общие принципы работы с worktree

### 1. Параллельная разработка
- Используйте worktree для параллельной разработки нескольких фич
- Каждый worktree изолирован от других
- Работайте с разными фичами одновременно без конфликта изменений

### 2. Изоляция изменений
- Каждый worktree имеет собственное рабочее состояние
- Изменения в одном worktree не влияют на другие
- Упрощает тестирование и отладку отдельных фич

### 3. Эффективность
- Не переключайтесь между ветками, используйте разные worktree
- Ускоряет разработку при работе с несколькими фичами
- Упрощает управление контекстом разработки

## Команды для работы с worktree

### 1. Создание worktree
- Используйте команду `/worktree-create feature-name` для создания нового worktree
- Команда создаст новую ветку и worktree для указанной фичи
- Worktree будет создан в подкаталоге `.worktrees/`

### 2. Просмотр worktree
- Используйте команду `/worktree-list` для просмотра всех worktree
- Команда покажет список всех созданных worktree
- Отобразит статус каждого worktree

### 3. Удаление worktree
- Используйте команду `/worktree-remove feature-name` для удаления worktree
- Команда удалит worktree и связанную ветку
- Очистит директорию worktree

## Использование worktree

### 1. Создание нового worktree
- Используйте команду `/worktree-create feature/new-feature` для создания нового worktree
- Перейдите в созданный worktree для начала разработки
- Работайте в изолированной среде без влияния на другие ветки

### 2. Управление несколькими worktree
- Используйте команду `/worktree-list` для отслеживания текущих worktree
- Переключайтесь между директориями worktree для работы с разными фичами
- Используйте Git-операции в каждом worktree независимо

### 3. Удаление завершенных worktree
- Используйте команду `/worktree-remove feature-name` для очистки
- Удаляйте worktree после завершения разработки фичи
- Освобождайте место и уменьшайте сложность проекта

## Практики работы с worktree

### 1. Регулярное управление
- Регулярно просматривайте список worktree
- Удаляйте завершенные или неиспользуемые worktree
- Поддерживайте актуальность списка worktree

### 2. Наименование
- Используйте понятные имена для фич при создании worktree
- Следуйте соглашениям об именовании веток
- Документируйте назначение каждого worktree

### 3. Синхронизация
- Регулярно синхронизируйте worktree с основной веткой
- Обновляйте worktree перед началом работы
- Сливайте изменения из worktree в основную ветку

## Рекомендации по использованию worktree

### 1. Количество worktree
- Создавайте worktree только при необходимости
- Не создавайте слишком много worktree одновременно
- Удаляйте worktree после завершения работы над фичей

### 2. Управление ресурсами
- Следите за использованием дискового пространства worktree
- Удаляйте неиспользуемые worktree для освобождения места
- Учитывайте влияние worktree на производительность

### 3. Интеграция с процессом разработки
- Интегрируйте использование worktree в процесс разработки
- Обучите команду работе с worktree
- Установите стандарты использования worktree в команде
