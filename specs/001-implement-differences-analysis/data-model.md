# Модель данных для реализации решений на основе анализа различий между Qwen Code CLI и Claude Code Orchestrator Kit

## Сущности

### 1. Агент (Agent)
- **Поля**:
  - name: строка (уникальное имя агента)
  - type: enum (orc - оркестратор, work - воркер)
  - domain: строка (область применения: dev, frontend, backend, doc, testing, research)
  - description: строка (описание функциональности)
  - tools: массив (инструменты, доступные агенту)
  - model: строка (модель ИИ, если применимо)
  - color: строка (цвет для визуализации)
- **Отношения**: Может вызывать другие агенты через систему задач
- **Правила проверки**: Имя должно следовать формату {type}_{domain}_имя

### 2. Навык (Skill)
- **Поля**:
  - name: строка (уникальное имя навыка)
  - description: строка (описание функциональности)
  - allowed-tools: массив (разрешенные инструменты)
  - triggers: массив (условия активации)
- **Отношения**: Используется агентами для выполнения специфических задач
- **Правила проверки**: Имя должно быть уникальным в системе

### 3. План задач (Task Plan)
- **Поля**:
  - phase: число (номер фазы выполнения)
  - config: объект (конфигурация выполнения)
  - validation: объект (критерии валидации)
  - mcpGuidance: объект (рекомендации по использованию MCP)
  - nextAgent: строка (имя следующего агента)
- **Отношения**: Связывает агентов в рабочем процессе
- **Правила проверки**: Должен содержать обязательные поля

### 4. Отчет агента (Agent Report)
- **Поля**:
  - header: объект (заголовок с метаданными)
  - executiveSummary: строка (исполнительное резюме)
  - workPerformed: массив (выполненные задачи)
  - changesMade: массив (внесенные изменения)
  - validationResults: массив (результаты валидации)
  - metrics: объект (метрики выполнения)
  - errorsEncountered: массив (обнаруженные ошибки)
  - nextSteps: массив (следующие шаги)
  - artifacts: массив (артефакты выполнения)
- **Отношения**: Создается агентом по завершении задачи
- **Правила проверки**: Должен содержать обязательные разделы

### 5. Контрольная точка качества (Quality Gate)
- **Поля**:
  - gateId: строка (идентификатор контрольной точки)
  - command: строка (команда проверки)
  - isBlocking: булево (блокирующая или нет)
  - expectedResult: строка (ожидаемый результат)
- **Отношения**: Используется в процессах валидации
- **Правила проверки**: Должна возвращать четкий результат (PASSED/FAILED/WARNING)

### 6. Конфигурация MCP (MCP Configuration)
- **Поля**:
  - serverType: строка (тип сервера)
  - recommended: массив (рекомендуемые серверы)
  - library: строка (библиотека для работы)
  - reason: строка (причина рекомендации)
- **Отношения**: Используется в планах задач
- **Правила проверки**: Должна соответствовать доступным MCP серверам

## Переходы состояний

### Агент
- CREATED → READY (после настройки инструментов)
- READY → EXECUTING (при получении задачи)
- EXECUTING → COMPLETED/FAILED (по завершении выполнения)
- COMPLETED/FAILED → READY (для повторного использования)

### План задач
- DRAFT → APPROVED (после проверки)
- APPROVED → IN_PROGRESS (при запуске)
- IN_PROGRESS → COMPLETED/BLOCKED/FAILED (по завершении или при ошибках)

### Контрольная точка
- PENDING → RUNNING (при запуске проверки)
- RUNNING → PASSED/FAILED/WARNING (по завершении проверки)
- FAILED → BLOCKED (для блокирующих проверок)
