# Краткое руководство по реализации решений на основе анализа различий между Qwen Code CLI и Claude Code Orchestrator Kit

## Обзор

Это руководство поможет вам быстро начать внедрение решений, основанных на анализе различий между Qwen Code CLI и Claude Code Orchestrator Kit. Оно охватывает основные компоненты и процессы, которые необходимо реализовать.

## Требования

- Установленный Qwen Code CLI
- Доступ к репозиторию проекта
- Права на создание файлов и веток в проекте
- Знание архитектуры Qwen Code CLI

## Быстрый старт

### 1. Структура агентов

#### Создание нового оркестратора
1. Создайте файл в `.qwen/agents/` с именем формата `orc_{domain}_имя_агента.md`
2. Добавьте YAML заголовок:
```yaml
---
name: orc_{domain}_имя_агента
description: Описание функциональности
tools:
 - read_file
 - write_file
 - task
---
```
3. Опишите логику оркестратора в теле файла

#### Создание нового воркера
1. Создайте файл в `.qwen/agents/` с именем формата `work_{domain}_имя_агента.md`
2. Добавьте YAML заголовок:
```yaml
---
name: work_{domain}_имя_агента
description: Описание функциональности
tools:
 - read_file
 - write_file
---
```
3. Опишите логику воркера в теле файла

### 2. Фаза планирования

#### Запуск фазы планирования
1. При запуске процесса реализации автоматически выполняется анализ задач
2. Система определяет необходимые типы агентов
3. Назначает исполнителей для каждой задачи
4. Создает план выполнения в формате JSON

### 3. Механизм создания агентов

#### Использование meta-agent-v3
1. Запустите агент с командой создания нового агента
2. Укажите тип (оркестратор/воркер), домен и функциональность
3. Агент автоматически создаст новый файл агента с правильной структурой

### 4. Стандартизация отчетов

#### Формат отчета агента
Каждый агент должен генерировать отчет со следующими разделами:
- Заголовок с метаданными
- Исполнительное резюме
- Выполненная работа
- Внесенные изменения
- Результаты валидации
- Метрики выполнения
- Обнаруженные ошибки
- Следующие шаги
- Артефакты выполнения

### 5. Контрольные точки качества

#### Добавление новой контрольной точки
1. Создайте скрипт в `scripts/quality-gates/`
2. Определите команду проверки и критерии прохождения
3. Укажите, является ли проверка блокирующей
4. Интегрируйте в процесс валидации

### 6. Навыки (Skills)

#### Создание нового навыка
1. Создайте файл в `.qwen/skills/{название_навыка}/SKILL.md`
2. Добавьте YAML заголовок с описанием
3. Определите инструменты, которые может использовать навык
4. Опишите логику выполнения задачи

### 7. Интеграция с MCP

#### Использование MCP серверов
1. Определите, какие MCP серверы подходят для задачи
2. Добавьте рекомендации в план задачи
3. Используйте соответствующие инструменты в агентах

### 8. Оркестрация

#### Создание сложного рабочего процесса
1. Определите фазы выполнения
2. Создайте оркестратора для координации
3. Используйте файлы планов для передачи данных между агентами
4. Реализуйте возвращение управления после выполнения задачи

## Примеры

### Пример оркестратора для домена testing
```markdown
---
name: orc_testing_bug_analysis
description: Оркестратор для анализа багов в тестировании
tools:
 - read_file
 - write_file
 - task
---

# Оркестратор анализа багов в тестировании

## Фаза 0: Планирование
- Анализ задач на тестирование
- Назначение воркеров для поиска багов

## Фаза 1: Выполнение
- Создание плана для bug-hunter
- Запуск анализа кода

## Фаза 2: Валидация
- Проверка результатов
- Генерация отчета
```

### Пример воркера для домена backend
```markdown
---
name: work_backend_api_validator
description: Воркер для валидации API в бэкенде
tools:
 - read_file
 - write_file
---

# Воркер валидации API бэкенда

## Инструкции
1. Прочитать спецификацию API
2. Проверить соответствие реализации спецификации
3. Сгенерировать отчет о несоответствиях
```

## Следующие шаги

После завершения базовой настройки:
1. Протестируйте созданные агенты
2. Проверьте работу фазы планирования
3. Убедитесь, что отчеты генерируются в правильном формате
4. Проверьте контрольные точки качества
5. Протестируйте интеграцию с MCP серверами
6. Проверьте работу оркестрации
