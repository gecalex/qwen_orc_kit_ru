# Исследование для реализации решений на основе анализа различий между Qwen Code CLI и Claude Code Orchestrator Kit

## Решенные вопросы и технические решения

### 1. Система именования агентов
- **Решение**: Внедрить плоскую, но структурированную систему именования: `orc_{domain}_имя_агента.md` для оркестраторов и `work_{domain}_имя_агента.md` для воркеров
- **Обоснование**: Позволяет легко определить тип и область применения агента, сохраняя плоскую структуру директорий
- **Рассмотренные альтернативы**: Иерархическая структура директорий, префиксы без домена, суффиксы вместо префиксов

### 2. Фаза планирования (Phase 0)
- **Решение**: Реализовать автоматизированную фазу планирования с анализом задач и назначением исполнителей
- **Обоснование**: Повышает эффективность разработки за счет автоматического анализа и распределения задач
- **Рассмотренные альтернативы**: Ручное планирование, частичная автоматизация, отсутствие фазы планирования

### 3. Механизм динамического создания агентов
- **Решение**: Создать meta-agent-v3 для генерации новых агентов по шаблонам
- **Обоснование**: Позволяет быстро создавать новые агенты с правильной структурой и функциональностью
- **Рассмотренные альтернативы**: Ручное создание агентов, использование универсальных агентов, шаблоны без автоматизации

### 4. Стандартизация отчетов
- **Решение**: Внедрить унифицированный формат отчетов от агентов
- **Обоснование**: Упрощает обработку результатов и интеграцию с другими компонентами
- **Рассмотренные альтернативы**: Различные форматы для разных агентов, отсутствие стандартизации, использование только JSON

### 5. Контрольные точки качества
- **Решение**: Расширить систему контрольных точек с блокирующими и неблокирующими проверками
- **Обоснование**: Обеспечивает высокое качество кода и предотвращает критические ошибки
- **Рассмотренные альтернативы**: Отсутствие проверок, только неблокирующие проверки, централизованная система проверок

### 6. Набор навыков (skills)
- **Решение**: Расширить набор инструментов для часто используемых операций
- **Обоснование**: Повышает переиспользуемость кода и упрощает разработку агентов
- **Рассмотренные альтернативы**: Включение всех функций в агентов, использование внешних библиотек, отсутствие навыков

### 7. Интеграция с внешними сервисами
- **Решение**: Улучшить интеграцию с MCP серверами с рекомендациями по использованию
- **Обоснование**: Повышает эффективность использования внешних ресурсов и инструментов
- **Рассмотренные альтернативы**: Отсутствие интеграции, фиксированные связи, ручное управление

### 8. Оркестрация и координация
- **Решение**: Расширить возможности оркестрации с поддержкой итеративной логики и возвращения управления
- **Обоснование**: Позволяет создавать сложные рабочие процессы с эффективной координацией
- **Рассмотренные альтернативы**: Простые линейные процессы, отсутствие оркестрации, централизованное управление

## Архитектурные решения

### Структура агентов
- Поддержка плоской структуры с префиксами для типизации
- Использование доменов: dev, frontend, backend, doc, testing, research и т.д.
- Стандартизированные YAML-заголовки для всех агентов

### Механизмы оркестрации
- Возвращение управления от агентов к оркестратору
- Использование файлов планов для координации
- Поддержка итеративных процессов

### Контрольные точки качества
- Использование навыков для выполнения проверок
- Поддержка блокирующих и неблокирующих проверок
- Интеграция с системой отчетов

## Технологические рекомендации

### Для разработчиков агентов
- Использование шаблонов при создании новых агентов
- Следование стандартам именования
- Интеграция с системой отчетов

### Для интеграции с MCP
- Использование рекомендованных серверов в зависимости от типа задачи
- Интеграция MCP в плановые файлы
- Учет MCP при координации задач

## Выводы
Все неопределенности из спецификации разрешены. План реализации включает 8 ключевых компонентов, каждый из которых имеет четкое техническое решение и обоснование. Архитектура учитывает специфику Qwen Code CLI и обеспечивает совместимость с существующими компонентами.
