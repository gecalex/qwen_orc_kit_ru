# Контракты для обновления архитектурных документов

## Обзор

В рамках реализации рекомендаций из отчета сравнения CLAUDE.md и QWEN.md определены следующие контракты для обновления архитектурных документов:

## Контракты

### 1. Контракт на добавление паттерна health-workflow

**Описание**: Определяет структуру и содержание раздела о паттерне health-workflow в архитектурном документе

**Требования**:
- Должны быть описаны команды `/health-bugs`, `/health-security`, `/health-cleanup`, `/health-deps`
- Должны быть указаны цели и задачи каждой команды
- Должны быть описаны сценарии использования
- Должны быть определены ответственные за выполнение команд

**Формат**: Markdown-разметка в соответствии с существующей структурой документа

### 2. Контракт на внедрение подхода "библиотека вперёд"

**Описание**: Определяет структуру и содержание раздела о подходе "библиотека вперёд"

**Требования**:
- Должно быть дано определение подхода
- Должны быть описаны шаги для реализации подхода
- Должны быть приведены примеры поиска существующих решений
- Должны быть указаны критерии выбора между существующим и новым решением

**Формат**: Markdown-разметка в соответствии с существующей структурой документа

### 3. Контракт на расширение фазы планирования

**Описание**: Определяет структуру и содержание раздела о фазе планирования

**Требования**:
- Должны быть описаны этапы фазы планирования
- Должны быть определены роли и ответственности
- Должны быть описаны процессы назначения исполнителей
- Должны быть определены критерии завершения фазы планирования

**Формат**: Markdown-разметка в соответствии с существующей структурой документа

### 4. Контракт на добавление соглашений проекта

**Описание**: Определяет структуру и содержание раздела о соглашениях проекта

**Требования**:
- Должны быть описаны соглашения об организации файлов
- Должны быть указаны стандарты кода
- Должны быть определены критерии выбора агентов
- Должны быть описаны операции с системами (например, Supabase)

**Формат**: Markdown-разметка в соответствии с существующей структурой документа

### 5. Контракт на адаптацию упоминаний Supabase

**Описание**: Определяет процесс адаптации или удаления упоминаний Supabase

**Требования**:
- Должны быть идентифицированы все упоминания Supabase
- Должно быть принято решение об адаптации или удалении каждого упоминания
- Должны быть предложены альтернативы для Qwen Code CLI
- Должны быть обновлены соответствующие разделы документа

**Формат**: Markdown-разметка в соответствии с существующей структурой документа

### 6. Контракт на добавление информации о контрольных точках качества

**Описание**: Определяет структуру и содержание раздела о контрольных точках качества

**Требования**:
- Должны быть описаны все контрольные точки (Gates)
- Должны быть определены критерии прохождения каждой контрольной точки
- Должны быть указаны ответственные за проверку
- Должны быть описаны процессы при несоответствии критериям

**Формат**: Markdown-разметка в соответствии с существующей структурой документа
