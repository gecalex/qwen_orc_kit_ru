# Data Model: Архитектурные решения для Qwen Code CLI

## Сущности

### 1. Агент (Agent)
**Описание:** Специализированный ИИ-компонент, выполняющий определенные задачи в рамках системы

**Поля:**
- name: строка (уникальное имя агента)
- description: строка (описание функциональности агента)
- tools: массив строк (инструменты, доступные агенту)
- color: строка (цветовая схема для визуализации)

**Отношения:**
- Может вызывать навыки (Skills)
- Может быть частью оркестратора (Orchestrator)

**Правила проверки:**
- Имя агента должно быть уникальным
- Должен иметь хотя бы один инструмент

### 2. Навык (Skill)
**Описание:** Переиспользуемая утилита, выполняющая конкретную задачу

**Поля:**
- name: строка (уникальное имя навыка)
- description: строка (описание функциональности навыка)
- instructions: строка (подробные инструкции по использованию)

**Отношения:**
- Может быть вызван одним или несколькими агентами

**Правила проверки:**
- Имя навыка должно быть уникальным
- Должен иметь инструкции по использованию

### 3. Команда (Command)
**Описание:** Интерфейс взаимодействия с системой через префикс "/"

**Поля:**
- name: строка (имя команды)
- description: строка (описание функциональности команды)
- handoffs: массив объектов (передачи управления другим агентам)

**Отношения:**
- Может вызывать одного или нескольких агентов
- Может использовать навыки

**Правила проверки:**
- Имя команды должно быть уникальным
- Должна иметь описание

### 4. Оркестратор (Orchestrator)
**Описание:** Компонент, координирующий работу нескольких агентов

**Поля:**
- name: строка (уникальное имя оркестратора)
- description: строка (описание функциональности оркестратора)
- agents: массив строк (агенты, которыми управляет оркестратор)

**Отношения:**
- Управляет несколькими агентами
- Может использовать навыки

**Правила проверки:**
- Имя оркестратора должно быть уникальным
- Должен управлять хотя бы одним агентом

### 5. Контрольная точка качества (Quality Gate)
**Описание:** Автоматическая проверка соответствия определенным критериям

**Поля:**
- name: строка (уникальное имя контрольной точки)
- description: строка (описание проверки)
- script_path: строка (путь к скрипту проверки)
- threshold: число (пороговое значение для прохождения)

**Отношения:**
- Может быть частью процесса реализации
- Может использовать навыки

**Правила проверки:**
- Имя контрольной точки должно быть уникальным
- Должен существовать скрипт проверки

### 6. MCP-конфигурация (MCP Configuration)
**Описание:** Настройки для динамического переключения между различными конфигурациями MCP

**Поля:**
- name: строка (уникальное имя конфигурации)
- description: строка (описание сценария использования)
- servers: массив объектов (MCP-серверы, включенные в конфигурацию)
- token_usage: число (примерное количество токенов, используемых конфигурацией)

**Отношения:**
- Может быть использована командой переключения
- Связана с системой агентов

**Правила проверки:**
- Имя конфигурации должно быть уникальным
- Должна иметь хотя бы один сервер

### 7. Worktree
**Описание:** Изолированное рабочее дерево Git для параллельной разработки

**Поля:**
- name: строка (имя worktree)
- path: строка (путь к директории worktree)
- branch: строка (ветка, на которой основано дерево)
- status: строка (статус: active, inactive, merged)

**Отношения:**
- Связано с системой Git
- Может использоваться для изолированной разработки фич

**Правила проверки:**
- Имя worktree должно быть уникальным
- Должен существовать путь к директории

### 8. Вебхук (Webhook)
**Описание:** Механизм уведомлений о событиях в системе

**Поля:**
- event_type: строка (тип события, которое вызывает вебхук)
- url: строка (URL для отправки уведомления)
- headers: объект (HTTP-заголовки для запроса)
- payload_template: строка (шаблон тела запроса)

**Отношения:**
- Связан с событиями в системе
- Может использоваться командами и агентами

**Правила проверки:**
- URL должен быть корректным
- Должен быть указан тип события
