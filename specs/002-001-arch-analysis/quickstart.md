# Quickstart: Быстрый старт с архитектурными улучшениями Qwen Code CLI

## Введение

Этот документ описывает, как быстро начать использовать новые архитектурные решения, внедренные в проект на основе анализа Claude Code Orchestrator Kit.

## Установка и настройка

### 1. Клонирование репозитория

```bash
# Клонируйте репозиторий проекта
git clone <ваш_репозиторий>
cd <название_проекта>
```

### 2. Настройка среды

```bash
# Убедитесь, что у вас установлены необходимые инструменты
# Qwen Code CLI
# Git
# Bash (для скриптов)
```

## Использование новых агентов

### 1. Запуск агента проверки безопасности

```bash
# Используйте команду для запуска агента проверки безопасности
/health-security
```

Агент выполнит сканирование кодовой базы на наличие уязвимостей безопасности и сгенерирует отчет.

### 2. Запуск агента анализа зависимостей

```bash
# Используйте команду для запуска агента анализа зависимостей
/health-deps
```

Агент проанализирует зависимости проекта и предложит обновления или улучшения.

### 3. Запуск агента обнаружения мертвого кода

```bash
# Используйте команду для запуска агента обнаружения мертвого кода
/health-cleanup
```

Агент найдет и предложит удалить неиспользуемый код.

## Использование контрольных точек качества

### 1. Запуск проверки покрытия кода тестами

```bash
# Запустите проверку покрытия кода тестами
/run-quality-gate coverage
```

Скрипт проверит, соответствует ли покрытие кода тестами заданному порогу.

### 2. Запуск проверки безопасности

```bash
# Запустите проверку безопасности
/run-quality-gate security
```

Скрипт выполнит аудит безопасности проекта.

### 3. Запуск проверки размера бандла

```bash
# Запустите проверку размера бандла
/run-quality-gate bundle-size
```

Скрипт проверит, не превышает ли размер бандла заданный лимит.

## Использование MCP-конфигураций

### 1. Переключение между конфигурациями

```bash
# Используйте скрипт для переключения между MCP-конфигурациями
./scripts/orchestration-tools/switch-mcp.sh
```

Выберите подходящую конфигурацию в зависимости от ваших текущих задач:
- BASE: минимальная конфигурация (~600 токенов)
- SUPABASE: для работы с базой данных (~2500 токенов)
- FRONTEND: для UI/UX разработки (~2000 токенов)
- FULL: все серверы (~5000 токенов)

После смены конфигурации обязательно перезапустите Qwen Code CLI.

## Использование системы worktree

### 1. Создание нового worktree для фичи

```bash
# Создайте новый worktree для разработки фичи
/worktree-create feature/new-auth-system
```

### 2. Работа в worktree

```bash
# Переключитесь в директорию worktree и работайте как обычно
cd .worktrees/feature/new-auth-system
# Выполняйте изменения, коммитьте и т.д.
```

### 3. Управление worktree

```bash
# Просмотр списка worktree
/worktree-list

# Удаление worktree после слияния
/worktree-remove feature/new-auth-system
```

## Использование вебхуков

### 1. Настройка вебхуков

Создайте или обновите файл `.qwen/settings.local.json`:

```json
{
  "hooks": {
    "Stop": [
      {
        "type": "command",
        "command": "notify-send 'Qwen Code CLI' 'Task completed!'"
      }
    ]
  }
}
```

### 2. Типы поддерживаемых событий

- `Stop`: событие завершения задачи
- `Start`: событие начала задачи
- `Error`: событие ошибки

## Примеры использования

### Пример 1: Полный цикл разработки новой фичи

1. Создайте worktree для новой фичи:
```bash
/worktree-create feature/payment-integration
```

2. Переключитесь на подходящую MCP-конфигурацию:
```bash
./scripts/orchestration-tools/switch-mcp.sh
# Выберите опцию 5 (FRONTEND) если работаете с UI
```

3. Разработайте фичу в worktree.

4. Запустите контрольные точки качества:
```bash
/run-quality-gate security
/run-quality-gate coverage
```

5. После завершения работы, удалите worktree:
```bash
/worktree-remove feature/payment-integration
```

### Пример 2: Регулярное обслуживание проекта

1. Запустите агента проверки безопасности:
```bash
/health-security
```

2. Запустите агента анализа зависимостей:
```bash
/health-deps
```

3. Запустите агента обнаружения мертвого кода:
```bash
/health-cleanup
```

4. Запустите контрольные точки качества:
```bash
/run-quality-gate security
/run-quality-gate bundle-size
```
