# Модель данных для улучшения архитектуры Qwen Orchestrator Kit

## Сущности системы

### 1. Агент (Agent)
- **Поля**:
  - `id`: уникальный идентификатор агента
  - `name`: имя агента (следует формату `типы}_{домен}_{имя}`)
  - `type`: тип агента (оркестратор/воркер)
  - `domain`: домен агента (dev, frontend, backend, testing, research и т.д.)
  - `description`: описание функций агента
  - `tools`: список разрешенных инструментов
  - `color`: цветовая метка для визуализации
  - `created_date`: дата создания
  - `last_updated`: дата последнего обновления

- **Правила проверки**:
  - Имя должно следовать формату `типы}_{домен}_{имя}`
  - Тип должен быть либо "оркестратор", либо "воркер"
  - Описание должно быть не пустым
  - Список инструментов должен быть валидным

### 2. Навык (Skill)
- **Поля**:
  - `id`: уникальный идентификатор навыка
  - `name`: имя навыка
  - `description`: описание функций навыка
  - `usage_context`: сценарии применения
  - `input_format`: формат входных данных
  - `output_format`: формат выходных данных
  - `examples`: примеры использования
  - `created_date`: дата создания
  - `last_updated`: дата последнего обновления

- **Правила проверки**:
  - Имя должно быть уникальным
  - Описание должно быть не пустым
  - Форматы ввода/вывода должны быть валидными

### 3. MCP-конфигурация (MCP Configuration)
- **Поля**:
  - `id`: уникальный идентификатор конфигурации
  - `name`: имя конфигурации
  - `type`: тип (Context7, Supabase, Sequential Thinking, Playwright, shadcn, n8n)
  - `settings`: параметры конфигурации
  - `description`: описание назначения
  - `associated_agents`: список агентов, использующих эту конфигурацию
  - `created_date`: дата создания
  - `last_updated`: дата последнего обновления

- **Правила проверки**:
  - Имя должно быть уникальным
  - Тип должен быть одним из разрешенных
  - Параметры должны быть валидными для выбранного типа

### 4. Задача (Task)
- **Поля**:
  - `id`: уникальный идентификатор задачи
  - `title`: заголовок задачи
  - `description`: описание задачи
  - `assigned_agent`: агент, назначенный для выполнения
  - `status`: статус (новая, в работе, выполнена, отменена)
  - `priority`: приоритет (низкий, средний, высокий, критический)
  - `created_date`: дата создания
  - `due_date`: дата выполнения
  - `dependencies`: зависимости от других задач

- **Правила проверки**:
  - Заголовок должен быть не пустым
  - Статус должен быть одним из разрешенных
  - Приоритет должен быть одним из разрешенных

### 5. Отчет (Report)
- **Поля**:
  - `id`: уникальный идентификатор отчета
  - `agent_id`: идентификатор агента, создавшего отчет
  - `type`: тип отчета (ошибки, безопасности, качества и т.д.)
  - `content`: содержимое отчета
  - `status`: статус обработки (новый, в работе, обработан)
  - `created_date`: дата создания
  - `processed_date`: дата обработки

- **Правила проверки**:
  - Содержимое должно быть не пустым
  - Тип должен быть одним из разрешенных
  - Статус должен быть одним из разрешенных

## Отношения между сущностями

- Агент может использовать множество навыков
- Навык может использоваться множеством агентов
- MCP-конфигурация может использоваться множеством агентов
- Задача может быть назначена одному агенту
- Отчет создается одним агентом
