# Быстрый старт для улучшения архитектуры Qwen Orchestrator Kit

## Введение

Этот документ описывает, как начать работу с улучшенной архитектурой Qwen Orchestrator Kit, включающей мета-агента, систему тегирования, MCP-конфигурации и другие улучшения.

## Установка

1. Убедитесь, что у вас установлена Qwen Code CLI
2. Клонируйте репозиторий:
   ```bash
   git clone <repository-url>
   cd qwen_orchestrator_kit
   ```

3. Установите зависимости:
   ```bash
   # Установка необходимых компонентов
   ```

## Использование мета-агента

Для создания нового агента используйте мета-агента:

```bash
# Пример создания нового агента
task '{
  "subagent_type": "meta-agent",
  "description": "Создание нового агента",
  "prompt": "Создай новый воркер агент для обработки задач в домене dev с именем work_dev_example_worker"
}'
```

Мета-агент автоматически создаст новый агент с необходимыми компонентами и документацией.

## Улучшенная навигация по агентам

Для улучшения навигации по агентам используйте альтернативные механизмы (вне YAML-заголовков):

```yaml
---
name: example-agent
description: Пример агента
tools:
 - read_file
 - write_file
 - edit
 - run_shell_command
 - grep_search
 - glob
 - web_search
 - web_fetch
 - read_many_files
 - todo_write
 - skill
 - task
color: blue
---
```

Для поиска агентов используйте поиск по описанию и другим полям:

```bash
# Поиск агентов по ключевым словам в описании
grep -r "dev\|utility\|helper" .qwen/agents/
```

## Использование MCP-конфигураций

Для выбора подходящей MCP-конфигурации используйте команду:

```bash
# Пример выбора конфигурации
skill: "select-mcp-server" {
  "taskType": "documentation",
  "domain": "react",
  "requirements": ["context-search", "pattern-validation"]
}
```

Доступные конфигурации:
- `context7`: для поиска контекста и документации библиотек
- `supabase`: для работы с базами данных Supabase
- `sequential-thinking`: для сложных рассуждений
- `playwright`: для автоматизации браузерных тестов
- `shadcn`: для работы с компонентами shadcn
- `n8n`: для управления рабочими процессами

## Автоматический анализ задач

Система автоматически анализирует поступающие задачи и назначает подходящие агенты. Для этого используется внутренний механизм, который анализирует:

- Тип задачи (баг, фича, улучшение и т.д.)
- Домен задачи (dev, frontend, backend и т.д.)
- Требуемые навыки
- Приоритет задачи

## Запуск тестов

Для проверки корректности работы компонентов запустите автоматические тесты:

```bash

# Запуск тестов для конкретного компонента
```

## Примеры использования

### Создание нового агента для обработки ошибок

```bash
task '{
  "subagent_type": "meta-agent",
  "description": "Создание агента для обработки ошибок",
  "prompt": "Создай новый воркер агент для обработки ошибок в домене health с именем work_health_bug_handler"
}'
```

### Использование альтернативных механизмов для улучшения навигации

После создания агента улучшите его описание для поиска:

```yaml
---
name: work_health_bug_handler
description: Агент для обработки ошибок, задач по обеспечению работоспособности системы и обслуживанию
# ... остальные поля
---
```

### Выбор подходящей MCP-конфигурации

Для задач, требующих доступа к документации библиотек:

```bash
skill: "select-mcp-server" {
  "taskType": "documentation",
  "domain": "react",
  "requirements": ["context-search", "pattern-validation"]
}
```

## Следующие шаги

4. Добавьте теги к новым агентам для улучшения навигации
5. Используйте подходящие MCP-конфигурации для ваших задач
