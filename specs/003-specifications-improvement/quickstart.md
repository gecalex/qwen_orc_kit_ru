# Быстрый старт для улучшения архитектуры Qwen Orchestrator Kit

## Введение

Этот документ описывает, как начать работу с улучшенной архитектурой Qwen Orchestrator Kit, включающей мета-агента, систему тегирования, MCP-конфигурации и другие улучшения.

   ```

3. Установите зависимости:

   ```

## Использование мета-агента

Для создания нового агента используйте мета-агента:

```bash
# Пример создания нового агента
task '{
  "subagent_type": "work_dev_meta_agent",
  "description": "Создание нового агента",
  "prompt": "Создай новый воркер агент для обработки задач в домене dev с именем work_dev_example_worker и описанием 'Агент для обработки примерных задач'"
}'
```

Мета-агент автоматически создаст новый агент с необходимыми компонентами и документацией.

### Примеры создания агентов разных типов

#### Создание воркера для домена dev
```bash
task '{
  "subagent_type": "work_dev_meta_agent",
  "description": "Создание воркера для анализа кода",
  "prompt": "Создай новый воркер агент для анализа кода в домене dev с именем work_dev_code_analyzer и описанием 'Агент для анализа кода и выявления потенциальных проблем'"
}'
```

#### Создание оркестратора для домена testing
```bash
task '{
  "subagent_type": "work_dev_meta_agent",
  "description": "Создание оркестратора для обеспечения качества",
  "prompt": "Создай новый оркестратор агент для координации задач тестирования в домене testing с именем orc_testing_quality_assurer и описанием 'Оркестратор для координации задач обеспечения качества'"
}'
```

#### Создание воркера для домена security
```bash
task '{
  "subagent_type": "work_dev_meta_agent",
  "description": "Создание агента для анализа безопасности",
  "prompt": "Создай новый воркер агент для анализа безопасности в домене security с именем work_security_vulnerability_scanner и описанием 'Агент для сканирования уязвимостей и анализа безопасности кода'"
}'
```

### Параметры создания агента

При создании агента через мета-агента укажите следующие параметры:

- **Тип агента**: `orc` (оркестратор) или `work` (воркер)
- **Домен**: `dev`, `frontend`, `backend`, `testing`, `research`, `security`
- **Имя**: в формате kebab-case (например: `my-agent-name`)
- **Описание**: краткое описание функций агента

### Проверка созданного агента

После создания агента автоматически запускаются проверки качества:
- Проверка корректности YAML заголовка
- Проверка структуры агента
- Проверка соответствия стандартам именования

Созданный агент будет доступен в директории `.qwen/agents/` и готов к использованию в проекте.

## Использование навыков

Навыки (skills) - это переиспользуемые функции, которые агенты могут вызывать для выполнения часто используемых операций. Вот несколько примеров использования навыков:

### Пример 1: Создание заголовка отчета
```markdown
skill: "generate-report-header" {
  "reportType": "Анализ безопасности",
  "agentName": "security-analyzer",
  "version": "1.0.0",
  "timestamp": "2026-01-31T10:00:00Z",
  "status": "success",
  "additionalMetadata": {
    "vulnerabilitiesFound": 3,
    "critical": 1
  }
}
```

### Пример 2: Выполнение контрольной точки качества
```markdown
skill: "run-quality-gate" {
  "command": "npx tsc --noEmit",
  "isBlocking": true,
  "expectedResult": "No type errors found",
  "timeout": 300
}
```

### Пример 3: Форматирование таблицы markdown
```markdown
skill: "format-markdown-table" {
  "headers": ["Файл", "Статус", "Время"],
  "rows": [
    ["src/app.ts", "Успешно", "2.1s"],
    ["src/utils.ts", "Ошибка", "1.5s"]
  ],
  "alignments": ["left", "center", "right"]
}
```

### Пример 4: Разбор логов ошибок
```markdown
skill: "parse-error-logs" {
  "logType": "typescript",
  "logContent": "src/example.ts(5,10): error TS2532: Object is possibly 'undefined'.",
  "includeWarnings": true
}
```

### Пример 5: Разбор статуса Git
```markdown
skill: "parse-git-status" {
  "gitStatusOutput": "M  src/example.ts\nA  .qwen/docs/readme.md\n?? tests/example.test.ts",
  "includeUntracked": true,
  "includeStaged": true,
  "includeUnstaged": true
}
```

### Пример 6: Извлечение версии
```markdown
skill: "extract-version" {
  "input": "Версия приложения: 2.4.1",
  "inputType": "text",
  "format": "semver"
}
```

Навыки позволяют агентам выполнять часто используемые операции последовательно и надежно, обеспечивая согласованность в подходах к обработке данных и выполнению задач.

## Улучшенная навигация по агентам

Для улучшения навигации по агентам используйте альтернативные механизмы (вне YAML-заголовков):

```yaml
---
name: example-agent
description: Пример агента
tools:
 - read_file
 - write_file
 - edit
 - run_shell_command
 - grep_search
 - glob
 - web_search
 - web_fetch
 - read_many_files
 - todo_write
 - skill
 - task
color: blue
---
```

Для поиска агентов используйте поиск по описанию и другим полям:

```bash
# Поиск агентов по ключевым словам в описании
grep -r "dev\|utility\|helper" .qwen/agents/
```

## Использование MCP-конфигураций

Для выбора подходящей MCP-конфигурации используйте команду:

```bash
# Пример выбора конфигурации
skill: "select-mcp-server" {
  "taskType": "documentation",
  "domain": "react",
  "requirements": ["context-search", "pattern-validation"]
}
```

Доступные конфигурации:
- `context7`: для поиска контекста и документации библиотек
- `supabase`: для работы с базами данных Supabase
- `sequential-thinking`: для сложных рассуждений
- `playwright`: для автоматизации браузерных тестов
- `shadcn`: для работы с компонентами shadcn
- `n8n`: для управления рабочими процессами

## Автоматический анализ задач

Система автоматически анализирует поступающие задачи и назначает подходящие агенты. Для этого используется внутренний механизм, который анализирует:

- Тип задачи (баг, фича, улучшение и т.д.)
- Домен задачи (dev, frontend, backend и т.д.)
- Требуемые навыки
- Приоритет задачи

## Запуск тестов

Для проверки корректности работы компонентов запустите автоматические тесты:

```bash
# Запуск всех тестов
bash .qwen/scripts/validation/check-standards-compliance.sh

# Запуск тестов для конкретного компонента
# (указать конкретный скрипт тестирования)
```
