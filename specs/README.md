# Директория спецификаций

В этой директории находятся спецификации функций и компонентов проекта.

## Структура

Каждая спецификация находится в своей поддиректории с именем в формате `номер-название-функции`:

```
specs/
├── 001-user-authentication/
│   └── specification.md
├── 002-task-management/
│   └── specification.md
└── ...
```

## Создание новой спецификации

Для создания новой спецификации:

1. Используйте команду `speckit.specify`:
   ```
   speckit.specify "Описание новой функции"
   ```

2. Или создайте вручную директорию и файл спецификации на основе шаблона из `.specify/templates/spec-template.md`

## Процесс разработки на основе спецификаций

1. **Создание спецификации** - формализация требований к функции
2. **Уточнение спецификации** - уточнение деталей с помощью `speckit.clarify`
3. **Планирование реализации** - создание плана с помощью `speckit.plan`
4. **Генерация задач** - создание задач для реализации с помощью `speckit.tasks`
5. **Реализация** - выполнение задач с помощью `speckit.implement`

## Проверка качества спецификаций

Спецификации должны соответствовать следующим критериям:

- Все обязательные разделы заполнены
- Требования ясны и однозначны
- Условия успеха измеримы
- Сценарии использования реалистичны
- Риски идентифицированы и оценены

Для проверки качества можно использовать навык `specification-analyzer`.
