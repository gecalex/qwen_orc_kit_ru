Подготовить проект Qwen Code Orchestrator Kit к использованию в качестве шаблона для создания новых проектов с помощью агентов.

## Подробное описание задач

### Задача 1: Скрипт очистки (cleanup-template.sh)
Создать скрипт, который:
- Удаляет директории `.git`, `specs`, `state`, `examples/specs`, `FEATURE_DIR`
- Удаляет временные файлы и артефакты
- Удаляет файлы конфигурации среды разработки
- Оставляет только шаблонные компоненты

### Задача 2: Скрипт инициализации (init-project.sh)
Создать скрипт, который:
- Принимает имя нового проекта
- Заменяет плейсхолдеры на реальные значения
- Инициализирует Git репозиторий
- Создает начальную структуру проекта
- Предоставляет инструкции по началу работы

### Задача 3: Адаптация документации
Обновить документацию, чтобы:
- Она описывала шаблон, а не конкретный проект
- Содержала инструкции по использованию шаблона
- Включала примеры для различных типов проектов

## Требования к шаблону

### Функциональные требования
- Шаблон должен быть легко клонируемым
- Шаблон должен включать все необходимые компоненты системы Speckit
- Шаблон должен быть готов к использованию сразу после инициализации
- Шаблон должен включать инструменты качества и контрольные точки

### Нефункциональные требования
- Шаблон должен быть безопасным (без специфичных данных)
- Шаблон должен быть легко настраиваемым
- Шаблон должен быть документированным
- Шаблон должен быть протестированным

## Критерии готовности релиза
- [ ] Все специфичные файлы удалены
- [ ] Плейсхолдеры заменены корректно
- [ ] Скрипт инициализации работает корректно
- [ ] Новый проект может быть успешно создан из шаблона
- [ ] Все компоненты системы работают в новом проекте
- [ ] Документация обновлена
- [ ] Примеры использования предоставлены
