# Отчет о файлах, которые не должны попадать в релиз как шаблон проекта

## Цель
Проект Qwen Code Orchestrator Kit предназначен для использования в качестве отправной точки для создания других проектов с помощью агентов. Поэтому в релиз не должны попадать файлы, специфичные для текущего проекта.

## Категории файлов

### 1. Файлы, которые НЕ должны попадать в релиз как шаблон:

#### 1.1. Системные файлы Git
- `.git/` - директория системы контроля версий
- `.gitignore` - может содержать специфичные для проекта исключения

#### 1.2. Файлы истории и состояния
- `.git/` - вся директория (включая историю коммитов, конфигурацию репозитория)
- `.gitattributes` - настройки Git для конкретного проекта

#### 1.3. Файлы специфичные для текущего проекта
- `specs/` - директория с конкретными спецификациями проекта
- `state/` - директория с состоянием системы выполнения задач
- `examples/specs/` - примеры спецификаций для текущего проекта
- `FEATURE_DIR/` - директория с конкретными реализациями функций
- `workspace/` - рабочая область проекта (если существует)

#### 1.4. Временные и артефактные файлы
- `*.log` - лог-файлы
- `*.tmp` - временные файлы
- `*.bak` - резервные копии
- `*.backup` - резервные копии файлов
- `__pycache__/` - кэш Python
- `*.pyc` - скомпилированные файлы Python
- `.DS_Store` - файлы macOS
- `Thumbs.db` - файлы Windows

#### 1.5. Файлы конфигурации среды разработки
- `.vscode/` - настройки VS Code для текущего проекта
- `.idea/` - настройки IntelliJ/PyCharm для текущего проекта
- `.editorconfig` - может содержать специфичные настройки проекта

#### 1.6. Файлы данных и артефактов сборки
- `dist/` - директория с результатами сборки
- `build/` - директория сборки
- `target/` - директория сборки (для Java проектов)
- `node_modules/` - зависимости (должны быть в .gitignore)
- `.venv/`, `venv/`, `env/` - виртуальные окружения

### 2. Файлы, которые НУЖНО адаптировать для шаблона:

#### 2.1. Файлы с именами/идентификаторами проекта
- `README.md` - может содержать название текущего проекта
- `QWEN.md` - может содержать специфичные для проекта указания
- `docs/` - документация может содержать специфичные для проекта примеры
- `pyproject.toml` или `package.json` - могут содержать имя проекта

#### 2.2. Файлы с путями к специфичным компонентам
- Скрипты, содержащие жестко закодированные пути
- Конфигурационные файлы с проект-специфичными настройками

### 3. Файлы, которые ДОЛЖНЫ остаться в шаблоне:

#### 3.1. Структура и шаблоны
- `.specify/templates/` - шаблоны для спецификаций (должны быть обобщенными)
- `.qwen/agents/` - шаблоны агентов (должны быть обобщенными)
- `.qwen/commands/` - команды (должны быть обобщенными)
- `.qwen/skills/` - навыки (должны быть обобщенными)

#### 3.2. Скрипты и инструменты
- `scripts/` - скрипты для оркестрации (должны быть универсальными)
- `docs/architecture/` - архитектурная документация (должна быть обобщенной)

#### 3.3. Основные файлы системы
- `QWEN.md` - поведенческая парадигма оркестратора
- `CONTRIBUTING.md` - руководство по участию (должно быть обобщенным)
- `INSTALLATION.md` - инструкция по установке (должна быть обобщенной)

## Рекомендации по подготовке шаблона

### 1. Создание шаблонной структуры
- Заменить все упоминания конкретного проекта на параметры/плейсхолдеры
- Создать скрипты инициализации для настройки нового проекта
- Обеспечить возможность настройки через переменные окружения или конфигурационные файлы

### 2. Очистка от специфичных данных
- Удалить или заменить все файлы с конкретными спецификациями
- Удалить временные и артефактные файлы
- Убедиться, что в шаблоне нет чувствительных данных

### 3. Создание скрипта инициализации
- Скрипт для настройки нового проекта на основе шаблона
- Скрипт для замены плейсхолдеров на реальные значения
- Скрипт для инициализации Git в новом проекте

## План действий

1. Создать скрипт очистки проекта от специфичных файлов
2. Адаптировать файлы с проект-специфичными данными
3. Создать скрипт инициализации нового проекта
4. Протестировать шаблон на примере создания нового проекта
5. Подготовить документацию по использованию шаблона
