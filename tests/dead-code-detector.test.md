# Тесты для агента обнаружения мертвого кода

## Цель
Проверить функциональность агента dead-code-detector и его способность находить неиспользуемый код.

## Тестовые сценарии

### Сценарий 1: Проверка на наличие неиспользуемых функций
**Входные данные**: Файл с кодом, содержащий функцию, которая нигде не вызывается
**Ожидаемый результат**: Агент должен обнаружить и отметить эту функцию как мертвый код

### Сценарий 2: Проверка на наличие неиспользуемых переменных
**Входные данные**: Файл с кодом, содержащий переменную, которая объявлена, но не используется
**Ожидаемый результат**: Агент должен обнаружить и отметить эту переменную как мертвый код

### Сценарий 3: Проверка на наличие закомментированного кода
**Входные данные**: Файл с длинными блоками закомментированного кода
**Ожидаемый результат**: Агент должен обнаружить и отметить эти блоки как потенциальный мертвый код

### Сценарий 4: Проверка на наличие недостижимого кода
**Входные данные**: Файл с кодом, который находится после оператора return или throw
**Ожидаемый результат**: Агент должен обнаружить и отметить этот код как недостижимый

### Сценарий 5: Проверка на наличие неиспользуемых импортов
**Входные данные**: Файл с импортами, которые не используются в коде
**Ожидаемый результат**: Агент должен обнаружить и отметить эти импорты как мертвый код

## Критерии успеха
- Все тестовые сценарии проходят успешно
- Агент корректно идентифицирует мертвый код
- Агент не выдает ложных срабатываний
- Отчет о мертвом коде генерируется в правильном формате
