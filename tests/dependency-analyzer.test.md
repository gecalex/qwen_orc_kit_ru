# Тесты для агента анализа зависимостей

## Цель
Проверить функциональность агента dependency-analyzer и его способность анализировать зависимости проекта.

## Тестовые сценарии

### Сценарий 1: Проверка на наличие устаревших зависимостей
**Входные данные**: Файл package.json с устаревшими зависимостями
**Ожидаемый результат**: Агент должен обнаружить и перечислить устаревшие зависимости с указанием актуальных версий

### Сценарий 2: Проверка на наличие уязвимостей в зависимостях
**Входные данные**: Файл package.json с зависимостями, содержащими известные уязвимости
**Ожидаемый результат**: Агент должен обнаружить и перечислить зависимости с уязвимостями с указанием типа и уровня уязвимости

### Сценарий 3: Проверка на наличие неиспользуемых зависимостей
**Входные данные**: Проект с установленными, но неиспользуемыми зависимостями
**Ожидаемый результат**: Агент должен обнаружить и перечислить неиспользуемые зависимости

### Сценарий 4: Проверка на наличие отсутствующих зависимостей
**Входные данные**: Код, использующий зависимости, которые не объявлены в файле зависимостей
**Ожидаемый результат**: Агент должен обнаружить и перечислить отсутствующие зависимости

### Сценарий 5: Проверка конфликтов зависимостей
**Входные данные**: Файл зависимостей с конфликтующими версиями
**Ожидаемый результат**: Агент должен обнаружить и описать конфликты зависимостей

## Критерии успеха
- Все тестовые сценарии проходят успешно
- Агент корректно идентифицирует проблемы с зависимостями
- Агент не выдает ложных срабатываний
- Отчет о зависимостях генерируется в правильном формате
