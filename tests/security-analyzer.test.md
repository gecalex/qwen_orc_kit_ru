# Тесты для агента проверки безопасности

## Цель
Проверить функциональность агента security-analyzer и его способность обнаруживать уязвимости в коде.

## Тестовые сценарии

### Сценарий 1: Проверка на наличие жестко закодированных паролей
**Входные данные**: Файл с кодом, содержащий строку "password = 'hardcoded_password'"
**Ожидаемый результат**: Агент должен обнаружить и отметить эту строку как потенциальную уязвимость

### Сценарий 2: Проверка на SQL-инъекции
**Входные данные**: Файл с кодом, содержащий небезопасное построение SQL-запроса
**Ожидаемый результат**: Агент должен обнаружить потенциальную уязвимость к SQL-инъекции

### Сценарий 3: Проверка на XSS-уязвимости
**Входные данные**: Файл с кодом, содержащий небезопасную вставку пользовательских данных в HTML
**Ожидаемый результат**: Агент должен обнаружить потенциальную XSS-уязвимость

### Сценарий 4: Проверка на insecure deserialization
**Входные данные**: Файл с кодом, содержащий небезопасную десериализацию данных
**Ожидаемый результат**: Агент должен обнаружить потенциальную уязвимость

### Сценарий 5: Проверка на утечку чувствительной информации
**Входные данные**: Файл с кодом, содержащий логирование чувствительных данных
**Ожидаемый результат**: Агент должен обнаружить потенциальную утечку информации

## Критерии успеха
- Все тестовые сценарии проходят успешно
- Агент корректно идентифицирует уязвимости
- Агент не выдает ложных срабатываний
- Отчет об уязвимостях генерируется в правильном формате
